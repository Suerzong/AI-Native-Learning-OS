- `profile.md`
- `plan/ability-framework.md`
- `learning-progress.md`
- `plan/roadmap.md`
- `course-index.md`
- `plan/school-calendar.md`
- `plan/timetable.md`
- `plan/schedule.md`
- `plan/daily-plan.md`

如果某个文件不存在，请明确说明缺失文件，并在不编造信息的前提下继续完成计划。

# 任务目标

你需要根据以下信息生成今天的学习启动计划：

1. 用户长期方向：Edge AI / 嵌入式 AI
2. 十二大能力模块
3. 当前完整能力画像
4. 当前阶段 roadmap
5. 今日课程时间
6. 今日课余时间
7. 当前学习目标
8. 昨日或最近一次 daily-plan / daily-log 中未完成的任务

---

# 工作要求

## 1. 识别今天的学习条件

请先整理：

- 今天日期
- 今天是否有课
- 今天课程时间段
- 今天可用于自主学习的课余时间
- 今天适合安排的学习强度
- 今天应优先推进的能力模块

如果 `plan/schedule.md` 中没有今日课程安排，请提示用户补充课程时间，但不要停止任务。

## 2. 读取当前能力地图和进度

请根据 `plan/ability-framework.md` 和 `learning-progress.md` 判断：

- 当前最强模块
- 当前短板模块
- 当前阶段最应该推进的模块
- 今天不适合过早推进的模块

不要夸大用户能力。能力状态必须区分：已了解、已学习、已实践、初步掌握、已掌握、已应用。

## 2.5. 复习状态检查

运行 `python3 tools/review_system.py status` 检查待复习卡片。

如果到期卡片 >= 5 张：
- 在今日任务列表中插入一条复习任务：
  ```
  复习任务：交互式主动回忆
  运行 `/review` 进行 15 分钟主动回忆复习
  对应模块：各课程覆盖
  优先级：高（错题优先于近期新学）
  ```

如果 neural-networks 课程有多张卡片到期且超过 2 天未复习：
- 在任务列表中加入特别标记："⚠️ 神经网络复习卡片已积压 [X] 天，今天务必处理"

如果到期卡片 < 5 张：
- 说明当前复习压力较小，可降低复习任务优先级，但仍在晚间安排 5 分钟轻量回顾。

## 2.6. UCloud 待办任务检查

检查 `ucloud-tasks/YYYY-MM-DD.json` 是否存在（今天日期）。

如果存在，读取其中的待办任务列表，筛选截止日期在 **3 天内**（含今天）的任务。

### 对于每个临近截止的任务

在今日任务列表中插入：

```
⚠️ UCloud 任务：[任务标题]
类型：作业/考试/问卷/互评
截止时间：[deadline]
链接：[url]
优先级：高（不可延期）
```

### 通知格式

在"今日学习原则"中加入提醒：

- 🔴 今日有 [N] 个 UCloud 待办任务将在 3 天内截止，必须安排时间完成。
- 🟡 今日有 [N] 个 UCloud 待办任务在 1 周内截止，注意提前规划。
- 🟢 当前无临近截止的 UCloud 任务，可以专注于主线学习。

### 汇总格式

在"今日最低完成线"中加入 UCloud 任务：

```
[UCloud] 完成 XXX 作业（截止 YYYY-MM-DD HH:MM）
```

如果 `ucloud-tasks/YYYY-MM-DD.json` 不存在，跳过此检查并注明"今日尚未抓取 UCloud 任务，请运行 python3 tools/ucloud_monitor.py"。

## 3. 生成今日学习主线

今天的计划必须围绕十二大能力模块展开。

请输出：

- 今日主线
- 今日重点模块
- 今日最低完成线
- 今日加分任务
- 今日输出物

每个学习任务都必须标注对应模块。

## 4. 生成今日任务安排

任务数量不要过多，优先保证可执行。请按照以下结构安排：

### 主任务
用于推进当前阶段最重要的能力模块。

### 复习任务
用于巩固已学但还不够稳定的内容。

### 输出任务
必须形成可沉淀的东西，例如笔记、代码、实验记录、错题、总结。

### 预备任务
为下一阶段能力做轻量铺垫。

# 文件更新要求

如果当前处于 Agent 模式并允许编辑文件：

1. 更新 `plan/daily-plan.md`
2. 创建 `plan/record/daily/YYYY-MM-DD.md`，写入今日启动计划
3. **不要修改 `learning-progress.md`**，除非用户明确说明今天已经完成学习并要求复盘更新

# 自动推送

文件更新完成后，执行以下 git 操作自动同步到远程仓库：

```bash
git add -A && git commit -m "$(cat <<'EOF'
[start-day] YYYY-MM-DD 今日学习计划：XXX

Co-Authored-By: Claude Opus 4.7 <noreply@anthropic.com>
EOF
)" && git push origin main
```

提交信息格式要求：
- 以 `[start-day]` 开头，加上今天的日期
- 简要说明今日主线任务和重点模块
- 示例：`[start-day] 2026-05-07 今日学习计划：STM32 串口通信 + 数学复习`

如果 git 操作失败（如网络问题），提示用户但不要中断任务。
