请执行"突发事件更新与今日计划调整"任务。

$ARGUMENTS

本任务不使用 `events.md`。所有突发事件都必须直接整合进 `plan/daily-plan.md`。

---

# 一、必须先读取的文件

- `profile.md`
- `plan/ability-framework.md`
- `learning-progress.md`
- `plan/roadmap.md`
- `course-index.md`
- `plan/school-calendar.md`
- `plan/timetable.md`
- `plan/schedule.md`
- `plan/daily-plan.md`

如果某个文件不存在，请明确说明缺失文件，并在不编造信息的前提下继续处理。

---

# 二、任务目标

1. 识别突发事件的类型
2. 判断该事件影响的日期和时间段
3. 检查它是否与校历、课程表、综合日程、今日学习计划发生冲突
4. 将该突发事件直接写入 `plan/daily-plan.md`
5. 重新评估今日可用学习时间
6. 重新生成调整后的今日学习计划
7. 保留今日最低完成线
8. 必要时压缩、推迟、拆分或取消原计划中的部分任务
9. 所有调整后的学习任务仍然必须围绕 Edge AI / 嵌入式 AI 长期目标
10. 所有学习任务都必须标注对应的十二大能力模块
11. **不要修改 `learning-progress.md`**，除非用户明确说明该事件对应已经完成的学习成果

---

# 三、突发事件类型

请将用户提供的事件归入以下类型之一：临时会议、调课/停课、作业截止、考试/测验、实验/课程设计、社团/班级事务、个人事务、身体状态变化、学习任务变更、其他。

---

# 四、优先级判断规则

**高优先级：** 考试、作业截止、必须参加的课程/实验/会议、身体不适、影响当天大块时间的事件、与今日主任务严重冲突的安排。

**中优先级：** 可调整但需要完成的任务、社团/班级/项目讨论、非紧急作业、普通复习任务。

**低优先级：** 可选活动、可延后的自学任务、非必要安排、加分任务。

---

# 五、今日计划调整原则

1. 优先保留今日最低完成线
2. 优先保留当前阶段主线任务
3. 删除或推迟加分任务
4. 缩短低优先级任务
5. 将大任务拆成更小任务
6. 如果时间不足，生成"最低可执行版本"
7. 不要因为突发事件就完全放弃当天学习
8. 不要随机安排任务，必须基于 `plan/ability-framework.md`、`learning-progress.md` 和 `plan/roadmap.md` 判断学习优先级
9. 所有任务都必须服务于 Edge AI / 嵌入式 AI 长期路线
10. 不要夸大用户能力，不要把临时事件写成学习成果

# 自动推送

文件更新完成后，执行以下 git 操作自动同步到远程仓库：

```bash
git add -A && git commit -m "$(cat <<'EOF'
[update-event] YYYY-MM-DD 突发事件：XXX

Co-Authored-By: Claude Opus 4.7 <noreply@anthropic.com>
EOF
)" && git push origin main
```

提交信息格式要求：
- 以 `[update-event]` 开头，加上今天的日期
- 简要说明突发事件和计划调整
- 示例：`[update-event] 2026-05-07 突发：电路实验改期，调整今日计划`

如果 git 操作失败（如网络问题），提示用户但不要中断任务。
