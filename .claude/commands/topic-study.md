description: 设置阶段性主题学习任务，并围绕该主题生成学习周期规划和执行安排
---

请执行"主题学习规划"任务。

$ARGUMENTS

用户将提供一个阶段性学习主题，例如：
- 接下来 7 天学习 C 语言指针与结构体
- 接下来 14 天主攻 STM32 串口通信
- 本周集中学习 FreeRTOS
- 未来 30 天入门 Python
- 两周内完成某门课程的某几章内容

用户输入通常包括：学习主题、学习周期、希望达到的目标、每天大概可用时间、是否有课程/考试/作业/项目等限制、是否需要每天围绕该主题安排任务。

---

# 一、必须先读取的文件

- `profile.md`
- `plan/ability-framework.md`
- `learning-progress.md`
- `course-index.md`
- `plan/roadmap.md`
- `plan/school-calendar.md`
- `plan/timetable.md`
- `plan/schedule.md`
- `plan/daily-plan.md`

如果某个文件不存在，请明确说明缺失文件，并在不编造信息的前提下继续完成规划。

---

# 二、任务目标

1. 判断该主题属于十二大能力模块中的哪些模块
2. 读取用户当前能力画像，判断该主题的当前基础水平
3. 判断该主题是否适合用户当前阶段
4. 如果适合，制定完整主题学习周期规划
5. 如果不完全适合，说明原因，并给出降级版或前置补课版方案
6. 将学习周期拆分为阶段目标
7. 将阶段目标拆分为每日任务
8. 所有安排都必须围绕该主题展开
9. 每个任务必须有明确输出物
10. 每个任务必须标注对应十二大能力模块
11. 必须考虑课程表、校历、已有日程和当前 roadmap
12. 必须说明该主题对 Edge AI / 嵌入式 AI 长期路线的作用
13. 如果允许编辑文件，请更新 `plan/daily-plan.md`（仅更新今日任务，不要覆盖整个文件）
14. **不要直接更新 `learning-progress.md`**，除非用户明确说明已经完成学习并形成能力

---

# 三、主题学习规则

核心原则：在指定学习周期内，所有学习安排都优先服务于这个主题，但不能完全破坏长期能力地图和现实课程安排。

1. 不要安排与主题无关的大任务
2. 可以保留少量必要复习任务
3. 可以安排前置知识补课
4. 可以安排配套实践任务
5. 可以安排输出任务（笔记、代码、实验记录、题目、总结）
6. 每天必须有一个围绕主题的主任务
7. 学习周期较长时需要分阶段推进
8. 每个阶段必须有验收标准
9. 任务不能只写"学习 XXX"，必须写清具体动作
10. 不要安排过满，要保留缓冲时间
11. 如果主题过大必须主动拆解
12. 如果主题过难必须先安排前置基础

---

# 四、主题与十二大能力模块映射

请将主题映射到十二大模块之一或多个。输出时必须说明：
- 主题主要对应哪些模块
- 次要支撑哪些模块
- 当前用户在这些模块中的已有基础
- 本次主题学习预期推进到什么状态

---

# 五、学习周期拆分规则

- **1-3 天**：生成短期冲刺计划，每日 1 个主任务 + 1 个输出物
- **4-7 天**：一周主题计划，分入门梳理、实践巩固、总结输出三个阶段
- **8-14 天**：两周主题计划，分基础补齐、核心学习、实践训练、总结复盘四个阶段
- **15-30 天**：月度主题计划，按周拆分，每周设目标和验收
- **超过 30 天**：长期主题路线，按月或阶段拆分，只给出阶段规划，不要过早细化每天任务

---

# 六、输出物要求

每个主题学习任务都必须有输出物：笔记、思维导图、代码、实验记录、仿真结果、题目练习、错题整理、项目 demo、课程章节总结、技术报告、学习复盘、可写入 `learning-progress.md` 的能力增量描述。

不要只安排"看视频""看书""了解一下"。必须尽量转化为：能写出来、能跑起来、能解释出来、能复现出来、能沉淀到仓库里。

---

# 七、文件更新

- 如果需要更新路线信息，在 `plan/roadmap.md` 中新增或更新"当前主题学习任务"部分
- 更新 `plan/daily-plan.md` 中的今日任务安排（不要覆盖整个文件）
- **不要更新 `learning-progress.md`**

---

# 自动推送

文件更新完成后，执行以下 git 操作自动同步到远程仓库：

```bash
git add -A && git commit -m "$(cat <<'EOF'
[topic-study] YYYY-MM-DD 主题学习规划：XXX

Co-Authored-By: Claude Opus 4.7 <noreply@anthropic.com>
EOF
)" && git push origin main
```

提交信息格式要求：
- 以 `[topic-study]` 开头，加上今天的日期
- 简要说明主题名称、周期和目标
- 示例：`[topic-study] 2026-05-07 主题规划：14天 STM32 串口通信专项`

如果 git 操作失败（如网络问题），提示用户但不要中断任务。
