# 会话前准备

1. 先执行 `date '+%Y-%m-%d %H:%M:%S %A'` 获取当前时间。
2. 读取 `plan/spaced-review-policy.md` 确认复习规则。
3. 运行 `python3 tools/review_system.py ingest` 收集最新的知识卡片（从 mistakes/mastery-tracker/daily记录 中抽取）。
4. 运行 `python3 tools/review_system.py quiz --limit 8` 获取今日待复习卡片。

如果返回空数组 `[]`，回复：
> 今天没有待复习卡片，恭喜！休息一下或者去看看新的知识吧。

然后直接结束。

# 开场展示

解析 JSON 输出，向用户展示总览：

```
📋 今日到期卡片：[N] 张
📌 本轮审查：[M] 张（优先级：未复测错题 > 低正确率 > 近期新学 > 长期巩固）

准备开始，每张卡片我会提问，你回答后我评分纠正。
说 "skip" 跳过，说 "end" 提前结束。
```

# 逐卡问答循环

对每张卡片按顺序处理。**每次只出一张卡，等待用户回答后再评估，不要提前出下一张。**

## 出题规则

根据卡片的 `source` 和内容字段，生成不同类型的问题：

如果卡片属于某个课程，必须先检查该课程的 `course-config.md`。若课程声明 `source_mode: textbook-only`：

- 复习问题只能使用该课程允许源中的已学内容。
- 不把归档旧资料、代码实操、工程部署、外部教程卡片重新混入当前复习。
- 如果卡片明显来自该课程的禁用源或归档源，问题改为：“这张卡片来自旧归档资料，不进入当前主课复习。请只回顾当前课本中对应的小节标题。”并给低优先级处理。
- 错题来源也只能要求口述课本正确理解、公式含义、小推导或课本习题思路，不要求写代码。

| source | 条件 | 出题方式 |
|---|---|---|
| mistakes | 有 `corrective` | 「请完成纠正动作：**{corrective}**」 |
| mistakes | 仅 `weakness`（无 corrective） | 「关于 **{module}**：{weakness}。**正确做法是什么？**」 |
| mastery-tracker | 有 `weakness`（非 "-"） | 「用自己的话解释 **{module}**。特别注意：{weakness}」 |
| mastery-tracker | 无有效 weakness | 「用自己的话解释 **{module}** 的核心要点」 |
| daily | — | 「回顾一下你所学的内容：**{content}**，能讲清楚吗？」 |
| inbox | — | 「你之前提到过：**{content}**。现在能独立解释一遍吗？」 |

**出题策略**：
- 错题来源：要求**口述正确做法**；仅当课程允许代码任务时才要求写代码
- mastery-tracker：要求**主动回忆**，引导用户用自己的话讲出来
- 如果是代码类内容（如 BST delete、链表操作），要求口述伪代码或核心逻辑
- 问题应简短（1-3 句话），不要啰嗦

## 展示格式

每次出题时展示：

```
─── 卡片 [n]/[total] [{course}] {module} [{source_tag}] ───
问题：{具体问题}
```

其中 source_tag：
- mistakes → `⚠️错题`
- mastery-tracker → `📊技能`
- daily → `📝复盘`
- inbox → `📥收件`

## 等待用户回答

用户回答后：
- 回答正确且完整 → 4 分
- 基本正确，有小遗漏 → 3 分
- 部分正确，关键点遗漏 → 2 分
- 完全错误或不知道 → 1 分

评估后即时展示纠正或确认，然后调用 `python3 tools/review_system.py feedback "{module}={score}"` 更新间隔。

## 特殊情况处理

- 用户说 **"skip"** → 给 2 分（模糊），继续下一张。连续 skip 3 次时提醒 "已经连续跳过 3 张，要继续吗？"
- 用户说 **"end"** → 立即结束审查，剩余卡片不评分。
- 用户说 **"重评：N"** → 如果用户认为你的评分不准，接受调整。

# 结束总结

审查完成后展示总结：

```
─── 审查完成 ───
本次审查：[M] 张卡片
清晰(4分)：[n] | 基本(3分)：[n] | 模糊(2分)：[n] | 遗忘(1分)：[n]
加权正确率：[X]%

⚠️ 需要重点巩固的模块：
- {module} ({reason})
- ...

下次到期高峰：[date]，共 [N] 张卡片。

建议：{一句话建议}
```

正确率计算：(4分×1 + 3分×0.75 + 2分×0.25 + 1分×0) / 总数 × 100%

如果该文件不存在或无法写入，跳过（不中断）并告知用户。

# 约束

- 不修改 `learning-progress.md`。复习不等于能力提升，只代表巩固。
- 单张卡片最多 3 分钟，总时长控制在 15 分钟内。
- 错题来源的卡片优先展示。
- 卡片数量由 `quiz --limit 8` 控制，无需人工数。
- 如果用户在中途说"先记下来"或其他非回答内容，正常处理但不计分。

# 自动推送

复习会话完成后，执行以下 git 操作自动同步到远程仓库：

```bash
git add -A && git commit -m "$(cat <<'EOF'
[review] YYYY-MM-DD 交互复习：X/Y 张卡片，正确率 Z%

Co-Authored-By: Claude Opus 4.7 <noreply@anthropic.com>
EOF
)" && git push origin main
```

提交信息填写规则：
- YYYY-MM-DD 替换为今天日期
- X/Y 替换为实际审查卡片数
- Z% 替换为加权正确率
- 如果 git 操作失败（如网络问题），提示用户但不中断任务。
