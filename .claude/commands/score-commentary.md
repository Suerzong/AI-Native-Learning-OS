请批改用户今天写的每日时评文章。

$ARGUMENTS

---

# 前置步骤

1. 执行 `date '+%Y-%m-%d'` 获取今天日期
2. 如果用户指定了日期（如 `/score-commentary 2026-05-16`），使用用户指定的日期

---

# 批改流程

## 1. 读取文章

进入 `daily-commentary/YYYY-MM-DD/`，读取所有 `topic-*.md` 文件。

## 2. 逐篇评分

按 `daily-commentary/scoring-rubric.md` 的 5 维度标准，每篇独立评分：

| 维度 | 满分 | 核心考察 |
|------|------|----------|
| 观点明确性 | 10 | 有没有明确立场？还是和稀泥？ |
| 逻辑严谨性 | 10 | 论证链是否完整？有没有跳跃或矛盾？ |
| 论据充分性 | 10 | 至少2个高质量论据？数据/案例/类比？ |
| 语言表达力 | 10 | 精准、简洁、有节奏感？ |
| 思想深度 | 10 | 有独到见解还是停留在常识层面？ |

总分 50 分，等级：
- 45-50: 优秀
- 35-44: 良好
- 25-34: 及格
- < 25: 不及格

## 3. 判断是否已写

如果某篇文章用户还没写（内容仍为"请在下方作答"或类似占位文字），跳过不评分，在最终总结中提醒"题X 尚未作答"。

## 4. 写入评分到文件末尾

对每篇已写的文章，在文件末尾追加：

```
---

## 评分结果 — YYYY-MM-DD

| 维度 | 得分 |
|------|------|
| 观点明确性 | X/10 |
| 逻辑严谨性 | X/10 |
| 论据充分性 | X/10 |
| 语言表达力 | X/10 |
| 思想深度 | X/10 |
| **总分** | **XX/50** |

**等级**：XXX

**亮点**：一句话，你做得最好的地方是什么

**问题**：一句话，最大的短板是什么

**改进建议**：一句可操作的具体建议
```

## 5. 输出评分汇总

在对话中展示汇总表：

```
─── YYYY-MM-DD 时评打分 ───

题一「题目」：X + X + X + X + X = XX/50 [等级]
  亮点：...
  问题：...
  → 改进：...

题二「题目」：X + X + X + X + X = XX/50 [等级]
  ...

题三「题目」：尚未作答，跳过。

─── 总结 ───
今日均分：XX/50
整体评价：一句话总结今日表现
```

---

# 评分要求

- **严格公正**：不要因为学生身份就放水，论据弱就是弱
- **具体可操作**：改进建议不能说"多练习""多思考"这类废话，要给出具体方向
- **批评带建设性**：指出问题的同时给出怎么做
- **尊重原文**：评分写在文末分隔线后，不修改用户原文

---

# 自动提交

评分完成后执行：

```bash
git add -A && git commit -m "$(cat <<'EOF'
[score-commentary] YYYY-MM-DD 时评打分：均分 XX/50

Co-Authored-By: Claude Opus 4.7 <noreply@anthropic.com>
EOF
)" && git push origin main
```

如果 git 操作失败，提示用户但不中断任务。
