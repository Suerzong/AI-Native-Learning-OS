$ARGUMENTS

本任务不使用 `events.md`。所有突发事件更新都直接记录到 `plan/daily-plan.md` 和 `plan/record/daily/YYYY-MM-DD.md`。

---

# 一、必须先读取的文件

请先读取以下文件：

- `profile.md`
- `plan/ability-framework.md`
- `learning-progress.md`
- `course-index.md`
- `plan/roadmap.md`
- `plan/school-calendar.md`
- `plan/timetable.md`
- `plan/schedule.md`
- `plan/daily-plan.md`

同时请尝试读取当天的 daily 记录文件：`plan/record/daily/YYYY-MM-DD.md`。如果当天 daily 文件不存在，请在允许编辑文件时创建它。

如果某个文件不存在，请明确说明缺失文件，并在不编造信息的前提下继续完成复盘。

---

# 二、任务目标

你需要根据用户输入的今日实际情况，完成以下任务：

1. 对照 `plan/daily-plan.md`，记录今天的任务完成情况
2. 记录今天实际学习到的内容
3. 记录今天的突发事件或突发事件更新
4. 判断突发事件是否影响了原有计划
5. 判断未完成任务的原因
6. 判断今天是否形成了可写入 `learning-progress.md` 的新能力
7. 必要时更新 `learning-progress.md`
8. 更新 `plan/daily-plan.md` 中今日任务状态
9. 创建或更新 `plan/record/daily/YYYY-MM-DD.md`
10. 如果今日实际情况影响了原计划，则重新规划后续任务
11. 所有学习成果和后续规划都必须围绕 Edge AI / 嵌入式 AI 的十二大能力模块展开

---

# 三、能力记录原则

更新 `learning-progress.md` 时必须谨慎。

**只有**当用户明确表示完成了以下内容时，才可以更新 `learning-progress.md`：
- 系统学习、实验、代码、仿真、调试、项目、课程设计、独立复现
- 明确形成了新的稳定知识点或工程能力

必须区分能力状态：已了解、已学习、已实践、初步掌握、已掌握、已应用。

**不要把以下内容写成能力提升：**
- 今天只是计划了某项学习
- 今天只是被安排了某件事
- 今天只是因为突发事件调整了计划
- 今天只是粗略看了一眼
- 今天任务没有真正完成
- 用户没有明确说明自己学会了什么

不要夸大能力。不要编造用户没有明确说明的学习成果。

---

# 四、今日任务完成状态分类

请把 `plan/daily-plan.md` 中的今日任务和用户实际完成情况进行对照，标注状态：已完成、部分完成、未完成、推迟、取消、被突发事件影响、需要明天继续、不再需要继续。

对于每个任务，请判断：原计划是什么、实际完成了什么、完成度如何、未完成原因是什么、是否需要重新安排、对应十二大能力模块是什么。

---

# 五、突发事件处理规则

突发事件类型包括：临时会议、调课/停课、作业截止、考试/测验、实验/课程设计、社团/班级事务、个人事务、身体状态变化、学习任务变更、其他。

请判断突发事件是否影响了原计划（占用了原定学习时间、导致主任务未完成、导致最低完成线未达成、让原计划优先级变化等）。

---

# 六、未完成任务原因判断

归因类型：时间不足、突发事件占用时间、任务难度高于预期、身体状态不佳、注意力不足、计划安排过满、前置知识不足、工具/环境/资料问题、任务优先级变化、其他。

归因时不要责备用户，重点是帮助用户调整下一步计划。

---

# 七、重新规划规则

如果突发事件、任务未完成或学习结果影响了原计划，请重新规划。

重新规划时必须遵守：
1. 优先保留当前阶段主线任务
2. 优先保留 `plan/roadmap.md` 中近期重点
3. 保留或重建最低完成线
4. 推迟加分任务
5. 将未完成的大任务拆成更小任务
6. 不要因为一天没完成就彻底打乱长期路线
7. 不要随机安排任务，必须基于 `plan/ability-framework.md`、`learning-progress.md`、`plan/roadmap.md` 判断优先级
8. 明确哪些任务转移到明天、本周后续、取消或降级
9. 所有重排任务必须标注对应十二大能力模块

# 九、文件编辑要求

- **必须更新** `plan/daily-plan.md`
- **必须创建或更新** `plan/record/daily/YYYY-MM-DD.md`
- **必须判断**是否更新 `learning-progress.md`（仅当形成了新能力时才更新）
- **不要创建** `events.md`
- **不要删除**今日原计划，应保留并标注实际完成情况
- **不要覆盖**长期路线 `plan/roadmap.md`，除非用户明确要求
- 如果原计划受影响，必须给出后续重排建议
- 所有重排建议必须对应十二大能力模块
- 所有内容必须服务于 Edge AI / 嵌入式 AI 长期目标

# 自动推送

文件更新完成后，执行以下 git 操作自动同步到远程仓库：

```bash
git add -A && git commit -m "$(cat <<'EOF'
[end-day] YYYY-MM-DD 每日复盘：XXX

Co-Authored-By: Claude Opus 4.7 <noreply@anthropic.com>
EOF
)" && git push origin main
```

提交信息格式要求：
- 以 `[end-day]` 开头，加上今天的日期
- 简要说明复盘结论和后续安排
- 示例：`[end-day] 2026-05-07 每日复盘：完成 STM32 串口实验，数学推迟至明天`

如果 git 操作失败（如网络问题），提示用户但不要中断任务。
