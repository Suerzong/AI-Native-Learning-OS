# 2026-05-11 AI 科技情报

## 今日一句话总览

今天最重要的信号是：端侧 AI 正在从“能不能跑”进入“如何在更多硬件上稳定、低成本、可部署地跑”。

## 5 行摘要

- ONNX Runtime v1.26.0 新增 RISC-V Vector 支持，这是 Edge AI 工具链覆盖开放硬件架构的重要信号。
- HN 热帖 "Local AI needs to be the norm" 说明本地推理、隐私和低延迟正在成为社区共识。
- Android 项目 Box 展示了 llama.cpp、whisper.cpp、stable-diffusion.cpp 的端侧多模态集成方式。
- ExecuTorch 代表 PyTorch 官方继续推进端侧部署工具链，值得长期跟踪。
- 今天你的主线仍然是神经网络基础：池化层、stride、输出尺寸和 LeNet-5，不要被新闻打断。

## Top 5 简报

1. **ONNX Runtime v1.26.0**：新增 RISC-V RVV CPU Execution Provider、WebGPU 增强和 .ort 内存映射加载。对你来说，先建立“模型部署需要推理引擎”的概念即可。
2. **Local AI needs to be the norm**：社区持续关注本地 AI，说明 Edge AI 方向的长期价值在增强。当前收藏做方向判断，不必被热点打断主线学习。
3. **Box Android 端侧 AI 套件**：把 LLM、语音识别和图像生成放到手机端运行。现在适合加入项目库，未来学完基础后再看源码。
4. **ExecuTorch v1.2.0**：PyTorch 官方端侧部署框架继续推进。现在只需要知道它在 “PyTorch -> 设备端运行” 链路中的位置。
5. **Agentic Search 检索论文**：提示 RAG / Agent 未来不会只靠语义相似度检索。当前了解即可，不影响今天 CNN 主线。

## 今日时代动向

- **端侧推理正在变成工程主线。** 未来岗位不会只问“会不会训练模型”，还会看你能不能把模型部署到真实设备。
- **硬件适配会越来越重要。** RISC-V、WebGPU、NPU、Jetson、RK3588 等方向都说明 AI 工程师需要懂一点系统和硬件。
- **你的学习路线是对的。** 现在把 CNN、NumPy/PyTorch、数据结构和嵌入式基础打稳，后面才能进入 ONNX、量化和端侧部署。

## 今天建议

1. 先把池化层输出尺寸、stride 和 LeNet-5 维度计算弄清楚。
2. 有 10 分钟的话，浏览 ONNX Runtime v1.26.0 Release Notes 的标题和目录。
3. 把 ExecuTorch、ONNX Runtime、端侧 Android AI 项目列入长期跟踪，不急着复现。
