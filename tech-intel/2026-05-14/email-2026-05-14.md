# 2026-05-14 AI 科技情报

## 今日一句话总览

PyTorch 2.12.0 昨天发布，torch.export 首次支持 MX 量化格式直接导出压缩模型——端侧部署的"最后一公里"正在被工具链铺平。

## 5 行摘要

- PyTorch 2.12.0 发布：torch.export 原生支持 MX 量化格式，可直接导出极低比特压缩模型用于端侧部署
- Needle 项目火爆：26M 参数模型蒸馏了 Gemini 的函数调用能力，可在微型设备上运行（HN 628分/GitHub 1293★）
- ExecuTorch v1.2.0：Cortex-M（STM32 级 MCU）成为一等嵌入式目标，集成 CMSIS-NN，4-bit 量化推理
- ONNX Runtime 1.26.0：新增 RISC-V Vector 扩展支持，覆盖 RISC-V 边缘设备
- 量化技术多元化：BFP、MX、INT8/INT4 并存，理解不同方案的 trade-off 是端侧工程师核心技能

## Top 5 简报

1. **PyTorch 2.12.0** — MX 量化格式 + torch.accelerator.Graph 统一后端 API + torch.cond in CUDA Graphs + 最低 CUDA 12.6 / C++20。MX 格式是 Edge AI 模型压缩的重要方向。[Release Notes](https://github.com/pytorch/pytorch/releases/tag/v2.12.0)

2. **Needle: 26M 函数调用模型** — 将 Gemini 工具调用蒸馏到 26M 参数 Gemma 模型，可在手机/嵌入式设备运行。知识蒸馏在 Edge AI 中的实战应用。[GitHub](https://github.com/cactus-compute/needle)

3. **ExecuTorch v1.2.0** — Cortex-M 成为一等目标，集成 CMSIS-NN；Metal 4-bit 量化；Vulkan int8 全面量化；新支持 Voxtral Realtime 语音和 Qwen3.5。[Release Notes](https://github.com/pytorch/executorch/releases/tag/v1.2.0)

4. **BFP 量化论文** — Block Floating Point 量化方案，同一 block 内数值共享缩放因子，与 INT8/MX 互补的另一种低精度推理路线。[arXiv](http://arxiv.org/abs/2605.12464v1)

5. **Elastic Attention Cores for ViT** — 降低 Vision Transformer 注意力计算复杂度，使 ViT 更适合端侧视觉推理。[arXiv](http://arxiv.org/abs/2605.12491v1)
