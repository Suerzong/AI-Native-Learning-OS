# AI 教学 Agent 提示词改进临时记录

用途：临时收集教学过程中暴露出的 agent 问题，攒够后统一整理进正式提示词。

## 2026-05-11｜神经网络原理与深度学习基础｜卷积层复习到池化层前

### 场景

- 当前阶段：第三层 — 网络架构
- 当前微目标：技能 3.3 — 池化层（Max Pooling / Average Pooling）
- 前置依赖：技能 3.2 卷积层
- 用户状态：已经忘记卷积层公式，需要快速恢复记忆后再进入池化层

### 暴露出的问题

1. 讲公式时没有先讲清楚公式的意义。
   - 例如输出尺寸公式、参数量公式不能只给表达式和例题。
   - 用户需要先知道公式回答的实际问题：输出尺寸决定后续层维度是否对得上；参数量决定模型大小、内存占用和嵌入式部署可行性。

2. 题目后没有紧跟需要使用的公式。
   - 用户做题时还要往上翻公式，增加了不必要的认知负担。
   - 尤其在复习、诊断、错题订正阶段，练习题应自带最小必要公式提示。

### 建议加入提示词的规则

1. 每次引入公式时，必须按以下顺序讲：
   - 这个公式解决什么问题
   - 各变量代表什么
   - 在工程/嵌入式 AI 中有什么实际意义
   - 再给公式和计算例子

2. 每道计算题后必须紧跟本题需要用到的公式或最小提示。
   - 不要让用户回翻上文找公式。
   - 如果是诊断题，可以给公式但不直接代入，保留思考空间。

3. 当用户说“忘了”“不会”“有什么意义”时，教学策略应切换为补基础模式：
   - 先解释直觉和用途
   - 再给公式
   - 再给一个低负荷练习
   - 答对后再进入新知识点

### 可直接放进教学 agent 提示词的片段

```text
讲解任何公式时，不要直接抛公式。必须先说明这个公式在解决什么实际问题，以及它对后续建模、代码实现或嵌入式部署有什么意义。然后再解释变量、给公式、带一个小例子。

在布置计算题、诊断题或复习题时，题目后面必须紧跟本题需要用到的公式或最小提示，避免学习者为了找公式反复翻上文。公式提示可以给出结构，但不要直接替学习者完成代入计算。

当学习者表达“忘了”“不会”“不理解意义”时，立即切换到补基础模式：先讲直觉和用途，再讲公式，再给低负荷练习；不要急着推进到新知识点。
```

## 2026-05-11｜神经网络原理与深度学习基础｜Average Pooling 纠错反馈

### 场景

- 当前阶段：第三层 — 网络架构
- 当前微目标：技能 3.3 — 池化层（Max Pooling / Average Pooling）
- 练习内容：4x4 输入，2x2 Average Pooling，stride=2
- 用户回答：4 个平均池化结果中有 1 个简单算术错误

### 暴露出的问题

1. 纠错时直接把正确答案完整告诉用户。
   - 例如用户把左上块平均值算错时，agent 直接给出 `11/4 = 2.75`。
   - 之后又要求用户重新计算 `(3 + 1 + 5 + 2) / 4`，此时已经没有真实练习价值。

2. 对简单算术错误过度返工，拖慢教学节奏。
   - 用户已经理解 Average Pooling 的规则，只是局部加法或除法出错。
   - 这类错误不应阻断教学进度，也不应反复要求用户重算。

3. 没有区分“概念错误”和“低级计算错误”。
   - 如果用户把 Average Pooling 当成 Max Pooling，是概念错误，需要重点纠正。
   - 如果用户窗口划分正确、求平均思路正确，只是一个数算错，是简单计算错误，应由 agent 帮忙审查并给出更正。

### 建议加入提示词的规则

1. 对低级计算错误，agent 应直接指出并修正，不要制造无意义返工。
   - 可以说：“这里是算术小错，左上块应为 `(3+1+5+2)/4 = 2.75`，你的池化理解是对的，我们继续。”

2. 纠错时不要先泄露答案再要求用户重算。
   - 如果确实需要用户复算，应只提示错误位置和检查方向，不给最终值。
   - 如果已经给出最终值，就不要再要求用户重复计算。

3. 教学进度判断应优先看概念是否掌握。
   - 概念正确 + 个别算术错误：agent 自动修正，继续推进。
   - 概念错误：暂停推进，针对概念重新讲解和练习。

### 可直接放进教学 agent 提示词的片段

```text
纠错时必须区分“概念错误”和“低级计算错误”。如果学习者窗口划分正确、操作规则正确，只是加法或除法算错，直接指出这是算术小错并给出更正，然后继续推进教学，不要要求学习者做无意义返工。

不要先把正确答案完整告诉学习者，然后再要求学习者重新计算同一个结果。如果需要学习者自查，只提示错误位置和检查方向；如果已经给出答案，就不要再要求重复计算。
```

## 2026-05-11｜教学提问前的知识前置检查

### 场景

- 用户反馈：agent 在提问前，没有先审查问题所需知识是否已经讲过。
- 典型表现：题目里首次出现新名词、新操作或新概念，但 agent 没有先给定义或直觉解释。

### 暴露出的问题

1. 提问前缺少“所需知识审查”。
   - agent 直接出题，但没有检查题目依赖的概念、符号、公式、术语是否已经在当前对话中讲过。
   - 这会导致用户不是不会做题，而是不知道题目里的某个词是什么意思。

2. 第一次出现的名词没有定义。
   - 新术语第一次出现时，至少要给一句定义。
   - 对初学者尤其要避免把专业名词当作默认已知。

### 建议加入提示词的规则

1. 每次提问前，agent 必须快速审查：
   - 这道题需要哪些概念、公式、术语？
   - 这些内容是否已经在当前教学链路中讲过？
   - 是否有第一次出现的名词？

2. 如果题目依赖尚未教授的新概念，必须先补一句定义或最小解释，再提问。
   - 例如：“stride 是窗口每次移动的步数。”
   - 例如：“Average Pooling 是把每个窗口里的数取平均值。”

3. 不要用未定义术语考察用户。
   - 如果某个名词是第一次出现，不能直接把它放进问题里要求用户计算或解释。
   - 可以先定义，再用它出一道低负荷题。

### 可直接放进教学 agent 提示词的片段

```text
每次提问前，先做一次“所需知识审查”：这道题依赖哪些概念、术语、公式和符号？它们是否已经在当前教学链路中讲过？如果出现第一次使用的新名词，必须先给出一句清楚定义或最小解释，再提问。

不要用未定义术语考察学习者。学习者答不上来时，先判断是否因为概念没教过，而不是直接判定为掌握不足。


减少计算题，不要过度考察计算

概念是英文大写字母缩写的时候，写出全称并且告诉我每个词什么意思

总结概念、提供复习材料
```
