正确：引用是别名，必须初始化、不能为 null、不能重新绑定；指针可以为 null、可以重新赋值。调用

### 2. [cpp] new/delete ⚠️错题
内容：错题：new/delete 和 malloc/free 混用
错误：用 `new` 分配后用 `free` 释放，或用 `malloc` 后用 `delete`
正确：`new` 调用构造函数，`delete` 调用析构函数；`malloc
弱点：用 `new` 分配后用 `free` 释放，或用 `malloc` 后用 `delete`
纠正任务：写一个带构造/析构打印的类，分别用 new/delete 和 malloc/free 管理，观察输出差异

### 3. [cpp] 类与对象 ⚠️错题
内容：错题：类定义末尾忘记分号
错误：`class Foo { ... }` 后面没有 `;`
正确：C++ 的 class 定义和 struct 定义一样，大括号后面必须有分号
弱点：`class Foo { ... }` 后面没有 `;`
纠正任务：每次写完类定义后检查分号

### 4. [cpp] 循环 ⚠️错题
内容：错题：range-based for 中不用引用导致无法修改元素
错误：`for (auto x : vec) x *= 2;` 发现 vec 中的元素没有变化
正确：要修改容器元素，必须用 `for (auto& x : vec) x *
弱点：`for (auto x : vec) x *= 2;` 发现 vec 中的元素没有变化
纠正任务：写一个程序分别用引用和不用引用修改 vector 元素，对比结果

### 5. [cpp] 多态 ⚠️错题
内容：错题：虚析构函数忘记声明
错误：通过基类指针 `delete` 派生类对象时，派生类的析构函数没有被调用
正确：如果一个类可能被继承并有多态删除，析构函数必须声明为 `virtual`
弱点：通过基类指针 `delete` 派生类对象时，派生类的析构函数没有被调用
纠正任务：写一个基类/派生类，基类析构函数不加 virtual，通过基类指针 delete，观察输出；然后加 virtual 对比

### 6. [cpp] 函数模板 / 类模板 ⚠️错题
内容：错题：模板定义放在 .cpp 文件中
错误：链接时报 undefined reference
正确：模板的定义通常放在头文件中，或者在 .cpp 底部显式实例化
弱点：链接时报 undefined reference
纠正任务：把模板定义移到头文件中，重新编译

### 7. [data-structures] 单链表 ⚠️错题
内容：错题：链表插入时断链
错误：在中间位置插入新节点后，后面的节点丢失
正确：必须先保存后继节点的地址（`newNode->next = prev->next`），再改变前驱指针（`prev->next = newNode`）
弱点：在中间位置插入新节点后，后面的节点丢失
纠正任务：画出插入前后链表的内存状态，标注每个指针的赋值顺序

### 8. [data-structures] 快速排序 ⚠️错题
内容：错题：快排分区指针方向搞反
错误：partition 后数组没有被正确分为"小于 pivot"和"大于 pivot"两部分
正确：以最右元素为 pivot 时，i 向右找 >= pivot 的，j 向左找 <= pivot 的，然后交换
弱点：partition 后数组没有被正确分为"小于 pivot"和"大于 pivot"两部分
纠正任务：手动模拟 partition([3,1,4,1,5,9,2,6]) 的每一步
