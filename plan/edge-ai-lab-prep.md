## 第 2 周：模型压缩实战（6/2–6/8）

### 本周目标
- 理解 INT8 量化的原理（对称量化、非对称量化、per-tensor vs per-channel）
- 对第 1 周训练的 CNN 做 PTQ 量化
- 对比 FP32 vs INT8 的精度、模型大小、推理速度

### 每日安排

| 日期 | 可用时段 | 任务 | 预计 |
|:--|:--|:--|:--|
| 6/2 一 | 18:10 后 (~3h) | **量化理论学习**。① 阅读 `materials/TFLite-模型转换与量化.md` 的 PTQ 部分 ② 阅读 `materials/PyTorch-ONNX-量化.md` 的量化部分 ③ 写一段笔记：用自己的话解释 `scale`、`zero_point`、`q = round(r/s) + z` 的含义 | 2.5h |
| 6/3 二 | 11:25-15:40 (4h) | **PyTorch 量化实战 1**。① 用 `torch.quantization.quantize_dynamic` 对 CNN 做动态量化 ② 对比量化前后的模型大小和准确率 ③ 理解为什么动态量化对 CNN 效果不如静态量化 | 3h |
| 6/4 三 | 15:30 后 (~6h) | **PyTorch 量化实战 2**。① 用 `torch.quantization.prepare_qat` + `convert` 走通 QAT 流程 ② 如果 QAT 太复杂，改走 PTQ（`torch.quantization.quantize` + `fuse_modules`）③ 目标：得到一个 INT8 模型，准确率下降 <3% | 4h |
| 6/5 四 | 9:35 后 (~8h) | **ONNX 导出 + 推理**。① 阅读 `materials/ONNX-导出与Runtime.md` ② 将 FP32 CNN 导出为 ONNX ③ 用 ONNX Runtime 加载并推理 ④ 对比 PyTorch 直接推理 vs ONNX Runtime 推理的速度差异 ⑤ 尝试将 INT8 模型也导出 ONNX | 5h |
| 6/6 五 | 12:15 后 (~7h) | **速度基准测试**。① 写一个 benchmark 脚本：测量 FP32/INT8 在 CPU 上的推理延迟（1000 次取平均）② 记录模型大小（MB）、准确率（%）、推理延迟（ms）③ 填入对比表 | 4h |
| 6/7 六 | 全天 (~10h) | **剪枝探索**。① 学习权重剪枝的基本概念（非结构化 vs 结构化）② 用 `torch.nn.utils.prune` 对 CNN 做 L1 非结构化剪枝 ③ 对比剪枝前后的稀疏度、准确率、模型大小（实际压缩要序列化稀疏格式）④ 可选：看 TFLite 转换文档 | 7h |
| 6/8 日 | 全天 (~8h) | **弹性补漏 + 项目框架搭建**。① 本周未完成任务补完 ② 建立项目目录结构：`projects/cifar10-deploy/`，包含 `train.py`, `quantize.py`, `export_onnx.py`, `benchmark.py` ③ 写 README 草稿（项目目标、文件说明） | 6h |

# 每日执行规则

1. 每天先看本计划文件，确认今日任务
2. 完成一项划掉一项
3. 遇到阻塞（环境问题、API 报错），20 分钟内解决不了就记录下来，跳过继续
4. 核心期每周日做一次 30min 周回顾，更新 mastery-tracker
5. 每周日晚在 `plan/record/daily/` 下写简短周记（本周完成什么、下周重点）
