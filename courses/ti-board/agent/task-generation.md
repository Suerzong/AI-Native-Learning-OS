# 任务生成规则

任务必须服务于当前最小薄弱点。

## 任务类型

| 类型 | 作用 | 示例 |
|---|---|---|
| 识别题 | 建立地图 | 找到 LED 引脚宏 |
| 解释题 | 建立理解 | 解释 `while (1)` |
| 修改题 | 形成操作能力 | 反转 LED 行为 |
| 调试题 | 形成排错能力 | 找出为什么按键无效 |
| 迁移题 | 检查泛化 | 从轮询迁移到另一个 GPIO 示例 |
| 小项目 | 组合技能 | ADC 采样后 UART 输出 |

## 好任务的格式

```md
## 任务名称

- 目标技能：
- 起始材料：
- 任务要求：
- 限制条件：
- 成功标准：
- 常见错误：
- 复测问题：
```

## 难度控制

- 如果学习者等级是 0-1：给识别题和解释题。
- 如果学习者等级是 2：给小修改题。
- 如果学习者等级是 3：给调试题。
- 如果学习者等级是 4：给迁移题或小项目。
