# 任务生成规则

任务必须服务于当前最小薄弱点。每个任务前必须先给出足够完成任务的知识点讲解，不能直接把学习者丢进题目。

## 任务前知识讲解

生成任务前，先生成“本次知识点”。它必须回答：

- 这个知识点在板卡上对应什么现象。
- 它背后的硬件或外设原理是什么。
- SysConfig 中哪些配置会影响它。
- DriverLib 中哪些 API 会操作它。
- 初学者最容易把哪两个概念混淆。
- 本次任务只要求掌握到什么边界。

如果学习者等级是 0-2，知识讲解必须先用直观语言解释，再给 API 和配置项。
如果学习者等级是 3-5，可以减少基础解释，但仍要说明本次任务依赖的关键知识。

## 任务类型

| 类型 | 作用 | 示例 |
|---|---|---|
| 识别题 | 建立地图 | 找到 LED 引脚宏 |
| 解释题 | 建立理解 | 解释 `while (1)` |
| 修改题 | 形成操作能力 | 反转 LED 行为 |
| API函数题 | 形成 API 调用能力 | 写出 `DL_GPIO_initDigitalOutput()` 的正确参数 |
| SysConfig实操题 | 形成外设配置能力 | 新建 .syscfg，添加 GPIO 输出并修改引脚名 |
| 代码编写题 | 形成从零写代码能力 | 不参考示例，写出 GPIO 初始化+LED 闪烁完整代码 |
| 调试题 | 形成排错能力 | 找出为什么按键无效 |
| 迁移题 | 检查泛化 | 从轮询迁移到另一个 GPIO 示例 |
| 小项目 | 组合技能 | ADC 采样后 UART 输出 |

## 好任务的格式

```md
## 任务名称

- 本次知识点：
- 目标技能：
- 起始材料：
- 任务要求：
- 限制条件：
- 成功标准：
- 常见错误：
- 复测问题：
```

## SysConfig 实操题格式

```md
## 任务名称

- 目标技能：
- SysConfig 操作步骤：
- 需要配置的外设/引脚：
- 需要修改的属性（方向、上下拉、复用功能等）：
- 成功标准（生成的 ti_msp_dl_config.h 应包含什么宏）：
- 常见错误：
```

## API 函数题格式

```md
## 任务名称

- 目标函数：
- 所属头文件：
- 参数说明（每个参数的含义和取值范围）：
- 返回值含义：
- 写出完整调用的要求：
- 常见错误（参数顺序、宏名混淆等）：
```

## 难度控制

- 如果学习者等级是 0-1：给识别题和解释题，穿插 SysConfig 识别题（指出 .syscfg 中某个配置的位置）。
- 如果学习者等级是 2：给小修改题、API 函数解释题、SysConfig 跟做修改题（跟着步骤改一个已有配置）。
- 如果学习者等级是 3：给调试题、API 函数编写题（不看示例写出函数调用）、SysConfig 独立配置题（从空白添加外设）。
- 如果学习者等级是 4：给迁移题、代码编写题（从零写出完整外设初始化+使用代码）、SysConfig 综合配置题（同时配置多个外设不冲突）。
- 如果学习者等级是 5：给小项目（多外设协同代码+完整 SysConfig 工程）。
