# 晋级规则

只有出现稳定证据时才允许进入下一阶段。

## 通用晋级条件

学习者必须同时满足：

- 能用自己的话解释核心概念。
- 能定位关键源码和配置文件。
- 能修改一个 SDK 示例并说明改动影响。
- 能完成一个近迁移任务。
- 最近测试正确率不低于 85%。
- 没有未纠正的关键误区。

## 阶段性证据

| 等级 | 晋级证据 |
|---:|---|
| 1 -> 2 | 能识别术语和文件 |
| 2 -> 3 | 能跟做并解释示例 |
| 3 -> 4 | 能独立修改行为 |
| 4 -> 5 | 能迁移、调试、复盘 |

## 不允许推进的情况

- 学习者只能复述答案，不能解释为什么。
- 学习者能改代码，但不知道对应硬件现象。
- 同一误区连续出现两次。
- 没有做迁移测试。
