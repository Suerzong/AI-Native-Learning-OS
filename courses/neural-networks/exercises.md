## PERCEPTRON-01：用 NumPy 实现感知机逻辑门

- 目标技能：理解感知机的数学表达和权重更新
- 起始材料：无（从零写）
- 任务要求：用 NumPy 实现一个感知机，训练它实现 AND 逻辑门
  - 输入：4 组二值输入 (0,0), (0,1), (1,0), (1,1)
  - 输出：对应 AND 结果 0, 0, 0, 1
  - 使用阶跃函数作为激活
- 限制条件：只能用 NumPy，不能用框架
- 成功标准：4 组输入全部预测正确；能写出最终的权重和偏置值
- 常见错误：
  - 忘记偏置项
  - 权重更新规则搞错（学习率过大导致震荡）
  - 阶跃函数的阈值设为 0 但输出不是 0/1
- 反馈重点：权重更新公式 Δw = η(y - y_hat)·x 的含义

## PERCEPTRON-02：理解 XOR 不可分

- 目标技能：理解线性可分和感知机的局限性
- 起始材料：PERCEPTRON-01 的代码
- 任务要求：
  1. 用同样的感知机代码尝试训练 XOR 门
  2. 观察结果，记录训练是否能收敛
  3. 画出 XOR 的 4 个输入点在二维平面上的位置
  4. 尝试画一条直线分开 0 和 1，说明为什么做不到
- 限制条件：不限训练轮数，但要说明为什么不收敛
- 成功标准：能解释 "XOR 不是线性可分的" 并用图示证明
- 常见错误：
  - 以为训练更多轮就能解决
  - 不理解线性可分的几何含义
- 反馈重点：画图 + 解释为什么单层感知机需要多层来解决 XOR

## PERCEPTRON-03：NumPy 实现感知机解决 OR 门

- 目标技能：修改感知机解决不同逻辑门
- 起始材料：PERCEPTRON-01 的代码
- 任务要求：修改训练数据为 OR 门，重新训练，记录收敛过程
- 限制条件：只改数据，不改算法
- 成功标准：收敛且能解释为什么 OR 比 AND 更容易收敛（分离超平面的选择更多）
- 常见错误：只改标签但忘记改对应的输入数据
- 反馈重点：不同逻辑门对应不同权重和决策边界

## ACTIVATION-01：绘制四种激活函数及其导数

- 目标技能：理解 Sigmoid、Tanh、ReLU 的函数形态和梯度行为
- 起始材料：无
- 任务要求：
  1. 用 NumPy 实现 Sigmoid、Tanh、ReLU 及其导数
  2. 用 matplotlib 画出函数曲线和导数曲线（上下排列）
  3. 在图上标注：梯度消失区域、零中心特性、死亡 ReLU 区域
- 限制条件：x 轴范围 [-5, 5]，至少 200 个采样点
- 成功标准：图正确、标注准确；能口头解释每种函数的优缺点
- 常见错误：
  - Sigmoid 导数公式写错（不是 σ(x)(1-σ(x)) 而是用了 x 的导数）
  - ReLU 导数在 x=0 处未定义不知道怎么处理
  - Tanh 和 Sigmoid 的关系不清楚
- 反馈重点：Sigmoid 导数用 σ(x) 表示比用 x 表示更简洁

## ACTIVATION-02：实现 Softmax 并验证概率归一化

- 目标技能：理解 Softmax 的计算过程和数值稳定性
- 起始材料：无
- 任务要求：
  1. 用 NumPy 实现 Softmax 函数
  2. 输入 z = [1.0, 2.0, 3.0]，输出概率
  3. 验证输出之和为 1
  4. 输入 z = [1000, 2000, 3000]，观察数值溢出
  5. 实现数值稳定版本（减去 max）并重新验证
- 限制条件：不用框架，纯 NumPy
- 成功标准：两种版本都能跑；能解释为什么要减 max
- 常见错误：
  - 忘记减 max 导致大数值溢出
  - 对每一行还是整个矩阵减 max 搞混
  - 不理解为什么要沿 axis 求和
- 反馈重点：数值稳定性不是可选项，是必须做的

## ACTIVATION-03：对比 ReLU 和 Sigmoid 在深层网络中的梯度

- 目标技能：理解梯度消失和 ReLU 的优势
- 起始材料：无
- 任务要求：
  1. 模拟一个 10 层网络的反向传播：每层梯度 = 上游梯度 × 本地梯度
  2. 用 Sigmoid 作为激活，随机初始化权重，观察第 10 层的梯度大小
  3. 用 ReLU 重复上述实验
  4. 对比并解释为什么 Sigmoid 在深层网络中梯度消失
- 限制条件：可以简化为标量计算，不需要完整矩阵运算
- 成功标准：能定量展示 Sigmoid 梯度逐层衰减；能解释 ReLU 为什么缓解这个问题
- 常见错误：
  - Sigmoid 的本地梯度在饱和区接近 0 但不知道这意味着什么
  - ReLU 的负半轴梯度为 0 导致的死亡 ReLU 问题
- 反馈重点：梯度消失 = 每层梯度 < 1 的连乘效应

## LOSS-01：实现 MSE 和交叉熵损失

- 目标技能：用 NumPy 实现常见损失函数
- 起始材料：无
- 任务要求：
  1. 实现 MSE：`mse(y_true, y_pred)`
  2. 实现 Binary Cross-Entropy：`bce(y_true, y_pred)`
  3. 测试：y_true=[1,0,1,1]，y_pred=[0.9,0.1,0.8,0.7]
  4. 手算 MSE 和 BCE 的值，然后用代码验证
- 限制条件：注意 log(0) 的数值稳定性（加 epsilon）
- 成功标准：手算和代码结果一致；能解释为什么 BCE 比 MSE 更适合分类
- 常见错误：
  - MSE 忘记取平均
  - BCE 忘记负号
  - log(0) 导致 -inf
- 反馈重点：加 epsilon = 1e-8 避免 log(0)

## LOSS-02：对比 MSE 和 BCE 在分类问题上的梯度行为

- 目标技能：理解为什么交叉熵比 MSE 更适合分类
- 起始材料：无
- 任务要求：
  1. 对于二分类：y_true=1，画出 MSE 和 BCE 在 y_pred ∈ [0,1] 上的损失曲线
  2. 计算并画出两者的梯度（对 y_pred 求导）
  3. 解释：当预测很自信但错误时（y_pred=0.01，y_true=1），哪个损失函数给的梯度更大？
- 限制条件：用 matplotlib 可视化
- 成功标准：能解释 "交叉熵在错误预测时给更大梯度" 的直觉
- 常见错误：
  - 混淆损失值和梯度
  - 不理解 "大梯度 = 学得快"
- 反馈重点：损失函数的选择直接影响学习速度

## FORWARD-01：手动计算 2 层网络的前向传播

- 目标技能：理解矩阵运算和维度跟踪
- 起始材料：无
- 任务要求：

给定一个 2 层网络：
- 输入 x = [[1], [2]]（2x1 列向量）
- W1 = [[0.1, 0.2], [0.3, 0.4]]（2x2），b1 = [[0.1], [0.1]]（2x1）
- 激活函数：ReLU
- W2 = [[0.5, 0.6]]（1x2），b2 = [[0.0]]（1x1）
- 输出激活：Sigmoid

手动计算：
1. z1 = W1·x + b1
2. a1 = ReLU(z1)
3. z2 = W2·a1 + b2
4. y_hat = Sigmoid(z2)

- 限制条件：先手算再写代码验证
- 成功标准：手算结果和代码一致；能说出每一步的矩阵维度
- 常见错误：
  - 矩阵乘法维度搞错（W1·x vs x·W1）
  - 偏置加法的 broadcasting 不理解
  - ReLU 和 Sigmoid 的计算搞混
- 反馈重点：维度是前向传播的第一公民
