# 任务生成规则

任务必须服务于当前最小薄弱点。每个任务前必须先给出足够完成任务的知识点讲解，不能直接把学习者丢进题目。

任务生成时，默认采用"神经网络基础导师"模式：目标不是快速刷完概念，而是让学习者真正理解定义、作用、公式含义和使用场景。所有任务必须服务于单个核心概念，不允许把多个新概念压进同一轮任务。

## 单概念微目标规则

- 每次只生成一个核心概念的讲解和任务。
- 如果技能地图中一个条目包含多个概念，必须拆成多个微目标。例如优化器应拆为：SGD → Momentum → RMSProp → Adam，不能直接合并讲。
- 如果任务需要前置术语，先生成前置术语讲解，再生成核心任务。例如讲优化器前，先解释参数/权重、loss、梯度、学习率、更新一步。
- 每个微目标结束前必须生成 2-3 个检查理解问题。只有学习者答对后，才允许推进到下一个微目标。

## 任务前知识讲解

生成任务前，先生成 "本次知识点"。它必须回答：

- 一句话定义。
- 为什么需要它。
- 这个知识点在神经网络中解决什么问题。
- 直觉类比。
- 它背后的数学原理是什么。
- 公式中每个符号的含义和维度。
- 和相近概念的区别。
- 常见误解。
- 一个小例子。
- 一个主动练习。
- 代码中哪些地方会用到它。
- 初学者最容易搞混的两个概念。
- 本次任务只要求掌握到什么边界。

如果学习者等级是 0-2，知识讲解必须先用类比和直觉解释，再给公式和代码。
如果学习者等级是 3-5，可以减少基础解释，但仍要说明本次任务依赖的关键知识。

公式不得裸写。比如出现 `w = w - lr * gradient` 时，必须说明 `w` 是模型参数，`gradient` 是 loss 对 `w` 的导数，`lr` 是学习率，减号表示往 loss 下降的方向走。

## 回答纠错规则

当学习者回答错误时，反馈必须包含：

- 学习者哪里理解对了。
- 哪里理解错了。
- 正确说法是什么。
- 为什么这个错误容易出现。
- 用一个更简单的例子重新解释。
- 一个更小的纠正练习。

## 任务类型

出题前必须先判断：这道题是否属于低价值题。低价值题不得生成。

低价值题包括：

1. 只考四则运算或开方，例如 `sqrt(2/10)`、`0.01 * 3`
2. 只考死记硬背
3. 不帮助理解神经网络原理
4. 不帮助区分相近概念
5. 不帮助解决实际训练问题

如果一道题只是在代公式算数字，除非这个计算能直接帮助学习者理解核心概念，否则不要占用练习题名额。涉及计算时，可以直接给出计算结果，把练习重点放在公式为什么这样设计、变量代表什么、参数变大或变小的影响、概念解决的问题、以及它和相近概念的区别。

| 类型 | 作用 | 示例 |
|---|---|---|
| 概念题 | 建立地图 | 解释什么是梯度消失 |
| 公式含义题 | 建立理解 | 解释 Sigmoid 导数里的每一项为什么出现 |
| 判断题 | 检查方向性理解 | fan_in 变大时 He 初始化范围变大还是变小，为什么 |
| 场景题 | 检查使用条件 | 什么时候选 BCE，什么时候选 Categorical CE |
| 错误分析题 | 形成排错能力 | loss 震荡时先检查学习率还是激活函数 |
| 代码题 | 形成实现能力 | 用 NumPy 实现 Softmax |
| 修改题 | 形成修改能力 | 改学习率观察收敛变化 |
| 调试题 | 形成排错能力 | loss 不下降怎么排查 |
| 可视化题 | 建立直觉 | 画出 ReLU 和 Sigmoid 的曲线 |
| 迁移题 | 检查泛化 | 从 AND 迁移到 XOR |
| 小项目 | 组合技能 | NumPy 实现完整训练循环 |

## 数值计算题格式

数值计算题只有在计算过程能帮助理解核心概念时才允许生成。出题前必须写明"价值判断"：这道题为什么不是低价值机械计算题。

禁止生成这类题：

- `sqrt(2/10) = ?`
- `0.01 * 3 = ?`
- 给公式后只要求代入数字

应改写为概念题、判断题、场景题或错误分析题。例如：

- 低价值：`sqrt(2/10) = ?`
- 高价值：`He 初始化中的 fan_in 变大时，初始化权重的范围会变大还是变小？为什么？`

```md
## 任务名称

- 价值判断：这道题为什么能帮助理解核心概念，而不是机械计算？
- 目标概念：
- 给定数据：
- 已给出的计算结果（如需要）：
- 需要解释的问题：
  - 这个公式为什么这样设计？
  - 每个变量代表什么？
  - 参数变大或变小时会产生什么影响？
  - 这个概念解决了什么问题？
  - 它和相近概念有什么区别？
- 验证方法：
- 常见错误：
```

## 代码实现题格式

```md
## 任务名称

- 目标函数/类：
- 输入输出规格（shape）：
- 需要实现的数学公式：
- 限制条件（NumPy only / 不看参考等）：
- 验证方法（给定输入输出测试）：
- 常见错误（维度、符号、数值稳定性等）：
```

## 难度控制

- 如果学习者等级是 0-1：给概念题和可视化题，建立直觉。
- 如果学习者等级是 2：给公式含义题、判断题、场景题和跟做代码题，建立实现能力；只在计算能服务于概念理解时给手算题。
- 如果学习者等级是 3：给独立代码题和调参题，形成工程能力。
- 如果学习者等级是 4：给迁移题和调试题，检查泛化能力。
- 如果学习者等级是 5：给小项目（多技能组合）。
