# NNDL-only 任务生成规则

本文件只服务 `/mastery-learn neural-networks` 的 NNDL 主课。任务生成必须遵守 `course-config.md` 的 `source_mode: textbook-only`。

---

# 源隔离

任务只能来自：

- 当前已讲的 NNDL 正文小节。
- 已解锁的 NNDL 前置小节。
- 当前章节的课本习题。
- `materials/nndl-teaching-index.md` 中明确列出的教学边界。

不得使用外部教材、旧教程、代码样例、工程项目、框架文档或部署专题。旧经验最多用于判断“不要默认已掌握”，不能作为出题材料。

---

# 三层标注规则

| 层级 | 任务生成规则 |
|---|---|
| 核心知识点 | 正式题主体。可生成概念解释、公式含义、小推导、课本习题拆解、常见误解纠正。 |
| 重要背景 | 只能轻量问“为什么重要”“和核心概念是什么关系”。不得生成复杂推导或章测大题。 |
| 了解 | 默认不出正式题。最多课堂中问一句“它属于哪类内容”，不计入掌握度。 |

如果一个“了解”内容还没在课本后文正式讲授，禁止提前升级为考点。

---

# 每道题的强制标注

每道正式题必须带两行边界标注：

```md
- 考察来源：核心知识点 / 重要背景 / 已学前置 / 课本习题
- 未学内容检查：无新增大概念 / 已给最小定义 / 推迟到后续章节
```

如果无法填写这两行，说明题目不合格，必须重写。

---

# 允许题型

| 类型 | 目标 | 示例 |
|---|---|---|
| 概念复述题 | 确认能用自己的话说清定义 | 用自己的话解释“表示学习” |
| 概念边界题 | 区分相近概念 | 机器学习和深度学习的关系是什么 |
| 公式含义题 | 理解符号和公式目的 | 经验风险中的每一项代表什么 |
| 小推导题 | 跟随课本推导关键一步 | 说明最小二乘解为什么来自导数为 0 |
| 课本习题拆解题 | 把课本习题拆成可完成步骤 | 先解释习题 2-1 要比较的两个对象 |
| 常见误解纠错题 | 发现并纠正错误理解 | “深度学习等于神经网络”哪里不严谨 |
| 前后关系题 | 串联已学小节 | 表示学习为什么引出深度学习 |

禁止生成代码实现、框架使用、工程调试、部署流程、外部数据集任务。

---

# 任务前讲解

生成任务前必须先给“本次知识点”：

1. NNDL 章节号和标题。
2. 本次知识点层级：核心知识点 / 重要背景 / 了解。
3. 一句话定义。
4. 课本为什么在这里讲它。
5. 需要的已学前置。
6. 关键公式或结构；没有公式时写“本节无核心公式”。
7. 公式中每个符号含义；没有公式则解释关键术语。
8. 和相近概念的区别。
9. 本轮只要求掌握到什么边界。

不得加入课本外扩展。若学习者问外部内容，先标记“不属于本轮 NNDL 主课”。

---

# 未学内容拦截

生成题目前逐题检查：

1. 是否依赖后续章节的大概念？
2. 是否依赖本轮没讲过的公式、模型、定理？
3. 是否必须查外部资料才能完成？
4. 是否把“了解”层内容当成正式考点？

处理规则：

- 小术语：给一句最小定义，再用轻量理解题。
- 大概念：推迟到对应章节。
- 必要前置：先补正式教学，再出题。
- 外部内容：删除题目。

---

# 难度控制

| 当前等级 | 可用题型 |
|---:|---|
| 0-1 | 概念复述、边界判断、课本段落复述 |
| 2 | 公式含义、关系解释、轻量课本习题拆解 |
| 3 | 小推导、同源变式、错因分析 |
| 4 | 完整课本习题、跨小节串联 |
| 5 | 讲解纠错、章节结构重构、前后文联系 |

难度只能由当前 NNDL 掌握证据决定，不能因为学习者过去做过代码或项目而提高。

---

# 出题后自检

每轮任务发出前必须自检：

| 检查项 | 要求 |
|---|---|
| 源可追溯 | 每题能指向 NNDL 小节或课本习题 |
| 层级匹配 | 核心重点考，背景轻量问，了解不正式考 |
| 前置已讲 | 不考尚未教学的大块内容 |
| 术语已定义 | 新术语已给最小定义 |
| 公式可达 | 所需公式已在题中或讲解中出现 |
| 无外部混入 | 不出现外部实操、部署、框架任务 |

若任一项不通过，必须重写任务。
