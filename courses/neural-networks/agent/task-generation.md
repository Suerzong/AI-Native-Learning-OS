# 任务生成规则

任务必须服务于当前最小薄弱点。每个任务前必须先给出足够完成任务的知识点讲解，不能直接把学习者丢进题目。

## 任务前知识讲解

生成任务前，先生成 "本次知识点"。它必须回答：

- 这个知识点在神经网络中解决什么问题。
- 它背后的数学原理是什么。
- 公式中每个符号的含义和维度。
- 代码中哪些地方会用到它。
- 初学者最容易搞混的两个概念。
- 本次任务只要求掌握到什么边界。

如果学习者等级是 0-2，知识讲解必须先用类比和直觉解释，再给公式和代码。
如果学习者等级是 3-5，可以减少基础解释，但仍要说明本次任务依赖的关键知识。

## 任务类型

| 类型 | 作用 | 示例 |
|---|---|---|
| 概念题 | 建立地图 | 解释什么是梯度消失 |
| 公式题 | 建立理解 | 写出 Sigmoid 的导数公式 |
| 手算题 | 验证理解 | 手算前向传播每一步 |
| 代码题 | 形成实现能力 | 用 NumPy 实现 Softmax |
| 修改题 | 形成修改能力 | 改学习率观察收敛变化 |
| 调试题 | 形成排错能力 | loss 不下降怎么排查 |
| 可视化题 | 建立直觉 | 画出 ReLU 和 Sigmoid 的曲线 |
| 迁移题 | 检查泛化 | 从 AND 迁移到 XOR |
| 小项目 | 组合技能 | NumPy 实现完整训练循环 |

## 好任务的格式

```md
## 任务名称

- 本次知识点：
- 目标技能：
- 起始材料：
- 任务要求：
- 限制条件：
- 成功标准：
- 常见错误：
- 复测问题：
```

## 数值计算题格式

```md
## 任务名称

- 目标概念：
- 给定数据：
- 计算步骤（要求学习者按步填写）：
- 验证方法：
- 常见错误：
```

## 代码实现题格式

```md
## 任务名称

- 目标函数/类：
- 输入输出规格（shape）：
- 需要实现的数学公式：
- 限制条件（NumPy only / 不看参考等）：
- 验证方法（给定输入输出测试）：
- 常见错误（维度、符号、数值稳定性等）：
```

## 难度控制

- 如果学习者等级是 0-1：给概念题和可视化题，建立直觉。
- 如果学习者等级是 2：给手算题和跟做代码题，建立实现能力。
- 如果学习者等级是 3：给独立代码题和调参题，形成工程能力。
- 如果学习者等级是 4：给迁移题和调试题，检查泛化能力。
- 如果学习者等级是 5：给小项目（多技能组合）。
