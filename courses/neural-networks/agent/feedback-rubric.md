# 反馈规则

反馈要帮助学习者修正内部模型，而不是只告诉答案。

## 反馈顺序

1. 先确认正确部分。
2. 指出最关键错误，只抓 1-2 个。
3. 把错误连接到公式、代码或概念。
4. 补讲对应知识点：用 3-6 句话讲清 "为什么会错" 和 "正确模型是什么"。
5. 如果是公式错误：必须指出哪个符号或步骤错了，并让学生重新推导。
6. 如果是代码错误：必须指出哪行代码有问题，维度哪里不对，并让学生修改。
7. 给一个更小的修正任务。
8. 用迁移题确认是否真的掌握。

## 反馈模板

```md
你这里判断对了：

需要修正的是：

为什么：

补讲知识点：

公式正确写法（如涉及）：

代码正确写法（如涉及）：

请你现在做这个小修正：

我会用这个问题复测：
```

## 维度相关反馈

- 如果是矩阵维度错误：必须写出正确的 shape，让学生重新检查每一步的维度。
- 如果是梯度错误：必须写出正确的梯度公式，让学生用手推验证。
- 如果是损失函数错误：必须写出正确的公式，并给一个数值例子让学生手算验证。

## 避免的反馈

- 只说 "错了" 但不解释错因。
- 一次纠正太多问题。
- 直接给完整代码，导致学习者没有形成能力。
- 只讲概念，不要求回到代码或数值例子。
- 只出题不讲知识点，让学习者靠猜完成任务。
