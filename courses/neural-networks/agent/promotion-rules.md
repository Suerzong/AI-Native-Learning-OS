# 晋级规则

只有出现稳定证据时才允许进入下一阶段。

## 通用晋级条件

学习者必须同时满足：

- 能用自己的话解释核心概念。
- 能写出核心公式并解释每个符号。
- 能用 NumPy/PyTorch 写出可运行的代码实现。
- 能解释矩阵运算的维度变化。
- 能修改代码（改网络结构、改超参数）并说明预期效果。
- 能完成一个近迁移任务。
- 最近测试正确率不低于 85%。
- 没有未纠正的关键误区。

## 阶段性证据

| 等级 | 晋级证据 |
|---:|---|
| 1 -> 2 | 能识别术语和概念 |
| 2 -> 3 | 能跟做教程并解释代码 |
| 3 -> 4 | 能独立实现和修改 |
| 4 -> 5 | 能迁移、调试、指导 |

## 各层特殊晋级要求

### 第一层 → 第二层

- 能用 NumPy 实现一个 2 层网络的前向传播。
- 能写出至少 3 种激活函数的公式和导数。
- 能解释为什么输出层激活要和损失函数匹配。

### 第二层 → 第三层

- 能手推一个 2 层网络的反向传播梯度。
- 能用 NumPy 实现完整训练循环并在 XOR 上收敛。
- 能解释 SGD、Momentum、Adam 的区别。

### 第三层 → 第四层

- 能用 PyTorch 搭建 CNN 并在 CIFAR-10 上达到 >80% 准确率。
- 能解释卷积层、BN、Dropout 各自的作用。
- 能识别过拟合并采取应对措施。

## 不允许推进的情况

- 学习者只能复述答案，不能解释为什么。
- 学习者能写代码但不能解释维度变化。
- 同一误区连续出现两次。
- 没有做迁移测试。
- 代码能跑但不知道每一步在做什么。
