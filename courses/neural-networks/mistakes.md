## 常见误区样例

### 矩阵维度搞混（行向量 vs 列向量）

- 所属技能：前向传播
- 错误表现：写 `x @ W` 还是 `W @ x` 不确定；shape (3,) 和 shape (3,1) 分不清
- 错误原因：没有建立 "维度必须能对齐才能乘" 的直觉
- 正确理解：`(n, m) @ (m, p) -> (n, p)`。如果 x 是 (3,) 列向量，W 是 (2,3)，则 `W @ x` 得 (2,) 输出
- 纠正任务：对每个矩阵运算先写出 shape，再决定用 @ 还是需要转置
- 是否已复测通过：否

### 反向传播梯度符号搞反

- 所属技能：反向传播
- 错误表现：`w = w + lr * grad` 而不是 `w = w - lr * grad`
- 错误原因：不理解梯度指向损失增大的方向，更新要走反方向
- 正确理解：梯度是 "损失对 w 的偏导数"，它指向损失增长最快的方向。我们要减小损失，所以 w -= lr * grad
- 纠正任务：用一个简单的 y = wx，手动算梯度并更新，画出损失曲线和参数移动方向
- 是否已复测通过：否

### Sigmoid 导数公式用 x 表示而非用 σ(x)

- 所属技能：激活函数
- 错误表现：写 Sigmoid 导数为 e^{-x}/(1+e^{-x})^2 而不是 σ(x)(1-σ(x))
- 错误原因：从原始定义推导，没有利用已计算的 Sigmoid 值
- 正确理解：反向传播时 Sigmoid 的值已经算过了，导数可以直接用 σ(x)(1-σ(x))，更简洁且数值稳定
- 纠正任务：两种公式分别实现，对比计算结果是否一致
- 是否已复测通过：否

### 推理时忘记关 Dropout

- 所属技能：正则化
- 错误表现：测试时 Dropout 还在随机丢弃神经元，导致预测不稳定
- 错误原因：不知道 Dropout 在训练和推理时行为不同
- 正确理解：训练时 Dropout 随机丢弃并缩放；推理时所有神经元都参与，输出稳定。PyTorch 用 `model.eval()` 切换
- 纠正任务：在同一个输入上分别用 `model.train()` 和 `model.eval()` 跑 10 次，观察输出方差
- 是否已复测通过：是（2026-05-12 练习中能识别"忘了开启eval"为预测不稳定的根因）

### BN 推理时统计量来源混淆（2026-05-12 复现）

- 所属技能：Batch Normalization
- 错误表现：问"推理时 BN 用谁的均值和方差"，回答"在嵌入式端后来的均值和方差"——即认为推理时用当前输入数据自己的统计量
- 错误原因：不理解 BN 训练时维护 running_mean / running_var 的机制，把"推理时 BN 仍工作"误解为"推理时 BN 用当前数据重新算统计量"
- 正确理解：训练时 BN 用当前 batch 统计量 + 指数移动平均累积到 running_mean/var；推理时 BN 用 running_mean/var（训练集历史统计），与当前输入无关
- 纠正任务：手写 BN 训练/推理两套逻辑，对比 train/eval 下同一输入的不同输出
- 是否已复测通过：否（概念已当场纠正，但无独立验证）

### 输出层激活和损失函数不匹配

- 所属技能：损失函数 / 网络设计
- 错误表现：回归问题用 Softmax 输出、分类问题用线性输出 + MSE
- 错误原因：不理解输出层激活、损失函数、任务类型三者必须匹配
- 正确理解：回归 = 线性输出 + MSE；二分类 = Sigmoid + BCE；多分类 = Softmax + CE
- 纠正任务：给出 5 种任务场景，写出正确的输出层激活和损失函数组合
- 是否已复测通过：否

### 维度计算时数值迁移失败（2026-05-12）

- 所属技能：RNN 前向传播
- 错误表现：题目给定 d_h=8，但计算时反复使用 128（之前例题的数值）
- 错误原因：没有切换到新题目的数值，停留在之前例题的记忆中
- 正确理解：每道题必须仔细读题，根据题目给定的数值计算
- 纠正任务：新题目必须先写出"已知：d_x=?, d_h=?"再计算
- 是否已复测通过：是（d_x=10, d_h=16 的复测题通过）

### LSTM 遗忘门与完整更新混淆（2026-05-12）

- 所属技能：LSTM 门控机制
- 错误表现：题目问"遗忘后"的 Cell State，回答了完整更新公式 f_t*C_{t-1} + i_t*C̃_t
- 错误原因：混淆了"遗忘操作"和"完整更新公式"
- 正确理解：遗忘门只负责 f_t * C_{t-1}，完整更新还需要加上输入门部分
- 纠正任务：区分"遗忘后"、"输入后"、"完整更新"三种问法
- 是否已复测通过：是（[0.5,0.8,0.2]*[100,50,80] 复测通过）
