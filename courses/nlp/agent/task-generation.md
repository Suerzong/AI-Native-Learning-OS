# 任务生成规则

任务必须服务于当前最小薄弱点。每个任务前必须先给出足够完成任务的知识点讲解，不能直接把学习者丢进题目。

## 任务前知识讲解

生成任务前，先生成"本次知识点"。它必须回答：

- 这个知识点在 NLP 流程中对应什么步骤或工具。
- 它背后的原理是什么。
- 输入数据在这一步会怎么变化（文本→分词→向量化→...）。
- 初学者最容易犯什么错误。
- 本次任务只要求掌握到什么边界。

如果学习者等级是 0-2，知识讲解必须先用直观语言解释，再给 API 调用和代码。
如果学习者等级是 3-5，可以减少基础解释，但仍要说明本次任务依赖的关键知识和数据变化。

## 任务类型

| 类型 | 作用 | 示例 |
|---|---|---|
| 识别题 | 建立概念地图 | NLP 的核心任务有哪些？ |
| 解释题 | 建立理解 | 解释 TF-IDF 的 TF 和 IDF 分别衡量什么 |
| 流程题 | 建立 pipeline 意识 | 写出文本分类的完整流程 |
| 修改题 | 形成操作能力 | 修改停用词表，观察分类效果变化 |
| API 编写题 | 形成 API 调用能力 | 写出 jieba 分词的正确代码 |
| 代码编写题 | 形成完整代码能力 | 从零写出文本分类 pipeline |
| 调试题 | 形成排错能力 | 找出 Tokenizer 不匹配的错误 |
| 对比题 | 建立选型能力 | 对比 BoW 和 TF-IDF 的效果差异 |
| 迁移题 | 检查泛化 | 把英文分类模型迁移到中文 |
| 小项目 | 组合技能 | 完整的情感分析项目 |

## 好任务的格式

```md
## 任务名称

- 本次知识点：
- 目标技能：
- 起始材料：
- 任务要求：
- 限制条件：
- 成功标准：
- 常见错误：
- 复测问题：
```

## 数据变化追踪题格式

NLP 中每一步都会改变数据的形态，追踪这种变化是理解 pipeline 的关键。

```md
## 任务名称

- 目标技能：追踪文本在 X 步骤后的形态变化
- 输入数据：一段原始中文文本
- 步骤：具体处理步骤描述
- 要求：不运行代码，写出每一步的数据形态（文本→词列表→向量→...）
- 必须写出的中间结果：
- 常见错误：
```

## API 编写题格式

```md
- 参数说明：
- 返回值：
- 写出调用的要求：

```

## 难度控制

- 如果学习者等级是 0-1：给识别题和解释题，不要求写代码。
- 如果学习者等级是 2：给流程题和 API 编写题，可以参考代码完成。
- 如果学习者等级是 3：给修改题和对比题，要求不看文档写出正确代码。
- 如果学习者等级是 4：给迁移题和代码编写题，要求从零写完整代码。
- 如果学习者等级是 5：给小项目（多模块协同）。

## NLP 特有的任务设计原则

1. **每个任务必须追踪数据形态变化**：文本→分词→清洗→向量化→模型输入，每步打印/标注。
2. **中文场景优先**：示例和练习使用中文数据，帮助理解中文 NLP 特殊性（分词、编码）。
3. **先手工再工具**：先让学生手动算 TF-IDF，再用 sklearn；先手动标 BIO，再用模型。
4. **对比驱动**：每个新方法都和已有方法对比（BoW vs TF-IDF vs Word2Vec）。
5. **端侧意识**：从第二层开始就问"这个模型在嵌入式端能跑吗"。
6. **错误驱动学习**：故意给有 bug 的文本处理代码（如编码错误、Tokenizer 不匹配），让学生调试。
7. **Hugging Face 生态优先**：优先使用 Hugging Face 工具链，这是工业界标准。
