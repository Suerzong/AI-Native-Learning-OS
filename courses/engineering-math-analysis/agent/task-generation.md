# 任务生成规则

任务必须服务于当前数学微目标。不得生成跨学科任务，不得引入编程、AI、工程、电路或信号处理内容。

## 资料优先级

1. `materials/toc.md`：确定当前教学顺序。
2. `materials/tutor-book/`：当前微目标的例题和训练题。
3. `materials/textbook/`：当前微目标的定义、定理、证明和概念解释。
4. `mistakes.md`：历史错因复测。
5. `exercises.md` 和 `exams.md`：已沉淀题目与阶段训练。

## 任务前讲解

生成任务前，必须先说明：

- 当前微目标是什么。
- 需要哪些前置定义。
- 本题最可能使用哪个定义或定理。
- 使用该定理需要满足哪些条件。
- 本题只要求掌握到什么边界。

## 任务类型

| 类型 | 作用 |
|---|---|
| 定义解释题 | 检查是否真正理解概念 |
| 条件判断题 | 防止乱用定理 |
| 基础例题 | 建立标准解题流程 |
| 关键步骤题 | 训练推导链条 |
| 改错题 | 识别常见误区 |
| 变式题 | 检查迁移能力 |
| 章末综合题 | 组合本章方法 |

## 好任务格式

```md
## 任务名称

- 当前章节：
- 当前微目标：
- 目标能力：
- 前置定义：
- 可用定理：
- 题目：
- 作答要求：
- 成功标准：
- 常见错误：
- 复测变式：
```

## 键盘作答要求

允许学生用普通文本写公式，遵守 `materials/typing-conventions.md`。

优先要求学生写：

1. 方法
2. 定理条件
3. 关键步骤
4. 结论

如果学生只写最终答案，不足以判定掌握，必须追问思路和条件。
