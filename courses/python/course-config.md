## 高频错误模式

### 1. 可变对象陷阱
```python
# 错误：列表作为默认参数
def add_item(item, lst=[]):
    lst.append(item)
    return lst

# 正确：
def add_item(item, lst=None):
    if lst is None:
        lst = []
    lst.append(item)
    return lst
```
**原因：** 默认参数在函数定义时求值一次，所有调用共享同一个列表。

### 2. 浅拷贝 vs 深拷贝
```python
import copy
a = [[1, 2], [3, 4]]
b = a[:]          # 浅拷贝！b[0] 和 a[0] 是同一个对象
c = copy.deepcopy(a)  # 深拷贝
```
**原因：** 切片和 .copy() 只拷贝最外层，嵌套的可变对象仍然是引用。

### 3. 循环中修改列表
```python
# 错误：在 for 循环中删除元素
lst = [1, 2, 3, 4, 5]
for item in lst:
    if item % 2 == 0:
        lst.remove(item)  # 跳过元素！

# 正确：
lst = [item for item in lst if item % 2 != 0]
# 或：lst[:] = [item for item in lst if item % 2 != 0]
```

### 4. 作用域混淆
```python
x = 10
def foo():
    print(x)    # UnboundLocalError!
    x = 20

# 原因：赋值操作使 x 成为局部变量，但 print 在赋值之前
```

### 5. is vs ==
```python
a = 256
b = 256
print(a is b)   # True（小整数缓存）

a = 257
b = 257
print(a is b)   # False（可能，取决于实现）

# 用 == 比较值，用 is 比较 None/True/False
```

### 6. 缩进错误
Python 使用缩进定义代码块，混用 Tab 和空格会导致 IndentationError。
**规则：** 始终使用 4 个空格，不要混用 Tab。

### 7. 字符串不可变性
```python
s = "hello"
s[0] = "H"  # TypeError!

# 正确：
s = "H" + s[1:]
```

### 8. 异常捕获过于宽泛
```python
# 错误：捕获所有异常
try:
    result = risky_operation()
except:
    pass  # 吞掉所有错误，包括 KeyboardInterrupt

# 正确：只捕获预期的异常
try:
    result = risky_operation()
except ValueError as e:
    print(f"值错误: {e}")
```

## 教学风格

### 新符号必讲规则（强制）

**任何在讲解中首次出现的 Python 符号（运算符、语法符号、特殊标记），必须立即给出说明。** 不允许跳过未解释的符号。

讲解格式：

```
符号      ← 出现位置的代码片段
```

- **名称**：符号的正式名称（中英文）
- **读法**：口语中怎么读（如 `*args` 读作 "star args"）
- **含义**：这个符号在当前上下文中表示什么
- **语法**：正确的使用语法
- **示例**：1-2 个典型用法
- **易错点**：（如有）初学者容易混淆的地方

**判断标准**：只要学生在之前课程中没有系统学过这个符号，就算"新符号"。

**需要讲解的符号类别**：
- 运算符：`**`, `//`, `%`, `:=`, `&`, `|`, `^`, `~`, `<<`, `>>`, `is`, `in`, `not in`, `is not`
- 赋值运算符：`+=`, `-=`, `*=`, `/=`, `**=`, `//=`  等（首次出现时讲解，后续同类不再重复）
- 解包/收集：`*`, `**`（用于参数收集或解包时）
- 语法符号：`@`（装饰器）、`->`（返回类型注解）、`:`（切片/类型注解）、`...`（Ellipsis）
- 推导式/生成器：`[x for x in ...]` 中的 `for...in...if...` 语法
- 上下文管理器：`with...as...` 中的 `as`
- 格式化：`f""`、`{:.2f}`、`{!r}`、`{!s}` 等 f-string 标记
- 类型注解：`list[int]`, `dict[str, int]`, `Optional[T]`, `Union[T1, T2]`, `|`（3.10+联合类型）
- 其他特殊标记：`__name__`, `__init__` 等双下划线（dunder）标记的含义

**不需重复讲解的符号**：
- Layer 1 已学过的基础运算符（`+`, `-`, `*`, `/`, `==`, `!=`, `<`, `>`, `<=`, `>=`, `and`, `or`, `not`）
- 标点类符号（括号、逗号、冒号用于控制流等基础用法）

### 新函数必讲规则（强制）

**任何在讲解中首次出现的 Python 函数，必须立即给出完整说明。** 不允许跳过未解释的函数。

讲解格式：

```
函数名(参数)        ← 调用示例
```

- **来源**：内置 / 标准库名.函数名 / 第三方库
- **签名**：`函数名(必选参数, 可选参数=默认值) -> 返回类型`
- **作用**：一句话说明这个函数做什么
- **参数**：逐个说明每个参数的含义（必选 vs 可选）
- **返回值**：返回什么类型、什么含义
- **常见用法**：1-2 个最典型的代码示例
- **易错点**：（如有）初学者最容易犯的错误

**判断标准**：只要学生在之前课程中没有系统学过这个函数，就算"新函数"。
- 内置函数（print, len, range, type 等）在 Layer 1 后不再重复讲解，除非用法有变体
- 标准库函数（open, json.load, os.path.join 等）在每课首次出现时必须讲解
- 第三方库函数（np.array, pd.read_csv, plt.plot 等）每次出现都要讲解

### Code-First 原则
1. **先给能跑的代码** —— 学生先运行，看到效果
2. **再解释为什么** —— 逐行分析代码逻辑（含新函数讲解）
3. **然后让学生改** —— 修改参数/逻辑，观察变化
4. **最后独立写** —— 在新情境中完成类似任务

### 掌握学习循环
```
诊断 → 讲解 → 练习 → 反馈 → 迁移测试 → 更新状态
  ↑                                        │
  └────────── 未达标则补救 ←───────────────┘
```

### 反馈原则
- **确认正确**：增强信心（"这段代码写得很好，特别是... "）
- **精确指出错误**：精确到行号和具体问题
- **解释原因**：不是"错了"，而是"为什么错"以及"正确的理解是什么"
- **给出修正任务**：让学生自己修改，而非直接给答案
- **验证理解**：修改后追问一个迁移问题

## 诊断维度

每次诊断从以下 5 个维度评估学生的 Python 能力，综合 Level = round(概念×0.15 + 代码×0.30 + 函数×0.20 + 调试×0.15 + 解决×0.20)。

| 维度 | 权重 | 评估目标 |
|------|------|----------|
| 概念理解 | 15% | 是否理解 Python 语言核心概念（变量、类型、数据结构等） |
| 代码书写 | 30% | 能否正确写出 Python 代码（语法、结构、可读性） |
| 函数使用 | 20% | 能否正确使用内置函数和标准库，能否自定义函数 |
| 调试能力 | 15% | 能否发现和修复代码中的错误 |
| 问题解决 | 20% | 能否用 Python 解决实际问题，能否迁移应用 |

**Level 标准（五维度通用）：**

| Level | 概念理解 | 代码书写 | 函数使用 | 调试能力 | 问题解决 |
|-------|----------|----------|----------|----------|----------|
| 0 | 不知道 | 不能写 | 不知道 | 不能识别 | 无思路 |
| 1 | 听说过 | 能抄写 | 能查找 | 能识别 | 有思路 |
| 2 | 能解释 | 能模仿 | 能使用 | 能定位 | 能实现 |
| 3 | 能区分 | 能独立写 | 能组合 | 能修复 | 能优化 |
| 4 | 能教别人 | 能优化 | 能自定义 | 能预防 | 能迁移 |

**诊断时机：**
- 前置诊断：首次学习前，概念快问 + 代码阅读 + 简单编程
- 每课诊断：学习新技能前，评估前置技能掌握程度
- 阶段诊断：每 10 课时后，全面评估并更新 mastery-tracker.md
