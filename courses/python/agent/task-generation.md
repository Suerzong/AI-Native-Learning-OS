# task-generation.md — Python 任务生成规则

## 任务类型定义

Python 课程使用以下 7 种任务类型，按难度递进排列：

### 类型 1: 识别题（Recognition）

**目标**：测试学生能否识别和理解代码

**形式**：
- 给出一段代码，要求说出输出
- 给出多个代码片段，要求选出正确的
- 给出代码，要求说出其功能

**难度范围**：Level 0-2

**示例**：
```
以下代码的输出是什么？
a = [1, 2, 3]
b = a[:]
b.append(4)
print(a)
```

**生成规则**：
- 代码长度：3-8 行
- 涉及的概念：当前学习的技能
- 不要求学生写代码
- 可以包含 1-2 个干扰项

---

### 类型 2: 解释题（Explanation）

**目标**：测试学生能否解释代码的工作原理

**形式**：
- 给出代码，要求逐行解释
- 给出代码的输出，要求解释为什么
- 要求解释某个概念的含义

**难度范围**：Level 1-3

**示例**：
```
解释以下代码为什么能实现列表去重：
lst = [1, 2, 2, 3, 3, 3]
unique = list(set(lst))
```

**生成规则**：
- 代码长度：3-10 行
- 要求用中文解释
- 关注"为什么"而非"是什么"
- 可以追问细节

---

### 类型 3: 修改题（Modification）

**目标**：测试学生能否修改现有代码实现新功能

**形式**：
- 给出一段代码，要求修改以实现新功能
- 要求在现有代码基础上添加功能
- 要求用不同的方法实现相同功能

**难度范围**：Level 2-4

**示例**：
```
以下代码统计列表中正数的个数。请修改为统计偶数的平均值。
numbers = [1, -2, 3, -4, 5, -6]
count = sum(1 for n in numbers if n > 0)
print(f"正数个数: {count}")
```

**生成规则**：
- 给出可运行的原始代码
- 修改幅度：2-5 行
- 新功能与原功能有相似之处
- 保留原始代码的风格

---

### 类型 4: 编写题（Code Writing）

**目标**：测试学生能否独立编写代码

**形式**：
- 给定需求描述，要求从零编写
- 给定函数签名，要求实现函数体
- 给定输入输出示例，要求编写代码

**难度范围**：Level 2-5

**示例**：
```
编写一个函数 merge_dicts(*dicts)，
接收任意数量的字典，合并成一个字典。
如果有重复的键，后面的字典覆盖前面的。
示例：
merge_dicts({"a": 1}, {"b": 2}, {"a": 3})
→ {"a": 3, "b": 2}
```

**生成规则**：
- 需求描述清晰、无歧义
- 提供输入输出示例
- 不要求特定的实现方式
- 评估标准：正确性 > 效率 > 可读性

---

### 类型 5: 调试题（Debug）

**目标**：测试学生能否发现和修复代码错误

**形式**：
- 给出有错误的代码，要求找出并修复
- 给出错误信息，要求推断原因
- 要求解释为什么某段代码不能工作

**难度范围**：Level 2-5

**示例**：
```
以下函数旨在返回字典中值最大的键，但有 bug。
找出并修复它。

def max_key(d):
    return max(d, key=d.values())

测试：max_key({"a": 3, "b": 1, "c": 2}) 应返回 "a"
```

**生成规则**：
- 每题包含 1-3 个错误
- 错误类型覆盖：语法错误、逻辑错误、类型错误
- 提供足够的信息让学生定位错误
- 不要给出错误信息（让学生产生和阅读错误信息）

---

### 类型 6: 迁移题（Transfer）

**目标**：测试学生能否将所学知识应用到新场景

**形式**：
- 给出一个新情境，要求用已学技能解决
- 结合多个技能解决问题
- 在不熟悉的领域应用熟悉的模式

**难度范围**：Level 3-5

**示例**：
```
你之前用字典统计过词频。现在有一个新场景：
有一个学生成绩列表，格式为 [("Alice", 85), ("Bob", 92), ("Alice", 78), ...]，
同一个学生可能有多次成绩。
请统计每个学生的最高分。
```

**生成规则**：
- 场景描述贴近实际（数据处理、自动化等）
- 可以使用之前学过的技能
- 不要求使用未学过的库
- 鼓励学生创造性地解决问题

---

### 类型 7: 小项目题（Mini-Project）

**目标**：测试学生能否综合运用多个技能完成实际任务

**形式**：
- 给出一个小型项目需求
- 要求设计和实现
- 包含多个步骤和组件

**难度范围**：Level 4-5

**示例**：
```
设计一个简单的学生成绩管理系统：
1. 使用类 Student 管理学生信息（姓名、成绩列表）
2. 能添加成绩、计算平均分、获取最高分
3. 使用类 GradeBook 管理多个学生
4. 能添加学生、按平均分排序、导出为 JSON
```

**生成规则**：
- 项目规模：50-150 行代码
- 涵盖 3-5 个技能
- 有明确的验收标准
- 允许使用标准库

---

## 任务难度分配

### Level 0-1 阶段
- 识别题 40%
- 解释题 30%
- 修改题 20%
- 编写题 10%

### Level 2 阶段
- 识别题 15%
- 解释题 20%
- 修改题 25%
- 编写题 25%
- 调试题 15%

### Level 3 阶段
- 修改题 20%
- 编写题 30%
- 调试题 25%
- 迁移题 25%

### Level 4-5 阶段
- 编写题 25%
- 调试题 20%
- 迁移题 30%
- 小项目题 25%

---

## 任务生成流程

1. **确定目标技能**：根据 mastery-tracker.md 确定当前要练习的技能
2. **确定目标 Level**：根据当前 Level 选择目标 Level（当前 Level + 1）
3. **选择任务类型**：根据目标 Level 和难度分配选择任务类型
4. **生成任务内容**：
   - 使用模板生成任务
   - 确保任务与当前技能相关
   - 避免使用未学过的概念
5. **设定成功标准**：明确、可衡量的完成条件
6. **预测常见错误**：根据 mistakes.md 预测学生可能犯的错误
