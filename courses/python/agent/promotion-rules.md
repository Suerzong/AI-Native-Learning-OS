# promotion-rubric.md — Python 技能晋升规则

## Level 晋升标准

### Level 0 → Level 1（未学习 → 已了解）

**证据要求**：
- 能说出该技能的名称和基本用途
- 能识别相关代码（知道这段代码在做什么）
- 能在文档中找到相关信息

**测试方法**：
- 概念问答 3 题，答对 2 题即通过
- 代码识别 2 题，答对 1 题即通过

**示例（字符串操作）**：
```
Q: "split() 方法是做什么的？"
A: "把字符串按指定的分隔符拆分成列表。"  → 通过
A: "拆字符串？"  → 勉强通过
A: "不知道。"  → 不通过
```

### Level 1 → Level 2（已了解 → 已学习）

**证据要求**：
- 能解释该技能的核心概念
- 能读懂使用该技能的代码
- 能参照示例写出类似代码（允许有小错误）
- 知道常见的使用场景

**测试方法**：
- 概念解释 2 题，完整且基本正确
- 代码阅读 2 题，能正确说出输出
- 代码补全 2 题，能补全并运行通过

**示例（列表操作）**：
```
Q: "解释一下切片 lst[1:4:2] 的含义"
A: "从索引 1 开始到索引 4（不包含），步长为 2，
    所以取索引 1 和 3 的元素。"  → 通过

Q: 补全代码：将列表 lst 中大于 10 的元素筛选出来
A: result = [x for x in lst if x > 10]  → 通过
A: result = []
   for x in lst:
       if x > 10:
           result.append(x)  → 通过（能工作但不是最优）
```

### Level 2 → Level 3（已学习 → 已实践）

**证据要求**：
- 能独立写出正确的代码（不需要参照示例）
- 能处理常见边界情况
- 能调试自己代码中的错误
- 能使用该技能完成练习题

**测试方法**：
- 独立编写 2 题，不看示例完成
- 调试 1 题，能发现并修复错误
- 完成 exercises.md 中对应难度的练习

**示例（字典操作）**：
```
Q: "编写一个函数，接收一个单词列表，返回每个单词长度的字典"
A:
def word_lengths(words):
    return {w: len(w) for w in words}
→ 通过（正确使用字典推导式）

Q: "以下代码有什么问题？"
   d = {}
   for item in data:
       d[item] = d[item] + 1  # 第一次出现时会报错
A: "第一次遇到新的 item 时，d[item] 不存在，
    应该用 d.get(item, 0) + 1 或 defaultdict。"
→ 通过
```

### Level 3 → Level 4（已实践 → 初步掌握）

**证据要求**：
- 能在较少参考资料的情况下完成任务
- 能选择合适的工具/方法（而非只会一种方式）
- 能解释代码的工作原理
- 能处理不太常见的边界情况
- 代码风格基本规范

**测试方法**：
- 开放题 1 题：给定需求，独立设计方案并实现
- 迁移题 1 题：在新情境中应用该技能
- 代码审查 1 题：审查并改进给定代码

**示例（装饰器）**：
```
Q: "编写一个装饰器，记录函数的调用次数"
A:
def count_calls(func):
    count = 0
    def wrapper(*args, **kwargs):
        nonlocal count
        count += 1
        print(f"{func.__name__} called {count} times")
        return func(*args, **kwargs)
    return wrapper
→ 通过（理解闭包和 *args/**kwargs）

Q: "你写的装饰器有什么问题？被装饰函数的 __name__ 和 __doc__ 还正确吗？"
A: "不对，wrapper 会覆盖原函数的元信息，
    应该加 @functools.wraps(func)。"
→ 通过（理解装饰器的副作用）
```

### Level 4 → Level 5（初步掌握 → 已掌握）

**证据要求**：
- 能不看任何资料完成复杂任务
- 能设计优雅、高效的解决方案
- 能向他人清楚地讲解该技能
- 能预见和避免常见陷阱
- 能将该技能与其他技能组合使用
- 能在完全新的场景中创造性地应用

**测试方法**：
- 设计题 1 题：设计一个使用该技能的小模块
- 教学题 1 题：向"初学者"解释该技能
- 综合题 1 题：将该技能与至少 2 个其他技能组合

**示例（函数与作用域）**：
```
Q: "设计一个缓存装饰器（memoization），能缓存函数的返回值"
A:
import functools

def memoize(func):
    cache = {}
    @functools.wraps(func)
    def wrapper(*args):
        if args not in cache:
            cache[args] = func(*args)
        return cache[args]
    return wrapper

@memoize
def fibonacci(n):
    if n < 2:
        return n
    return fibonacci(n-1) + fibonacci(n-2)
→ 通过（理解闭包、装饰器、字典、递归的组合应用）

Q: "这个装饰器有什么局限性？如何改进？"
A: "参数必须是可哈希的（不能用列表作为参数），
    没有缓存大小限制（可能内存溢出），
    没有考虑线程安全。
    可以用 functools.lru_cache 替代。"
→ 通过（能分析局限性和提出改进）
```

## 晋升规则

### 自动晋升条件
- 每日测试中某技能得分 ≥ 80%，自动晋升 1 级
- 阶段考试中某技能相关题目全部正确，自动晋升 1 级

### 强制保留条件
- 即使测试通过，如果该技能的前置技能 Level 不够，不能晋升
  - 例：装饰器 Level 3 需要 闭包 Level ≥ 2
- 如果最近 7 天内未使用该技能，Level 最多保持不变（不会因不使用而降级）

### 降级条件
- 连续 3 次涉及该技能的测试得分 < 50%，降 1 级
- 阶段考试中该技能相关题目全部错误，降 1 级
- Level 最低为 0，不会降到负数

### 跨级晋升
- 在特殊情况下（如学生已有相关编程经验），可以跳级
- 需要在诊断中连续通过 2 个 Level 的测试
- 例如：从 Level 0 直接到 Level 2，需要同时通过 Level 1 和 Level 2 的测试
