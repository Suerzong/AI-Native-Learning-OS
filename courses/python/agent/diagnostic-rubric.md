# diagnostic-rubric.md — Python 诊断评估标准

## 五维评估体系

每次诊断从以下 5 个维度评估学生的 Python 能力。

### 维度 1: 概念理解（Concept）

**评估目标**：学生是否理解 Python 语言的核心概念

| Level | 描述 | 评估标准 |
|-------|------|----------|
| 0 | 不知道 | 无法说出概念名称 |
| 1 | 听说过 | 能说出概念名称，但无法解释 |
| 2 | 能解释 | 能用自己的话解释概念，但可能有小错误 |
| 3 | 能区分 | 能区分相似概念（如 list vs tuple, is vs ==） |
| 4 | 能教别人 | 能清晰地向他人解释概念及其应用场景 |

**评估方法**：
- 直接提问："解释一下什么是列表推导式？"
- 对比提问："list 和 tuple 有什么区别？"
- 场景提问："什么情况下应该用集合而不是列表？"

### 维度 2: 代码书写（Code Writing）

**评估目标**：学生能否正确地写出 Python 代码

| Level | 描述 | 评估标准 |
|-------|------|----------|
| 0 | 不能写 | 无法写出任何代码 |
| 1 | 能抄写 | 能照着示例写，但不理解 |
| 2 | 能模仿 | 能参照示例写出类似代码，可能有语法错误 |
| 3 | 能独立写 | 能不看示例写出正确的代码 |
| 4 | 能优化 | 能写出高效、简洁、可读的代码 |

**评估方法**：
- 填空题：给出部分代码，让学生补全
- 从零编写：给定需求，让学生独立编写
- 代码审查：评估学生代码的可读性和效率

### 维度 3: 函数使用（Function Usage）

**评估目标**：学生能否正确使用 Python 内置函数和标准库

| Level | 描述 | 评估标准 |
|-------|------|----------|
| 0 | 不知道 | 不知道有哪些常用函数 |
| 1 | 能查找 | 知道在文档中查找函数用法 |
| 2 | 能使用 | 能正确使用常见函数（len, range, enumerate 等） |
| 3 | 能组合 | 能将多个函数组合使用解决问题 |
| 4 | 能自定义 | 能定义自己的函数并正确使用参数/返回值 |

**评估方法**：
- 快速提问："如何将字符串按空格分割？"
- 组合任务："不使用内置函数 sort，自己实现排序"
- API 考察："enumerate() 返回什么？如何使用？"

### 维度 4: 调试能力（Debug）

**评估目标**：学生能否发现和修复代码中的错误

| Level | 描述 | 评估标准 |
|-------|------|----------|
| 0 | 不能识别 | 无法识别错误信息 |
| 1 | 能识别 | 能看懂错误信息（如 SyntaxError, TypeError） |
| 2 | 能定位 | 能根据错误信息找到出错的代码行 |
| 3 | 能修复 | 能独立修复常见错误 |
| 4 | 能预防 | 能在编写代码时预见并避免常见错误 |

**评估方法**：
- 给出有 bug 的代码，让学生找出并修复
- 给出错误信息，让学生推断错误原因
- 让学生解释自己代码中的错误

### 维度 5: 问题解决（Problem Solving）

**评估目标**：学生能否用 Python 解决实际问题

| Level | 描述 | 评估标准 |
|-------|------|----------|
| 0 | 无思路 | 面对问题完全不知道如何下手 |
| 1 | 有思路 | 能描述解决问题的步骤，但不会实现 |
| 2 | 能实现 | 能实现简单的解决方案 |
| 3 | 能优化 | 能分析方案的效率并改进 |
| 4 | 能迁移 | 能将解决方案应用到类似的新问题 |

**评估方法**：
- 开放题："给定一组数据，如何找出重复项？"
- 迁移题："你之前用字典解决了词频统计，现在用它统计学生成绩分布"
- 优化题："这段代码能工作，但很慢，如何改进？"

## 诊断评估流程

### 前置诊断（首次学习前）

1. **概念快问**（5 题，每题 1 分钟）
   - 什么是变量？
   - Python 有哪些基本数据类型？
   - 列表和元组有什么区别？
   - 什么是函数？
   - 什么是循环？

2. **代码阅读**（3 题，每题 2 分钟）
   - 给出简单代码，要求说出输出

3. **代码补全**（2 题，每题 3 分钟）
   - 给出部分代码，要求补全

4. **独立编写**（1 题，5 分钟）
   - 简单编程任务

### 每课诊断（学习新技能前）

针对当前要学习的技能，快速评估前置技能的掌握程度。

例如，学习"推导式"之前：
- 确认"列表操作" Level ≥ 2
- 确认"循环" Level ≥ 2
- 确认"条件语句" Level ≥ 2

### 阶段诊断（每 10 课时后）

全面评估前一阶段所有技能的掌握程度，更新 mastery-tracker.md。

## 综合评分规则

将 5 个维度的 Level 加权计算综合 Level：

| 维度 | 权重 |
|------|------|
| 概念理解 | 15% |
| 代码书写 | 30% |
| 函数使用 | 20% |
| 调试能力 | 15% |
| 问题解决 | 20% |

综合 Level = round(概念×0.15 + 代码×0.30 + 函数×0.20 + 调试×0.15 + 解决×0.20)
