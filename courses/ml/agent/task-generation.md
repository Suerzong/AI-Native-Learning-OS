# task-generation.md — 机器学习任务生成规则

## 任务类型定义

机器学习课程使用以下 7 种任务类型，按难度递进排列：

### 类型 1: 识别题（Recognition）

**目标**：测试学生能否识别算法类型、数据问题和模型行为

**形式**：
- 给出一段代码或输出，要求说出使用了什么算法
- 给出数据集，要求判断适合的算法类型（分类/回归/聚类）
- 给出模型评估结果，要求判断是过拟合/欠拟合/正常

**难度范围**：Level 0-2

**示例**：
```
以下 sklearn 代码使用了什么算法？它是分类还是回归？
from sklearn.ensemble import RandomForestClassifier
model = RandomForestClassifier(n_estimators=100)
model.fit(X_train, y_train)
```

**生成规则**：
- 代码长度：3-10 行
- 涉及的概念：当前学习的技能
- 不要求学生写代码
- 可以包含干扰项（如相似的算法代码）

---

### 类型 2: 解释题（Explanation）

**目标**：测试学生能否解释算法原理、代码行为和评估结果

**形式**：
- 给出代码，要求解释每一步在做什么
- 给出模型输出，要求解释为什么得到这个结果
- 要求解释某个 ML 概念（如"什么是偏差-方差权衡"）

**难度范围**：Level 1-3

**示例**：
```
解释以下代码为什么会导致数据泄漏：
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)
X_train, X_test, y_train, y_test = train_test_split(X_scaled, y, test_size=0.2)
```

**生成规则**：
- 代码长度：5-15 行
- 要求用中文解释
- 关注"为什么"而非"是什么"
- 可以追问细节（如"如果用 MinMaxScaler 会有同样的问题吗？"）

---

### 类型 3: 修改题（Modification）

**目标**：测试学生能否修改现有代码实现新的建模需求

**形式**：
- 给出一个模型代码，要求更换算法
- 要求修改超参数观察效果
- 要求在现有预处理基础上添加新步骤

**难度范围**：Level 2-4

**示例**：
```
以下代码使用决策树对数据进行分类。请修改为使用随机森林，
并打印特征重要性排序。
from sklearn.tree import DecisionTreeClassifier
model = DecisionTreeClassifier(max_depth=5)
model.fit(X_train, y_train)
print(f"准确率: {model.score(X_test, y_test):.3f}")
```

**生成规则**：
- 给出可运行的原始代码
- 修改幅度：2-8 行
- 新功能与原功能有相似之处
- 保留原始代码的风格

---

### 类型 4: 编写题（Code Writing）

**目标**：测试学生能否独立编写 ML 代码

**形式**：
- 给定数据集和目标，要求从零编写建模代码
- 给定函数签名，要求实现完整的 ML 流程
- 给定输入输出示例，要求编写代码

**难度范围**：Level 2-5

**示例**：
```
编写代码完成以下任务：
1. 使用 make_classification 生成二分类数据（200 样本，10 特征）
2. 划分训练/测试集（80/20）
3. 训练逻辑回归模型
4. 输出混淆矩阵和分类报告
```

**生成规则**：
- 需求描述清晰、无歧义
- 提供数据集的生成方式或来源
- 不要求特定的实现方式
- 评估标准：正确性 > 可读性 > 效率

---

### 类型 5: 调试题（Debug）

**目标**：测试学生能否发现和修复 ML 代码中的错误

**形式**：
- 给出有数据泄漏、过拟合或逻辑错误的代码，要求找出并修复
- 给出错误信息，要求推断原因
- 要求解释为什么某段代码的评估结果不正确

**难度范围**：Level 2-5

**示例**：
```
以下代码的评估结果有问题。找出并修复。

scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)
X_train, X_test, y_train, y_test = train_test_split(X_scaled, y, test_size=0.2)
model = LogisticRegression()
model.fit(X_train, y_train)
print(f"测试准确率: {model.score(X_test, y_test):.3f}")
```

**生成规则**：
- 每题包含 1-3 个错误
- 错误类型覆盖：数据泄漏、指标误用、参数错误、代码逻辑错误
- 提供足够的信息让学生定位错误
- 错误应该是常见的、有教学价值的

---

### 类型 6: 迁移题（Transfer）

**目标**：测试学生能否将所学知识应用到新场景

**形式**：
- 给出一个新数据集，要求选择合适的算法并评估
- 要求将分类技能迁移到回归问题
- 结合多个技能解决实际问题

**难度范围**：Level 3-5

**示例**：
```
你之前用随机森林做过分类任务。现在有一个新的回归任务：
预测波士顿房价。请选择合适的模型、评估指标，
并说明与分类任务的主要区别。
```

**生成规则**：
- 场景描述贴近实际（医学诊断、金融风控、推荐系统等）
- 可以使用之前学过的技能
- 不要求使用未学过的库
- 鼓励学生思考算法选择的依据

---

### 类型 7: 小项目题（Mini-Project）

**目标**：测试学生能否综合运用多个技能完成端到端 ML 项目

**形式**：
- 给出一个完整的项目需求
- 要求设计和实现完整的 ML 流程
- 包含数据处理、模型训练、评估、可视化

**难度范围**：Level 4-5

**示例**：
```
完成一个客户流失预测项目：
1. 加载并探索数据集
2. 数据清洗（处理缺失值、编码类别变量）
3. 特征工程（创建新特征、特征选择）
4. 对比至少 3 种分类器
5. 用交叉验证评估，选择最优模型
6. 输出分类报告、ROC 曲线、特征重要性
7. 讨论模型的业务价值和局限性
```

**生成规则**：
- 项目规模：50-150 行代码
- 涵盖 4-6 个技能
- 有明确的验收标准
- 允许使用 sklearn / pandas / numpy / matplotlib

## ML 特有的任务适配规则

### 数据场景设计
- 使用真实或模拟的数据集（如 Titanic、Iris、房价数据）
- 数据量适中（100-10000 行），便于快速运行
- 包含常见的数据问题（缺失值、异常值、类别不平衡）

### 代码评估标准
1. **正确性**：模型能训练、预测、评估
2. **规范性**：先划分再预处理，使用 Pipeline
3. **可复现性**：设置了 random_state
4. **评估完整性**：使用了合适的评估指标
5. **代码可读性**：变量命名清晰，有注释

### 常见考察点
- fit vs fit_transform vs transform 的区别
- 训练集和测试集的正确使用
- 评估指标的选择依据
- 过拟合/欠拟合的判断和应对
- 算法选择的数据依据

## 任务生成流程

1. **确定目标技能**：根据 mastery-tracker.md 确定当前要练习的技能
2. **确定目标 Level**：根据当前 Level 选择目标 Level（当前 Level + 1）
3. **选择任务类型**：根据目标 Level 和难度分配选择任务类型
4. **生成任务内容**：
   - 使用模板生成任务
   - 确保任务与当前技能相关
   - 避免使用未学过的概念
   - 提供足够的数据和代码上下文
5. **设定成功标准**：明确、可衡量的完成条件
6. **预测常见错误**：根据 mistakes.md 预测学生可能犯的错误
