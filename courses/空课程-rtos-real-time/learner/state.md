## 高发弱点预判

### 裸机思维残留

- 症状：在 RTOS 任务中用 while(flag==0); 忙等
- 根因：不理解阻塞（Blocked）——你应该用 ulTaskNotifyTake(pdTRUE, portMAX_DELAY) 等待事件
- 纠正方法：对比 CPU 利用率——忙等 100%，阻塞 0%，展示差距

### IPC 选择

- 症状：所有任务间通信用全局变量+volatile
- 根因：裸机编程中全局变量是唯一的"跨 ISR 通信"方式
- 纠正方法：解释"为什么全局变量在多任务中不安全"（非原子操作、编译器重排、缓存一致性问题）

### Binary Semaphore vs Mutex

- 症状：用 Semaphore 做互斥保护
- 根因：两个 API 看起来一样（都是 Take/Give），但 Semaphore 没有优先级继承
- 纠正方法：用 3 任务优先级反转的场景演示"用 Mutex 能解决，用 Semaphore 不能"

### ISR 中调阻塞 API

- 症状：ISR 中用 xQueueSend(..., portMAX_DELAY)
- 根因：裸机中 ISR 是最高优先级的"大循环"
- 纠正方法：解释"ISR 不是任务，没有 TCB，阻塞了调度器不知道怎么唤醒你"
