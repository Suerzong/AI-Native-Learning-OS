## 常见误区样例

### 混淆引用和指针

- 所属技能：函数新特性
- 错误表现：认为 `int& r = x;` 和 `int* p = &x;` 是一回事，调用时不知道什么时候用 `&`
- 错误原因：C 中没有引用概念，直接用 C 的指针类比
- 正确理解：引用是别名，必须初始化、不能为 null、不能重新绑定；指针可以为 null、可以重新赋值。调用引用传参函数时不需要 `&`，直接传变量
- 纠正任务：写两个 swap 函数（指针版和引用版），对比调用处的语法差异
- 是否已复测通过：否

### new/delete 和 malloc/free 混用

- 所属技能：new/delete
- 错误表现：用 `new` 分配后用 `free` 释放，或用 `malloc` 后用 `delete`
- 错误原因：以为它们是等价的
- 正确理解：`new` 调用构造函数，`delete` 调用析构函数；`malloc`/`free` 不调用。混用会导致未定义行为（构造函数不调用或析构函数不调用）
- 纠正任务：写一个带构造/析构打印的类，分别用 new/delete 和 malloc/free 管理，观察输出差异
- 是否已复测通过：否

### 类定义末尾忘记分号

- 所属技能：类与对象
- 错误表现：`class Foo { ... }` 后面没有 `;`
- 错误原因：C 的 struct 定义后也需要分号，但可能习惯性忘记
- 正确理解：C++ 的 class 定义和 struct 定义一样，大括号后面必须有分号
- 纠正任务：每次写完类定义后检查分号
- 是否已复测通过：否

### range-based for 中不用引用导致无法修改元素

- 所属技能：循环
- 错误表现：`for (auto x : vec) x *= 2;` 发现 vec 中的元素没有变化
- 错误原因：`auto x` 是值拷贝，修改的是副本
- 正确理解：要修改容器元素，必须用 `for (auto& x : vec) x *= 2;`
- 纠正任务：写一个程序分别用引用和不用引用修改 vector 元素，对比结果
- 是否已复测通过：否

### 虚析构函数忘记声明

- 所属技能：多态
- 错误表现：通过基类指针 `delete` 派生类对象时，派生类的析构函数没有被调用
- 错误原因：基类析构函数没有声明为 `virtual`
- 正确理解：如果一个类可能被继承并有多态删除，析构函数必须声明为 `virtual`
- 纠正任务：写一个基类/派生类，基类析构函数不加 virtual，通过基类指针 delete，观察输出；然后加 virtual 对比
- 是否已复测通过：否

### 模板定义放在 .cpp 文件中

- 所属技能：函数模板 / 类模板
- 错误表现：链接时报 undefined reference
- 错误原因：模板在编译期实例化，编译器在编译 .cpp 时看不到模板定义
- 正确理解：模板的定义通常放在头文件中，或者在 .cpp 底部显式实例化
- 纠正任务：把模板定义移到头文件中，重新编译
- 是否已复测通过：否
