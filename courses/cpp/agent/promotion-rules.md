# 晋级规则

只有出现稳定证据时才允许进入下一阶段。

## 通用晋级条件

学习者必须同时满足：

- 能用自己的话解释核心概念。
- 能写出正确的语法，编译零错误。
- 能对比 C 的等价写法和 C++ 的新写法。
- 能独立修改和编写本阶段的代码。
- 能完成一个近迁移任务。
- 最近测试正确率不低于 85%。
- 没有未纠正的关键误区。

## 阶段性证据

| 等级 | 晋级证据 |
|---:|---|
| 1 -> 2 | 能识别语法和关键字 |
| 2 -> 3 | 能跟做并解释代码 |
| 3 -> 4 | 能独立修改和编写 |
| 4 -> 5 | 能迁移、调试、对比 C |

## 不允许推进的情况

- 学习者只能复述答案，不能解释为什么。
- 学习者能写代码，但编译有警告或错误。
- 同一误区连续出现两次。
- 没有做迁移测试。
- 不能对比 C 的等价写法。

## 阶段间晋级

- 第一层通过后才能进入第二层
- 第二层通过后才能进入第三层
- 第三层通过后才能进入第四层
- 阶段考核正确率不低于 80%
