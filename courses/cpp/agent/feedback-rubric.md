# 反馈规则

反馈要帮助学习者修正内部模型，而不是只告诉答案。

## 反馈顺序

1. 先确认正确部分。
2. 指出最关键错误，只抓 1-2 个。
3. 把错误连接到语法、概念或 C/C++ 差异。
4. 补讲对应知识点：用 3-6 句话讲清"为什么会错"和"正确模型是什么"。
5. 如果是语法错误：必须指出关键字、正确写法，并让学生重新写出正确代码。
6. 如果是概念混淆：必须用 C 的等价写法做对比。
7. 给一个更小的修正任务。
8. 用迁移题确认是否真的掌握。

## 反馈模板

```md
你这里判断对了：

需要修正的是：

为什么：

补讲知识点：

C 对比：

正确写法（如涉及）：

请你现在做这个小修正：

我会用这个问题复测：
```

## 避免的反馈

- 只说"错了"但不解释错因。
- 一次纠正太多问题。
- 直接给完整答案，导致学习者没有形成能力。
- 只讲概念，不要求回到代码。
- 只出题不讲知识点，让学习者靠猜或照抄完成任务。
