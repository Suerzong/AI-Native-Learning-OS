# 任务生成规则

任务必须服务于当前最小薄弱点。每个任务前必须先给出足够完成任务的知识点讲解，不能直接把学习者丢进题目。

## 任务前知识讲解

生成任务前，先生成"本次知识点"。它必须回答：

- 这个知识点解决什么问题。
- C 语言中的等价写法是什么。
- C++ 的新写法是什么。
- 为什么 C++ 的写法更好。
- 初学者最容易把哪两个概念混淆。
- 本次任务只要求掌握到什么边界。

如果学习者等级是 0-2，知识讲解必须先用直观语言解释，再给语法。
如果学习者等级是 3-5，可以减少基础解释，但仍要说明本次任务依赖的关键知识。

## 任务类型

| 类型 | 作用 | 示例 |
|---|---|---|
| 识别题 | 建立地图 | 识别 auto 推导的类型 |
| 解释题 | 建立理解 | 解释引用和指针的区别 |
| C→C++ 改写题 | 形成迁移能力 | 用 string 替换 char[] |
| 语法编写题 | 形成语法能力 | 写出带初始化列表的构造函数 |
| 代码编写题 | 形成从零写代码能力 | 不参考示例，写出完整的类定义 |
| 调试题 | 形成排错能力 | 找出编译错误并修复 |
| 迁移题 | 检查泛化 | 将学到的模式应用到新场景 |
| 小项目 | 组合技能 | 用类组织一个简单的命令行工具 |

## 好任务的格式

```md
## 任务名称

- 本次知识点：
- 目标技能：
- 起始材料：
- 任务要求：
- 限制条件：
- 成功标准：
- 常见错误：
- 复测问题：
```

## C→C++ 改写题格式

```md
## 任务名称

- 目标技能：
- C 原始代码：
- 改写要求（哪些部分用 C++ 特性替代）：
- 必须使用的新特性：
- 成功标准：
- 常见错误：
```

## 语法编写题格式

```md
## 任务名称

- 目标语法：
- 所属知识点：
- 写出完整代码的要求：
- 常见错误（语法遗漏、关键字混淆等）：
```

## 难度控制

- 如果学习者等级是 0-1：给识别题和解释题。
- 如果学习者等级是 2：给 C→C++ 改写题、语法解释题。
- 如果学习者等级是 3：给调试题、语法编写题（不看示例写出代码）。
- 如果学习者等级是 4：给迁移题、代码编写题（从零写出完整程序）。
- 如果学习者等级是 5：给小项目（多文件、多类、STL 综合使用）。
