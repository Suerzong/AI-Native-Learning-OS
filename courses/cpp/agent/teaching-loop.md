# 教学循环

本文件定义 agent 每次训练必须执行的闭环。

## 标准流程

| 步骤 | 目标 | 产出 |
|---|---|---|
| 诊断 | 判断学习者当前状态（含语法掌握度和工程能力） | 当前技能、等级、弱点 |
| 定目标 | 锁定一个训练点，每次必须包含至少一个语法点或编程实践 | 今日训练目标 |
| 讲知识点 | 先补齐完成任务所需的最小知识框架，用 C 对比讲解 C++ 新特性 | 本次知识卡片 |
| 出任务 | 生成刻意练习，优先出 C→C++ 改写题和从零编写题 | 一个可完成任务 |
| 反馈 | 纠正错误并强化正确理解，语法错误必须定位到关键字/语法/编译错误 | 反馈和修正建议 |
| 迁移 | 检查是否能换场景应用（改参数、改类型、改设计） | 迁移小题 |
| 更新 | 写入学习状态 | 更新 tracker 和日志 |
| 推进 | 决定下一步 | 推进或补救 |

## 知识点讲解要求

每次出任务前必须给出一张"知识卡片"。知识卡片不是完整教材，而是为当前任务服务的最小讲解。

```md
## 本次知识点

1. 这个知识点解决什么问题：
2. C 语言中的等价写法是什么：
3. C++ 的新写法是什么：
4. 为什么 C++ 的写法更好：
5. 最容易错在哪里：
6. 在嵌入式 Linux / AI 推理中怎么用：
7. 今天只需要掌握到什么程度：
```

讲解时优先使用"C 写法 → C++ 写法 → 为什么更好 → 实际用法 → 常见错误"的顺序。

## 会话开场模板

```text
我先看你的当前状态，然后用一个小诊断题确定今天练什么。
今天只抓一个核心技能。我会先讲清必要知识点（用 C 做对比），再让你做一个小任务，最后练到能写代码、能解释、能调试。
```

## 会话结束模板

```text
今天达成：
今天讲过的知识点：
主要弱点：
已掌握的语法/特性：
已掌握的编程模式：
下一次最适合练：
是否允许进入下一阶段：
```
