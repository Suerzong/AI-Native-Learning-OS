# 阶段考核

阶段考核用于判断是否形成稳定能力，而不是检查是否背过答案。

## 考核模板

```md
## 阶段名称

- 覆盖技能：
- 考核时间：
- 允许资料：
- 任务：
- 评分标准：
- 晋级条件：
```

---

## 第一层考核：C++ 基础

- 覆盖技能：开发环境、数据类型、new/delete、循环、函数新特性、string 类
- 考核时间：45 分钟
- 允许资料：cppreference.com、自己的笔记、编译器
- 任务：
  1. 用 C++ 写一个命令行学生成绩管理程序：
     - 用 `std::string` 存储学生姓名
     - 用 `std::vector<int>` 存储成绩（或者用数组）
     - 写重载函数 `print` 分别打印单个学生和所有学生
     - 用引用传参实现 `addScore` 函数
     - 用 range-based for 遍历输出
  2. 解释程序中每个 C++ 新特性对应的 C 等价写法
  3. 说明如果把 `string` 换成 `char[]` 会有什么问题
- 晋级条件：
  - 程序能编译运行
  - 正确使用至少 5 个 C++ 新特性
  - 能解释引用和指针的区别
  - 能解释 new/delete 和 malloc/free 的区别

---

## 第二层考核：面向对象

- 覆盖技能：类与对象、构造析构、继承、多态、运算符重载
- 考核时间：60 分钟
- 允许资料：cppreference.com、自己的笔记
- 任务：
  1. 设计一个嵌入式设备驱动框架：
     - 抽象基类 `Device`：纯虚函数 `init()`、`read()`、`write()`、虚析构函数
     - 派生类 `UARTDevice`：实现串口读写（模拟实现即可）
     - 派生类 `SPIDevice`：实现 SPI 读写
     - 重载 `operator<<` 输出设备信息
     - 用基类指针 `Device*` 创建派生类对象，调用虚函数实现多态
  2. 解释构造函数和析构函数的调用顺序
  3. 解释为什么基类析构函数要声明为 virtual
  4. 解释运算符重载用成员函数和友元函数的区别
- 晋级条件：
  - 多态正确工作
  - 构造析构顺序正确
  - 无内存泄漏（通过 valgrind 或 ASAN 检查）
  - 能解释 vtable 的基本概念

---

## 第三层考核：泛型编程与 STL

- 覆盖技能：函数模板、类模板、STL 容器、STL 算法、智能指针
- 考核时间：60 分钟
- 允许资料：cppreference.com、自己的笔记
- 任务：
  1. 用模板和 STL 实现传感器数据管理系统：
     - 类模板 `SensorBuffer<T>`：内部用 `std::vector<T>` 存储数据，提供 `add()`、`size()`、`getAverage()` 方法
     - 实例化 `SensorBuffer<double>` 存储温度数据
     - 用 `std::sort` 按温度排序
     - 用 `std::find_if` 查找超过阈值的数据
     - 用 `std::for_each` + Lambda 打印所有数据
     - 用 `std::shared_ptr` 管理 SensorBuffer 对象
  2. 解释模板实例化的概念
  3. 解释 `vector` 的内存增长策略
  4. 解释 `shared_ptr` 的引用计数机制
- 晋级条件：
  - 程序能编译运行
  - 正确使用至少 3 个 STL 容器和 3 个 STL 算法
  - 无内存泄漏
  - 能解释 vector 和 list 的性能差异

---

## 第四层考核：工程实战

- 覆盖技能：异常处理、命名空间、文件 I/O、Lambda、多线程、CMake
- 考核时间：90 分钟
- 允许资料：cppreference.com、CMake 文档、自己的笔记
- 任务：
  1. 用 CMake 组织一个多文件 C++ 项目：
     - `include/sensor.h`：声明 Sensor 类
     - `src/sensor.cpp`：实现 Sensor 类
     - `src/main.cpp`：主程序
     - `CMakeLists.txt`：构建配置
     - 功能：读取传感器数据文件（文本格式），用多线程分离读取和处理，处理结果写入输出文件
     - 用 `std::thread` 和 `std::mutex` 实现线程安全
     - 用 Lambda 作为线程函数
     - 用异常处理文件打开失败
     - 代码在 `sensor` 命名空间内
  2. 写出交叉编译的 CMake 命令（指定工具链文件）
  3. 解释嵌入式环境下哪些 C++ 特性要慎用
- 晋级条件：
  - CMake 构建成功
  - 多线程正确运行，无竞态条件
  - 文件读写正确
  - 能解释交叉编译流程
  - 能列出至少 3 个嵌入式环境的 C++ 约束
