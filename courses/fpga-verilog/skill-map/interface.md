# 接口协议技能图

## 目标

掌握常用串行通信协议（UART、SPI、I2C）和显示接口（VGA）的 Verilog 实现。

## 必知概念

- 波特率（Baud Rate）和采样时机
- SPI 的 CPOL/CPHA 四种模式
- I2C 的起始/停止条件和应答机制
- VGA 的行同步/场同步时序

## 核心协议概览

### UART

- 异步串行通信，无需时钟线
- 帧格式：起始位(1) + 数据位(8) + 停止位(1)
- 波特率：9600、115200 等
- 关键模块：TX 发送器、RX 接收器

### SPI

- 同步串行通信，4 线制（SCLK、MOSI、MISO、CS）
- 主从模式，全双工
- CPOL（时钟极性）和 CPHA（时钟相位）决定采样时机

### I2C

- 同步串行通信，2 线制（SCL、SDA）
- 多主多从，半双工
- 起始条件：SCL 高时 SDA 下降
- 停止条件：SCL 高时 SDA 上升

### VGA

- 模拟视频接口，行同步 + 场同步
- 时序：同步脉冲 → 后沿 → 有效区域 → 前沿
- 像素时钟由分辨率决定（640×480@60Hz → 25.175MHz）

## 常见错误

| 协议 | 错误 | 纠正 |
|------|------|------|
| UART | 采样时机不在位中间 | 在每位中间点采样 |
| SPI | CPOL/CPHA 搞混 | 查表确认 4 种模式 |
| I2C | SDA 在 SCL 高电平变化 | 只在 SCL 低电平时改变 SDA |
| VGA | 同步参数计算错误 | 严格按标准时序参数 |

## 掌握标准

- 能实现 UART TX 和 RX，支持 8N1
- 能实现 SPI Master，支持基本读写
- 能实现 VGA 控制器，显示彩条
