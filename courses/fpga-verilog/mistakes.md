## 常见误区样例

### 阻塞赋值用于时序逻辑

- 所属技能：D 触发器与寄存器
- 错误表现：在 always @(posedge clk) 中使用 `q = d` 而非 `q <= d`
- 错误原因：把 Verilog 当 C 语言，不理解并行执行模型
- 正确理解：时序逻辑必须用非阻塞赋值（<=），保证所有寄存器在同一时钟沿同时更新
- 纠正任务：记住口诀——"组合用 =，时序用 <="
- 是否已复测通过：否

### if 缺 else 导致锁存器

- 所属技能：多路选择器
- 错误表现：组合逻辑 always 块中 if 没有写 else
- 错误原因：习惯性按 C 语言写 if，忽略硬件推断规则
- 正确理解：组合逻辑的 if-else 必须覆盖所有情况，否则综合器会推断锁存器来保持未赋值时的状态
- 纠正任务：每次写 if 时先写好 else 分支
- 是否已复测通过：否

### 位宽不匹配导致截断

- 所属技能：加法器与减法器
- 错误表现：8 位加法结果赋给 8 位变量，进位丢失
- 错误原因：没有考虑到 8 位 + 8 位最大可能产生 9 位结果
- 正确理解：N 位 + N 位最多产生 N+1 位结果，需要用拼接 `{cout, sum}` 接收
- 纠正任务：每次做算术运算前检查结果位宽
- 是否已复测通过：否

### 计数器模值判断错误

- 所属技能：计数器
- 错误表现：10 进制计数器写成 `if (cnt == 10) cnt <= 0;`
- 错误原因：忘记计数从 0 开始，10 进制应该数 0-9
- 正确理解：N 进制计数器的终值是 N-1，即 `if (cnt == N-1) cnt <= 0;`
- 纠正任务：画出计数序列 0,1,2,...,N-1,0,1,... 确认
- 是否已复测通过：否

### 逗号与分号混用（2026-05-16 新增）

- 所属技能：Verilog 基础语法
- 错误表现：
  1. 端口列表最后的 `)` 后忘记加分号 → `);` 写成 `)`
  2. 多条 assign 语句之间用逗号分隔 → `assign x=a, assign y=b`
- 错误原因：初学语法细节，与 C 语言语句习惯混用
- 正确理解：端口列表用逗号分隔信号，列表结束后用分号 `);`；每条 assign 独立，末尾分号
- 纠正任务：记住口诀——"端口用逗号，语句用分号；列表结束加分号"
- 是否已复测通过：否

### 用 C 语言思维写 Verilog（2026-05-16 新增）

- 所属技能：多位运算与总线
- 错误表现：用 `a = input[3];` 语法给信号赋值，逐位写 `assign y[0]=..., assign y[1]=...`
- 错误原因：将 Verilog 信号当 C 数组使用，不理解总线是整捆操作
- 正确理解：多位信号是整捆线，`[3:0]` 一次声明，整捆一起操作 `assign x = a & b;`
- 纠正任务：每次看到 `[N:0]`，在脑子里画 N+1 根并排的线，不要拆成单个 bit
- 是否已复测通过：否

### ? : 映射方向混淆（2026-05-17 新增）

- 所属技能：多路选择器
- 错误表现：4:1 MUX 写出 `s[0]?d0:d1`，导致 00 选 d1 而非 d0（映射全反）
- 错误原因：未建立"二进制选择信号 = 编号"的直觉（s=00 → 编号 0 → 选 d0）
- 正确理解：`s[0]?d1:d0`（s[0]=0 选 d0，符合二进制计数约定）
- 纠正任务：每次写 MUX 前先在纸上画真值表，确认 00→d0, 01→d1, 10→d2, 11→d3
- 是否已复测通过：是

### assign 关键字遗漏（2026-05-17 新增）

- 所属技能：组合逻辑基础
- 错误表现：module 内部直接写 `y = ...;`，缺少 `assign` 关键字
- 错误原因：受 C 语言习惯影响，认为 `=` 本身就是赋值语句
- 正确理解：Verilog 中 `=` 不能单独出现在 module 内部，组合逻辑必须用 `assign y = ...;`
- 纠正任务：记住口诀——"module 里面没有裸 =，要么 assign 要么 always"
- 是否已复测通过：否

### 端口位宽声明语法（2026-05-16 首次出现，2026-05-17 重复）

- 所属技能：多位运算与总线
- 错误表现：`input d0[3:0]` 写成 `input [3:0] d0` 的反向（位宽放信号名后）
- 错误原因：与 C 数组 `int arr[4]` 混淆
- 正确理解：Verilog 位宽在信号名前 `input [N:0] signal_name`，与 C 数组相反
- 纠正任务：每次声明端口时先写位宽再写名：`[3:0]` 然后 `d0`
- 是否已复测通过：否（5/17 再次出现）

### 状态机缺 default 状态

- 所属技能：有限状态机
- 错误表现：case 语句没有 default 分支
- 错误原因：认为所有状态都已覆盖，不需要 default
- 正确理解：综合后的电路可能因为干扰进入非法状态，必须有 default 分支将状态机拉回合法状态
- 纠正任务：每个状态机的 case 语句都加 default: next = S0;
- 是否已复测通过：否
