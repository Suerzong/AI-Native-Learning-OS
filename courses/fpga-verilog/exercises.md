## 门电路-01：基本门电路

- 目标技能：用 assign 实现基本逻辑门
- 起始文件：无
- 任务要求：用 Verilog 实现以下模块，每个门用一行 assign 语句
  ```verilog
  module gates(input a, b, output and_out, or_out, not_out, xor_out, nand_out, nor_out);
  ```
- 限制条件：只用 assign，不用 always
- 成功标准：仿真输入 a=1,b=0 时，and_out=0, or_out=1, not_out=0, xor_out=1, nand_out=1, nor_out=0
- 常见错误：混淆 `~`（按位取反）和 `!`（逻辑取反）；与非门写成 `~&` 而非 `~(a & b)`
- 反馈重点：每条 assign 对应一个门电路，画出门电路符号

## 门电路-02：多位门电路

- 目标技能：对多位信号进行按位运算
- 起始文件：无
- 任务要求：实现 4 位按位取反模块
  ```verilog
  module not4(input [3:0] in, output [3:0] out);
  ```
- 限制条件：一行 assign 语句
- 成功标准：输入 4'b1010 输出 4'b0101
- 常见错误：把 `~in` 写成 `!in`（逻辑取反只产生 1 位结果）
- 反馈重点：`~` 是按位操作，`!` 是逻辑操作，结果不同

## MUX-01：2 选 1 多路选择器

- 目标技能：用条件运算符实现 MUX
- 起始文件：无
- 任务要求：实现 2 选 1 MUX
  ```verilog
  module mux2(input d0, d1, s, output y);
  ```
- 限制条件：只用 assign + 条件运算符
- 成功标准：s=0 时 y=d0，s=1 时 y=d1
- 常见错误：条件写反（s?d0:d1 而非 s?d1:d0）
- 反馈重点：三元运算符 `? :` 综合后就是 MUX 电路

## MUX-02：4 选 1 多路选择器

- 目标技能：用嵌套条件运算符实现 4:1 MUX
- 起始文件：无
- 任务要求：实现 4 选 1 MUX
  ```verilog
  module mux4(input d0, d1, d2, d3, input s0, s1, output y);
  ```
- 限制条件：只用 assign
- 成功标准：s1s0=00→d0, 01→d1, 10→d2, 11→d3
- 常见错误：选择信号的位顺序搞反
- 反馈重点：4:1 MUX 需要 2 位选择信号，对应 2 级 MUX

## 编码-01：3-8 译码器

- 目标技能：用移位操作实现译码器
- 起始文件：无
- 任务要求：实现 3-8 译码器
  ```verilog
  module decoder38(input [2:0] in, output [7:0] out);
  ```
- 限制条件：一行 assign
- 成功标准：in=3'd3 时 out=8'b00001000
- 常见错误：移位方向搞反（应该是 `1'b1 << in`）
- 反馈重点：译码器本质就是"1 左移 N 位"

## 编码-02：8-3 优先编码器

- 目标技能：用 casex 实现优先编码
- 起始文件：无
- 任务要求：实现 8-3 优先编码器，带 valid 输出
  ```verilog
  module encoder83(input [7:0] in, output reg [2:0] out, output reg valid);
  ```
- 限制条件：用 always @(*) + casex
- 成功标准：in=8'b00100000 时 out=3'd5, valid=1；in=0 时 valid=0
- 常见错误：casex 的 x 匹配规则不清；忘记 default 处理全 0 输入
- 反馈重点：优先编码器从高位开始匹配，casex 中 x 匹配 0 和 1

## 加法器-01：半加器与全加器

- 目标技能：理解半加器和全加器的区别
- 起始文件：无
- 任务要求：分别实现半加器和全加器
- 限制条件：只用 assign
- 成功标准：半加器 1+1=0 进位 1；全加器 1+1+1=1 进位 1
- 常见错误：全加器的进位表达式写错（应该是 a&b | a&cin | b&cin）
- 反馈重点：半加器不考虑进位输入，全加器考虑

## 加法器-02：4 位加法器

- 目标技能：用算术运算符和拼接实现多位加法
- 起始文件：无
- 任务要求：实现 4 位全加器
  ```verilog
  module adder4(input [3:0] a, b, input cin, output reg [3:0] sum, output reg cout);
  ```
- 限制条件：用 always @(*) 和拼接运算符
- 成功标准：a=4'd15, b=4'd1, cin=0 → sum=0, cout=1
- 常见错误：`{cout, sum} = a + b + cin` 的拼接顺序
- 反馈重点：拼接运算符 `{}` 可以把进位和和拼在一起

## 触发器-01：D 触发器

- 目标技能：理解非阻塞赋值和边沿触发
- 起始文件：无
- 任务要求：实现 D 触发器
  ```verilog
  module dff(input clk, d, output reg q);
  ```
- 限制条件：用 always @(posedge clk)，必须用 <=
- 成功标准：clk 上升沿时 q 更新为 d
- 常见错误：用 `=` 而非 `<=`（非阻塞赋值是时序逻辑的标准写法）
- 反馈重点：非阻塞赋值保证所有寄存器同时更新

## 计数器-01：4 位二进制计数器

- 目标技能：实现带复位的计数器
- 起始文件：无
- 任务要求：实现 4 位二进制计数器
  ```verilog
  module counter4(input clk, rst, output reg [3:0] cnt);
  ```
- 限制条件：同步复位，always @(posedge clk)
- 成功标准：复位时 cnt=0，每个 clk 上升沿加 1，到 15 后回 0
- 常见错误：复位条件写成 `if (!rst)` 而非 `if (rst)`（高有效复位）
- 反馈重点：4 位计数器自然溢出到 0，不需要额外判断

## 状态机-01：简单三状态状态机

- 目标技能：用三段式实现状态机
- 起始文件：无
- 任务要求：实现一个检测连续两个 1 的状态机
  - S0：初始状态，输出 0
  - S1：收到一个 1，输出 0
  - S2：收到连续两个 1，输出 1
- 限制条件：用三段式（状态转移、次态逻辑、输出逻辑分开）
- 成功标准：输入 1,1,0,1,1 对应输出 0,1,0,0,1
- 常见错误：三段混在一个 always 块中；输出逻辑用组合还是时序搞混
- 反馈重点：三段式的好处是清晰分离功能，便于维护
