# 任务生成规则

任务必须服务于当前最小薄弱点。每个任务前必须先给出足够完成任务的知识点讲解，不能直接把学习者丢进题目。

## 参考资料优先级

生成任务和知识点讲解时，按以下顺序查阅资料：

1. **本地题库**：`~/Edge-AI/temp/zzfpga-question-bank/fpga-question-bank.md` — F 学社 FPGA 题库（345 道含答案）
2. **课程技能图**：`skill-map/*.md` — 对应主题的概念和代码示例
3. **在线平台**：https://www.zzfpga.com — 在线提交和验证
4. **参考教程**：HDLBits、runoob.com Verilog 教程

## 任务前知识讲解

生成任务前，先生成"本次知识点"。它必须回答：

- 这个电路模块的功能是什么（用真值表或波形图说明）
- 它对应的硬件结构是什么（画门电路图或框图）
- Verilog 实现的关键代码结构是什么
- 综合后会生成什么电路资源
- 初学者最容易犯的错误是什么
- 本次任务只要求掌握到什么边界

如果学习者等级是 0-2，知识讲解必须先用电路图和真值表解释，再给 Verilog 代码。
如果学习者等级是 3-5，可以减少基础解释，但仍要说明本次任务依赖的关键知识和易错点。

## 任务类型

| 类型 | 作用 | 示例 |
|------|------|------|
| 概念题 | 建立直觉 | wire 和 reg 的区别是什么？ |
| 电路题 | 理解硬件映射 | 画出 assign out = a & b 对应的门电路 |
| 代码题 | 形成实现能力 | 手写 4 位加法器 |
| 分析题 | 形成分析能力 | 分析以下代码综合后的电路结构 |
| 找错题 | 形成调试能力 | 找出以下代码中的锁存器推断问题 |
| 迁移题 | 检查泛化 | 从 2:1 MUX 迁移到 8:1 MUX |
| 仿真题 | 形成验证能力 | 给出仿真波形，判断代码是否正确 |
| 综合题 | 组合技能 | 设计一个带使能和复位的 8 位计数器 |


## 难度递进原则

1. **识别**：能认出门电路的 Verilog 表示
2. **理解**：能解释代码对应的硬件电路
3. **模仿**：能跟着示例写代码并仿真
4. **独立**：能不看参考写出正确代码
5. **迁移**：能将学到的模块应用到新问题
6. **选型**：能在多种实现方式中选择最优的

## Verilog 特有的教学要点

### 必须解释的概念（首次出现时）

- **assign vs always**：assign 是连线，always 是行为描述
- **wire vs reg**：wire 用于 assign 目标，reg 用于 always 内部
- **阻塞 vs 非阻塞**：组合逻辑用 =，时序逻辑用 <=
- **可综合 vs 不可综合**：$display、initial、#delay 不可综合
- **锁存器 vs 触发器**：电平敏感 vs 边沿敏感

### 每个代码题必须包含

1. 电路需求描述（真值表或功能描述）
2. 对应的电路图或框图
3. Verilog 代码模板（module 声明）
4. 仿真验证方法（测试用例）

### 答案纠正规则

纠正错误时，按以下顺序：
1. 指出错误所在行
2. 说明错误类型（语法/逻辑/硬件映射）
3. 解释为什么错（对应的电路行为）
4. 给出正确写法
5. 说明正确写法对应的电路
6. 给出一个验证方法

## 题库使用规则

- 优先从 F 学社题库中选取对应技能点的题目
- 题目格式：Q93（或门）、Q129（D 触发器）等
- 题库中已有完整答案（含 Verilog 参考代码），可直接使用
- 对于题库中没有覆盖的知识点，自行设计题目
