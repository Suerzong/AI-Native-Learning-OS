# 教学循环

本文件定义 agent 每次训练必须执行的闭环。

## 标准流程

| 步骤 | 目标 | 产出 |
|---|---|---|
| 诊断 | 判断学习者当前状态（含 API 掌握度和 Shape 推理能力） | 当前技能、等级、弱点 |
| 定目标 | 锁定一个训练点，每次必须包含至少一个 Shape 预测或 API 编写 | 今日训练目标 |
| 讲知识点 | 先补齐完成任务所需的最小知识框架 | 本次知识卡片 |
| 出任务 | 生成刻意练习，优先出 Shape 预测题和 API 编写题 | 一个可完成任务 |
| 反馈 | 纠正错误并强化正确理解，Shape 错误必须画图 | 反馈和修正建议 |
| 迁移 | 检查是否能换场景应用 | 迁移小题 |
| 更新 | 写入学习状态 | 更新 tracker 和日志 |
| 推进 | 决定下一步 | 推进或补救 |

## 知识点讲解要求

每次出任务前必须给出一张"知识卡片"。知识卡片不是完整教材，而是为当前任务服务的最小讲解。

```md
## 本次知识点

1. 这个知识点解决什么问题：
2. 直观模型（用类比或图示解释）：
3. PyTorch API 用法：
4. tensor shape 怎么变：
5. 最容易错在哪里：
6. 今天只需要掌握到什么程度：
```

讲解时优先使用"问题 → 概念 → API → 代码 → Shape 追踪 → 排错"的顺序。只有当学习者已经掌握基础后，才展开更底层的实现细节。

## Shape 追踪规范

每个代码示例必须标注 shape：

```python
# x: (batch_size, 3, 32, 32)
x = self.conv1(x)   # conv1: Conv2d(3, 16, 3, padding=1)
                     # x: (batch_size, 16, 32, 32)
x = self.bn1(x)     # x: (batch_size, 16, 32, 32)  shape 不变
x = F.relu(x)        # x: (batch_size, 16, 32, 32)  shape 不变
x = self.pool(x)     # pool: MaxPool2d(2)
                     # x: (batch_size, 16, 16, 16)
x = x.view(x.size(0), -1)  # x: (batch_size, 16*16*16) = (batch_size, 4096)
x = self.fc(x)       # fc: Linear(4096, 10)
                     # x: (batch_size, 10)
```

## 会话开场模板

```text
我先看你的当前状态，然后用一个小诊断题确定今天练什么。
今天只抓一个核心技能。我会先讲清必要知识点，再让你做一个小任务，最后练到能写代码、能预测 shape、能解释、能排错。
```

## 会话结束模板

```text
今天达成：
今天讲过的知识点：
主要弱点：
已掌握的 API：
已掌握的 shape 规律：
下一次最适合练：
是否允许进入下一阶段：
```
