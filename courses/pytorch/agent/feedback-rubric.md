# 反馈规则

反馈要帮助学习者修正内部模型，而不是只告诉答案。

## 反馈顺序

1. 先确认正确部分。
2. 指出最关键错误，只抓 1-2 个。
3. 把错误连接到代码、tensor shape 或概念。
4. 补讲对应知识点：用 3-6 句话讲清"为什么会错"和"正确模型是什么"。
5. 如果是 API 调用错误：必须指出函数名、参数含义、正确写法，并让学生重新写出正确调用。
6. 如果是 Shape 错误：必须在纸上画出 tensor shape 的变化过程，指出哪一步 shape 不对。
7. 给一个更小的修正任务。
8. 用迁移题确认是否真的掌握。

## 反馈模板

```md
你这里判断对了：

需要修正的是：

为什么：

补讲知识点：

正确代码/shape：

请你现在做这个小修正：

我会用这个问题复测：
```

## 避免的反馈

- 只说"错了"但不解释错因。
- 一次纠正太多问题。
- 直接给完整代码，导致学习者没有形成能力。
- 只讲概念，不要求回到代码或输出结果。
- 只出题不讲知识点，让学习者靠猜或照抄完成任务。

## PyTorch 特有的反馈重点

### Shape 错误反馈

必须画出 shape 变化链：
```
输入: (batch, 3, 32, 32)
Conv2d(3, 16, 3, padding=1): (batch, 16, 32, 32)  ← 如果这里算错，指出公式
ReLU: (batch, 16, 32, 32)  ← shape 不变
MaxPool2d(2): (batch, 16, 16, 16)  ← 每个维度除以 kernel_size
flatten(): (batch, 16*16*16)  ← 展平
Linear(4096, 10): (batch, 10)  ← 如果这里维度不对，往前追溯
```

### Device 错误反馈

必须指出哪个 tensor 在哪个设备上：
```
model → GPU ✓
input → CPU ✗  ← 问题在这里
修改：input = input.to(device)
```

### 训练循环错误反馈

必须标出每一步的顺序：
```
1. optimizer.zero_grad()     ← 如果漏了这步，梯度会累加
2. output = model(input)      ← forward
3. loss = criterion(output, target)
4. loss.backward()            ← 反向传播
5. optimizer.step()           ← 参数更新
```
