## 2026-05-06：倍增/折半嵌套循环复杂度迁移失败

- 所属技能：算法分析基础（大O）
- 错误表现：将 `for(i=1; i<n; i*=2) { for(j=0; j<i; j++) }` 的复杂度误判为 O(n²)
- 错误原因：看到嵌套循环就套 O(n²)，没有分析外层循环的实际执行次数
- 正确理解：外层 i 取值 1,2,4,...,n（log n 次），内层总和是等比数列 1+2+4+...+n = O(n)
- 纠正任务：画外层 i 的取值表，算内层总操作次数
- 是否已复测通过：是

## 2026-05-06：结构体指针访问写法错误

- 所属技能：数组与动态数组
- 错误表现：写 `da->data[size - 1]` 而不是 `da->data[da->size - 1]`
- 错误原因：通过结构体指针访问成员时，忘记用 `da->size`，直接写了 `size`
- 正确理解：结构体成员必须通过指针访问：`da->成员名`，不能只写成员名
- 纠正任务：写出通过指针访问结构体成员的正确语法
- 是否已复测通过：否

## 2026-05-06：数组下标边界检查不完整

- 所属技能：数组与动态数组
- 错误表现：`get` 函数只检查了 `index >= size`，没有检查 `index < 0`
- 错误原因：只想到上界越界，忽略了负数也是非法下标
- 正确理解：数组下标合法范围是 `0 <= index < size`，两端都要检查
- 纠正任务：写出完整的边界检查条件
- 是否已复测通过：是

## 2026-05-06：链表遍历条件写错（连犯两次）

- 所属技能：单链表
- 错误表现：遍历链表时写 `while (curr->next != NULL)`，导致最后一个节点被跳过
- 错误原因：混淆了"curr 的下一个"和"curr 本身"的判断
- 正确理解：遍历链表的循环条件永远是 `while (curr != NULL)`，需要检查当前节点本身
- 纠正任务：写链表遍历时，先写 `while (curr != NULL)`，养成肌肉记忆
- 是否已复测通过：是（后续 countVal、contains 均正确使用）

## 常见误区样例

### 链表插入时断链

- 所属技能：单链表
- 错误表现：在中间位置插入新节点后，后面的节点丢失
- 错误原因：先将新节点的 next 设为 NULL，再将前驱节点的 next 指向新节点，导致原后继节点丢失
- 正确理解：必须先保存后继节点的地址（`newNode->next = prev->next`），再改变前驱指针（`prev->next = newNode`）
- 纠正任务：画出插入前后链表的内存状态，标注每个指针的赋值顺序
- 是否已复测通过：否

### 快排分区指针方向搞反

- 所属技能：快速排序
- 错误表现：partition 后数组没有被正确分为"小于 pivot"和"大于 pivot"两部分
- 错误原因：j 从右往左找小于 pivot 的元素，但写成了找大于的
- 正确理解：以最右元素为 pivot 时，i 向右找 >= pivot 的，j 向左找 <= pivot 的，然后交换
- 纠正任务：手动模拟 partition([3,1,4,1,5,9,2,6]) 的每一步
- 是否已复测通过：否

### 二分查找边界条件错误

- 所属技能：二分查找
- 错误表现：查找存在元素时返回 -1，或死循环
- 错误原因：使用 `left < right` 但 mid 赋值为 `left` 没有 +1，导致死循环
- 正确理解：当使用 `left < right` 时，mid 需要用 `left + (right - left) / 2`，且更新边界时要 `left = mid + 1` 或 `right = mid`
- 纠正任务：写出两种边界写法（`left <= right` 和 `left < right`），分别适用什么场景
- 是否已复测通过：否

### 递归忘记写终止条件

- 所属技能：递归
- 错误表现：程序运行时栈溢出（Segmentation fault）
- 错误原因：直接写递归步骤，没有先写 base case
- 正确理解：写递归时必须先写终止条件，再写递归步骤。每个递归必须确保向终止条件收敛
- 纠正任务：重写 fibonacci 函数，先写 `if (n <= 1) return n;` 再写递归调用
- 是否已复测通过：否

### 循环队列空满判断混淆

- 所属技能：队列
- 错误表现：入队时误判队列已满，或空队列时出队
- 错误原因：用 `(rear + 1) % capacity == front` 判断满，但没有区分空和满
- 正确理解：循环队列区分空和满有三种常用方法：(1) 浪费一个位置 (2) 增加 size 计数器 (3) 增加 flag 标记
- 纠正任务：实现用 size 计数器区分空和满的循环队列
- 是否已复测通过：否

### malloc 后忘记 free

- 所属技能：链表 / 内存管理
- 错误表现：程序长时间运行后内存占用持续增长
- 错误原因：只关注了功能实现，没有考虑资源释放
- 正确理解：每个 malloc 必须有对应的 free。删除节点时要先 free 再断链，或先保存指针再断链再 free
- 纠正任务：为单链表实现 `void destroyList(ListNode* head)` 函数，释放所有节点
- 是否已复测通过：否

## 2026-05-09：创建节点时用栈变量而非 malloc

- 所属技能：单链表
- 错误表现：
```c
struct Node* createNode(int data) {
    struct Node new_node;        // ← 栈变量
    new_node.data = data;
    return &new_node;            // ← 返回栈变量地址 = 悬空指针
}
```
- 错误原因：不理解栈变量的生命周期——函数返回后栈变量被销毁，返回其地址会导致悬空指针
- 正确理解：链表节点必须用 malloc 在堆上分配，堆内存在 free 前一直有效
- 纠正任务：记住规则——凡是需要在函数外继续存在的数据，必须用 malloc 分配
- 是否已复测通过：是（后续 insertAt 中仍犯一次，经提醒后修正）

## 2026-05-13：结构体指针 . vs -> 反复混用

- 所属技能：栈、队列
- 错误表现：`s` 是 `SeqStack*` 指针时，反复写 `s.data[...]`、`s.top`，应写 `s->data[...]`、`s->top`
- 错误原因：看到变量名 `s` 未在大脑中自动判断它是指针还是实体，直接打 `.`
- 正确理解：函数参数中的 `s` 几乎总是指针（需要修改结构体内容），必须用 `->`
- 纠正任务：写完每行后立即检查：`s` 是指针？→ 用 `->` 不是 `.`
- 是否已复测通过：经 4 次纠正后基本稳定，仍需持续自查

## 2026-05-13：栈 pop 时忘记解引用 *x

- 所属技能：栈
- 错误表现：pop 中写 `x = s->data[s->top--]` 而非 `*x = s->data[s->top--]`
- 错误原因：push 的 x 是 int（直接使用），pop 的 x 是 int*（需要解引用写入），两者格式不同但大脑按 push 的惯性写
- 正确理解：`int *x` 中 `x` 是地址，`*x` 才是调用者的变量。传给调用者的值必须写到 `*x`
- 纠正任务：识别函数参数中 `int` vs `int*` 的区别：`int*` 是出口参数，赋值必须 `*x = ...`
- 是否已复测通过：经 3 次纠正后掌握

## 2026-05-13：deQueue 误用 rear 和自减

- 所属技能：队列
- 错误表现：deQueue 中操作 `q->rear` 而非 `q->front`，且用 `--` 代替取模前进
- 错误原因：deQueue = 出队，大脑联想到 pop 的 `--top`，把 rear 当 top 用
- 正确理解：出队操作的是队头 front，入队操作的是队尾 rear。队列指针只前进不后退：`(指针 + 1) % MAXSIZE`
- 纠正任务：区分栈（一端操作，top 可上可下）和队列（两端操作，front/rear 只前进不复退）
- 是否已复测通过：是

## 2026-05-13：动态数组边界用 capacity 而非 size

- 所属技能：数组与动态数组
- 错误表现：`da_get` 中写 `if (index < 0 || index >= da->capacity)`，而非 `da->size`
- 错误原因：混淆了 capacity（已分配空间）和 size（实际元素个数）。`[size, capacity-1]` 是未初始化区域，不应访问
- 正确理解：数组合法下标范围是 `[0, size-1]`，边界检查必须用 `da->size`，不是 `da->capacity`
- 纠正任务：区分 size（已用）和 capacity（总容量），边界检查永远用 size
- 是否已复测通过：否

## 2026-05-09：指针已是指针时仍加 &

- 所属技能：单链表
- 错误表现：
```c
struct Node* n2 = createNode(20);  // n2 已经是指针
n1->next = &n2;                    // ← 又加了 &，变成指针的地址
```
- 错误原因：不理解 createNode 返回的就是地址（指针），习惯性加 &
- 正确理解：`n2` 本身就是 `struct Node*`（地址），直接赋值 `n1->next = n2` 即可。`&n2` 是"指针变量 n2 自身的地址"，类型是 `struct Node**`
- 纠正任务：赋值前检查两边类型——`n1->next` 是 `struct Node*`，`n2` 也是 `struct Node*`，类型匹配直接赋值
- 是否已复测通过：是
