## 常见误区样例

### 链表插入时断链

- 所属技能：单链表
- 错误表现：在中间位置插入新节点后，后面的节点丢失
- 错误原因：先将新节点的 next 设为 NULL，再将前驱节点的 next 指向新节点，导致原后继节点丢失
- 正确理解：必须先保存后继节点的地址（`newNode->next = prev->next`），再改变前驱指针（`prev->next = newNode`）
- 纠正任务：画出插入前后链表的内存状态，标注每个指针的赋值顺序
- 是否已复测通过：否

### 快排分区指针方向搞反

- 所属技能：快速排序
- 错误表现：partition 后数组没有被正确分为"小于 pivot"和"大于 pivot"两部分
- 错误原因：j 从右往左找小于 pivot 的元素，但写成了找大于的
- 正确理解：以最右元素为 pivot 时，i 向右找 >= pivot 的，j 向左找 <= pivot 的，然后交换
- 纠正任务：手动模拟 partition([3,1,4,1,5,9,2,6]) 的每一步
- 是否已复测通过：否

### 二分查找边界条件错误

- 所属技能：二分查找
- 错误表现：查找存在元素时返回 -1，或死循环
- 错误原因：使用 `left < right` 但 mid 赋值为 `left` 没有 +1，导致死循环
- 正确理解：当使用 `left < right` 时，mid 需要用 `left + (right - left) / 2`，且更新边界时要 `left = mid + 1` 或 `right = mid`
- 纠正任务：写出两种边界写法（`left <= right` 和 `left < right`），分别适用什么场景
- 是否已复测通过：否

### 递归忘记写终止条件

- 所属技能：递归
- 错误表现：程序运行时栈溢出（Segmentation fault）
- 错误原因：直接写递归步骤，没有先写 base case
- 正确理解：写递归时必须先写终止条件，再写递归步骤。每个递归必须确保向终止条件收敛
- 纠正任务：重写 fibonacci 函数，先写 `if (n <= 1) return n;` 再写递归调用
- 是否已复测通过：否

### 循环队列空满判断混淆

- 所属技能：队列
- 错误表现：入队时误判队列已满，或空队列时出队
- 错误原因：用 `(rear + 1) % capacity == front` 判断满，但没有区分空和满
- 正确理解：循环队列区分空和满有三种常用方法：(1) 浪费一个位置 (2) 增加 size 计数器 (3) 增加 flag 标记
- 纠正任务：实现用 size 计数器区分空和满的循环队列
- 是否已复测通过：否

### malloc 后忘记 free

- 所属技能：链表 / 内存管理
- 错误表现：程序长时间运行后内存占用持续增长
- 错误原因：只关注了功能实现，没有考虑资源释放
- 正确理解：每个 malloc 必须有对应的 free。删除节点时要先 free 再断链，或先保存指针再断链再 free
- 纠正任务：为单链表实现 `void destroyList(ListNode* head)` 函数，释放所有节点
- 是否已复测通过：否
