# 复杂度分析 技能地图

## 目标

学习者能用大 O 表示法分析任意代码段的时间和空间复杂度，能推导递归复杂度，能比较不同算法的效率。

## 必会概念

- 时间复杂度：算法执行基本操作的次数随输入规模的增长趋势
- 空间复杂度：算法占用额外内存随输入规模的增长趋势
- 大 O 表示法：描述渐进上界，忽略常数因子和低阶项
- 最好 / 最坏 / 平均情况
- 递推式与主定理

## 常见复杂度对比

| 复杂度 | 名称 | 示例 |
|--------|------|------|
| O(1) | 常数 | 数组随机访问、哈希表查找 |
| O(log n) | 对数 | 二分查找 |
| O(n) | 线性 | 线性查找、遍历链表 |
| O(n log n) | 线性对数 | 归并排序、快排平均 |
| O(n^2) | 平方 | 冒泡排序、嵌套循环 |
| O(2^n) | 指数 | 递归 Fibonacci |
| O(n!) | 阶乘 | 全排列 |

## 分析技巧

### 循环复杂度

| 结构 | 复杂度 | 说明 |
|------|--------|------|
| 单层循环 `for(i=0;i<n;i++)` | O(n) | 线性遍历 |
| 嵌套循环 `for(i) for(j)` | O(n^2) | 两层独立循环 |
| 内层依赖外层 `for(i) for(j<i)` | O(n^2) | 三角形区域 |
| 倍增循环 `for(i=1;i<n;i*=2)` | O(log n) | 每次翻倍 |
| 倍减循环 `for(i=n;i>0;i/=2)` | O(log n) | 每次减半 |

### 递归复杂度

| 递推式 | 复杂度 | 示例 |
|--------|--------|------|
| T(n) = T(n-1) + O(1) | O(n) | 阶乘 |
| T(n) = 2T(n-1) + O(1) | O(2^n) | Fibonacci（无记忆化） |
| T(n) = 2T(n/2) + O(n) | O(n log n) | 归并排序 |
| T(n) = 2T(n/2) + O(1) | O(n) | 二叉树遍历 |
| T(n) = T(n/2) + O(1) | O(log n) | 二分查找 |

## 训练阶梯

1. **识别**：给出常见复杂度名称，能说出对应的增长曲线
2. **单层循环**：分析只有一个 for 循环的代码
3. **嵌套循环**：分析多层嵌套循环
4. **非标准循环**：分析倍增/倍减循环
5. **简单递归**：分析 T(n) = T(n-1) + O(1) 类型
6. **分治递归**：分析 T(n) = 2T(n/2) + O(n) 类型
7. **空间复杂度**：分析递归调用栈和辅助数组的空间

## 常见错误

- 把 O(n^2) 的嵌套循环误判为 O(n)
- 忽略递归的空间复杂度（调用栈 O(n)）
- 最好情况和最坏情况搞混
- 忘记取对数（倍增循环是 O(log n) 不是 O(n)）
