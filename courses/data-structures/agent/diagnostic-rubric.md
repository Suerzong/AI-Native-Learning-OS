# 诊断量表

每个技能从四个维度诊断。

| 维度 | 诊断问题 | 证据 |
|---|---|---|
| 概念 | 学习者知道这个数据结构解决什么问题吗？ | 能用自己的话解释 |
| 实现 | 学习者能用 C/C++ 写出核心代码吗？ | 能写出可编译的代码 |
| 分析 | 学习者能分析时间/空间复杂度吗？ | 能推导并解释复杂度 |
| 调试 | 学习者知道代码出错时怎么排查吗？ | 能提出检查顺序 |
| 选型 | 学习者知道什么场景用什么数据结构吗？ | 能结合场景选择并解释理由 |

## 诊断输出格式

```md
## 诊断结果

- 当前技能：
- 当前等级：
- 主要弱点：
- 证据：
- 今日训练目标：
- 推荐任务：
```

## 等级判断

| 等级 | 表现 |
|---:|---|
| 0 | 没接触过 |
| 1 | 听过术语，能识别数据结构的图形 |
| 2 | 能跟着示例写出代码并解释每一步 |
| 3 | 能独立手写实现，不看参考 |
| 4 | 能迁移到相近问题（如单链表反转迁移到判断回文链表） |
| 5 | 能分析复杂度、比较替代方案、针对场景选型 |

## 各维度评分细则

### 概念维度
- 0 分：不知道这个数据结构的存在
- 1 分：能画出结构图，但解释不清楚
- 2 分：能解释"是什么"和"为什么需要"
- 3 分：能解释"和类似结构的区别"（如栈 vs 队列、数组 vs 链表）

### 实现维度
- 0 分：写不出来
- 1 分：看着参考能照抄
- 2 分：看着参考能改出来
- 3 分：不看参考能写出来，但可能有小 bug
- 4 分：不看参考能写出来，边界条件处理完整

### 分析维度
- 0 分：不知道什么是复杂度
- 1 分：能背出常见操作的复杂度
- 2 分：能推导简单循环的复杂度
- 3 分：能推导递归的复杂度，能用主定理
- 4 分：能比较不同实现的复杂度差异

### 调试维度
- 0 分：出错不知道怎么办
- 1 分：知道用 printf 调试
- 2 分：能定位到指针错误或内存问题
- 3 分：能系统性地排查（画图 → 打印中间状态 → 边界测试）
