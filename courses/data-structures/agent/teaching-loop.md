# 教学循环

本文件定义 agent 每次训练必须执行的闭环。

## 标准流程

| 步骤 | 目标 | 产出 |
|---|---|---|
| 诊断 | 判断学习者当前状态（含代码能力和复杂度分析能力） | 当前技能、等级、弱点 |
| 定目标 | 锁定一个训练点，每次必须包含至少一个代码实现或复杂度分析 | 今日训练目标 |
| 讲知识点 | 先补齐完成任务所需的最小知识框架，避免学生只靠猜或照抄 | 本次知识卡片 |
| 出任务 | 生成刻意练习，优先出代码实现题和复杂度分析题 | 一个可完成任务 |
| 反馈 | 纠正错误并强化正确理解，代码错误必须定位到行号/变量名 | 反馈和修正建议 |
| 迁移 | 检查是否能换场景应用 | 迁移小题 |
| 更新 | 写入学习状态 | 更新 tracker 和日志 |
| 推进 | 决定下一步 | 推进或补救 |

## 知识点讲解要求

每次出任务前必须给出一张"知识卡片"。知识卡片不是完整教材，而是为当前任务服务的最小讲解。

```md
## 本次知识点

1. 这个数据结构/算法解决什么问题：
2. 核心原理（画图说明）：
3. C/C++ 实现要点：
4. 时间/空间复杂度：
5. 最容易错在哪里：
6. 今天只需要掌握到什么程度：
```

讲解时优先使用"问题场景 -> 数据结构选择 -> 画图理解 -> 代码实现 -> 复杂度分析 -> 变体"的顺序。只有当学习者已经掌握实现后，才展开更底层的优化、变体和选型。

## 会话开场模板

```text
我先看你的当前状态，然后用一个诊断题确定今天练什么。
今天只抓一个核心技能。我会先讲清必要知识点，再让你写代码，最后分析复杂度。
目标是能写、能跑、能分析。
```

## 会话结束模板

```text
今天达成：
今天讲过的知识点：
主要弱点：
已掌握的数据结构/算法：
已掌握的复杂度分析：
下一次最适合练：
是否允许进入下一阶段：
```

## 代码审查检查清单

每次学习者提交代码时，按以下清单检查：

- [ ] 编译无错误和警告
- [ ] 边界情况处理（空输入、单元素、负数等）
- [ ] 指针操作正确（无空指针解引用、无内存泄漏）
- [ ] 复杂度分析正确
- [ ] 代码风格清晰（变量命名、缩进、注释）
- [ ] 测试用例覆盖（正常、边界、异常）
