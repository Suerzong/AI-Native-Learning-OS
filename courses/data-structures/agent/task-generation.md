# 任务生成规则

任务必须服务于当前最小薄弱点。每个任务前必须先给出足够完成任务的知识点讲解，不能直接把学习者丢进题目。

## 任务前知识讲解

生成任务前，先生成"本次知识点"。它必须回答：

- 这个数据结构解决什么问题。
- 它的核心原理是什么（用图解说明）。
- C/C++ 实现的关键代码结构是什么。
- 每个操作的时间和空间复杂度是多少。
- 初学者最容易把哪两个概念混淆。
- 本次任务只要求掌握到什么边界。

如果学习者等级是 0-2，知识讲解必须先用直观语言解释（画图、类比），再给代码。
如果学习者等级是 3-5，可以减少基础解释，但仍要说明本次任务依赖的关键知识和易错点。

## 任务类型

| 类型 | 作用 | 示例 |
|---|---|---|
| 概念题 | 建立直觉 | 解释为什么链表插入是 O(1) |
| 画图题 | 理解内存结构 | 画出单链表 1->2->3 的内存布局 |
| 代码题 | 形成实现能力 | 手写反转单链表 |
| 复杂度题 | 形成分析能力 | 分析二分查找的时间复杂度 |
| 找错题 | 形成调试能力 | 找出链表插入代码中的断链 bug |
| 迁移题 | 检查泛化 | 从栈实现迁移到队列实现 |
| 选型题 | 形成工程能力 | 嵌入式场景下用数组还是链表 |
| 综合题 | 组合技能 | 用栈+链表实现浏览器前进后退 |

## 好任务的格式

```md
## 任务名称

- 本次知识点：
- 目标技能：
- 起始材料：
- 任务要求：
- 限制条件：
- 成功标准：
- 常见错误：
- 复测问题：
```

## 代码题格式

```md
## 任务名称

- 目标技能：
- 语言：C / C++ / 都可以
- 起始代码：无 / 给定部分代码
- 任务要求：
- 函数签名：（给出要求的函数签名）
- 限制条件：（如不能用 STL、必须手动管理内存等）
- 成功标准：（输入输出示例 + 边界情况）
- 常见错误：
- 复测问题：（一个变体题）
```

## 难度递进原则

1. **识别**：能认出数据结构的图形
2. **理解**：能解释原理和复杂度
3. **模仿**：能跟着示例写代码
4. **独立**：能不看参考写出代码
5. **迁移**：能将学到的结构应用到新问题
6. **选型**：能在多个数据结构中选择最合适的

每个技能必须至少经过 1-4 阶段才能推进。
