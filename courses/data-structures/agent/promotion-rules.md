# 晋级规则

只有出现稳定证据时才允许进入下一阶段。

## 通用晋级条件

学习者必须同时满足：

- 能用自己的话解释核心概念和适用场景。
- 能不看参考代码手写实现（允许有小错误，但逻辑框架完整）。
- 能正确分析每个操作的时间和空间复杂度。
- 能完成一个近迁移任务（如学会了链表反转后能判断回文链表）。
- 最近测试正确率不低于 80%。
- 没有未纠正的关键误区。

## 阶段性证据

| 等级 | 晋级证据 |
|---:|---|
| 0 -> 1 | 能识别数据结构的图形表示，知道名称 |
| 1 -> 2 | 能跟着示例写出代码并解释每一步 |
| 2 -> 3 | 能独立手写实现，不参考示例 |
| 3 -> 4 | 能迁移到相近问题并正确求解 |
| 4 -> 5 | 能分析不同实现的优劣，能针对场景选型 |

## 不允许推进的情况

- 学习者只能照抄代码，不能解释为什么。
- 学习者能写出代码，但复杂度分析完全错误。
- 同一误区连续出现两次。
- 没有做迁移测试。
- 指针操作有未纠正的错误（可能导致后续学习中不断踩坑）。

## 阶段晋级特别条件

### 第一层 -> 第二层

必须额外满足：
- 能不看参考实现单链表的增删查改
- 能用栈或队列解决一个实际问题（如括号匹配）
- 能区分数组和链表的时间复杂度差异

### 第二层 -> 第三层

必须额外满足：
- 能手写 BST 的删除操作（三种情况）
- 能解释堆的上浮和下沉操作
- 能实现哈希表的基本操作

### 第三层 -> 第四层

必须额外满足：
- 能手写快排和归并排序
- 能用 DFS/BFS 遍历图
- 能对比 6 种排序算法的复杂度和稳定性
