# 薄弱点追踪

本文件记录学习者的系统性薄弱点。只记录会阻碍后续学习的、反复出现的问题。

## 模板

```md
### 薄弱点标题

- 首次出现：
- 出现次数：
- 影响范围：

```

## 常见薄弱点模式（预设）

以下是从大量数据结构学习者中总结的高频薄弱点，用于快速定位。

### 指针操作不熟练

- 首次出现：待观察
- 出现次数：待观察
- 影响范围：链表、树、图等所有使用指针的数据结构
- 根本原因：C 语言指针练习不足，对内存模型理解不够
- 纠正策略：每次涉及指针操作时，要求画出内存图；从简单链表开始逐步增加难度
- 当前状态：待观察

### 递归思维薄弱

- 首次出现：待观察
- 出现次数：待观察
- 影响范围：树的遍历、归并排序、快排、DFS、分治、DP
- 根本原因：不理解"大问题分解为小问题"的思维模式
- 纠正策略：先用迭代模拟递归过程，再转为递归代码；从阶乘和斐波那契开始
- 当前状态：待观察

### 复杂度分析不系统

- 首次出现：待观察
- 出现次数：待观察
- 影响范围：无法评估算法效率，无法在面试中回答复杂度问题
- 根本原因：只会背结论，不会推导
- 纠正策略：每个代码实现后必须分析复杂度；用具体数字验证
- 当前状态：待观察

### 边界条件处理不完整

- 首次出现：待观察
- 出现次数：待观察
- 影响范围：代码正确性，测试用例通过率
- 根本原因：没有系统性地考虑空输入、单元素、负数等特殊情况
- 纠正策略：写代码前先列出所有边界 case；测试驱动开发
- 当前状态：待观察
