# 薄弱点追踪

## 已出现的薄弱点

### 嵌套循环误判为并列

- 首次出现：2026-05-05
- 出现次数：1
- 影响范围：复杂度分析的准确性
- 根本原因：未区分嵌套结构和并列结构的代码形态差异
- 纠正策略：通过执行流程判断——嵌套是循环体内含循环，而并列是两个独立循环块
- 当前状态：部分解决

### 对数循环识别不足

- 首次出现：2026-05-05
- 出现次数：1
- 影响范围：复杂度分析的准确性
- 根本原因：对 `i /= 2` 等变步长循环的复杂度直觉不足
- 纠正策略：识别"步长不是+1"的循环，理解其等比关系 → O(log n)
- 当前状态：部分解决

### 常数复杂度归一化遗漏

- 首次出现：2026-05-05
- 出现次数：1
- 影响范围：复杂度表达的规范性
- 根本原因：大 O 表示法的归一化规则理解不深
- 纠正策略：强调"所有常数阶都统一写成 O(1)"
- 当前状态：部分解决
