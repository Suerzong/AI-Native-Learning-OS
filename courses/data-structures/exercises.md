## 复杂度-01：分析代码复杂度

- 目标技能：用大 O 表示法分析时间复杂度
- 起始文件：无
- 任务要求：分析以下代码的时间复杂度并解释推导过程
  ```c
  for (int i = 0; i < n; i++)
      for (int j = 0; j < i; j++)
          printf("%d %d\n", i, j);
  ```
- 限制条件：必须写出推导步骤，不能只写答案
- 成功标准：正确得出 O(n^2) 并解释内层循环执行次数为 n(n-1)/2
- 常见错误：忽略内层循环的上界是 i 而不是 n
- 反馈重点：画执行表，逐行数基本操作次数

## 复杂度-02：递归复杂度

- 目标技能：分析递归算法的时间复杂度
- 起始文件：无
- 任务要求：分析以下递归函数的时间复杂度
  ```c
  int fib(int n) {
      if (n <= 1) return n;
      return fib(n-1) + fib(n-2);
  }
  ```
- 限制条件：画出递归调用树或写出递推式
- 成功标准：正确得出 O(2^n)，能解释为什么有大量重复计算
- 常见错误：误认为 O(n) 因为"每次减 1"
- 反馈重点：画出 fib(5) 的递归树，标注重复计算的子问题

## 链表-01：头插法建立链表

- 目标技能：理解单链表的头插法
- 起始文件：无
- 任务要求：用 C 语言实现头插法建立链表，输入数组 [1,2,3,4,5]，输出链表遍历结果
- 限制条件：必须手动 malloc，不能用 C++ new
- 成功标准：输出 5->4->3->2->1->NULL，没有内存泄漏
- 常见错误：
  - 忘记将新节点的 next 设为原头节点
  - 忘记更新 head 指针
  - 没有为节点分配内存
- 反馈重点：画出每一步的链表内存状态

## 链表-02：反转单链表

- 目标技能：理解链表指针操作
- 起始文件：无
- 任务要求：实现 `ListNode* reverseList(ListNode* head)`，用迭代方式
- 限制条件：必须用三个指针（prev, curr, next），不能用额外数组
- 成功标准：输入 1->2->3->NULL，输出 3->2->1->NULL
- 常见错误：
  - 忘记保存 next 导致断链
  - 最后忘记返回 prev
  - 处理空链表时崩溃
- 反馈重点：画出每一步三个指针的位置和链表状态

## 链表-03：检测链表环

- 目标技能：Floyd 判圈算法（快慢指针）
- 起始文件：无
- 任务要求：实现 `bool hasCycle(ListNode* head)`，判断链表是否有环
- 限制条件：空间复杂度 O(1)
- 成功标准：正确判断有环和无环两种情况
- 常见错误：
  - 快指针移动两步时忘记先检查 fast 和 fast->next 是否为 NULL
  - 循环条件写错
- 反馈重点：为什么快指针每次走两步一定能追上慢指针

## 链表-04：合并两个有序链表

- 目标技能：链表的合并操作
- 起始文件：无
- 任务要求：实现 `ListNode* mergeTwoLists(ListNode* l1, ListNode* l2)`，将两个升序链表合并为一个升序链表
- 限制条件：不能创建新节点，只能调整指针
- 成功标准：合并后链表保持升序，不丢失任何节点
- 常见错误：
  - 某个链表遍历完后忘记链接剩余部分
  - 使用了额外的数组空间
- 反馈重点：哨兵节点技巧——用 dummy head 简化边界处理

## 链表-05：删除链表倒数第 N 个节点

- 目标技能：快慢指针 / 双指针技巧
- 起始文件：无
- 任务要求：实现 `ListNode* removeNthFromEnd(ListNode* head, int n)`
- 限制条件：只遍历一次链表
- 成功标准：正确删除倒数第 n 个节点，处理边界情况（删除头节点）
- 常见错误：
  - 遍历两次而不是一次
  - 忘记处理 n 等于链表长度的情况
  - 忘记释放被删除节点的内存
- 反馈重点：让 fast 先走 n 步，然后 fast 和 slow 同步走

## 栈-01：用栈实现括号匹配

- 目标技能：理解栈的 LIFO 特性
- 起始文件：无
- 任务要求：实现 `bool isValid(string s)`，判断括号字符串是否有效（支持 ()[]{}）
- 限制条件：必须用栈实现
- 成功标准：`"()[]{}"` 返回 true，`"([)]"` 返回 false
- 常见错误：
  - 遇到右括号时栈为空没有处理
  - 匹配条件写反（应该是右括号和栈顶匹配）
  - 遍历结束后忘记检查栈是否为空
- 反馈重点：用哈希表存储括号映射关系

## 排序-01：手写快速排序

- 目标技能：理解分区（partition）操作
- 起始文件：无
- 任务要求：实现 `void quickSort(int arr[], int low, int high)`
- 限制条件：必须实现独立的 partition 函数
- 成功标准：能正确排序 [3,1,4,1,5,9,2,6]，并分析最坏情况
- 常见错误：
  - 分区时 i 和 j 的移动方向搞反
  - 递归终止条件写错（应该是 low < high）
  - pivot 选择不当导致最坏情况
- 反馈重点：partition 过程的动画演示

## 排序-02：手写归并排序

- 目标技能：理解分治和合并操作
- 起始文件：无
- 任务要求：实现 `void mergeSort(int arr[], int left, int right)`
- 限制条件：必须实现独立的 merge 函数
- 成功标准：正确排序并说明空间复杂度 O(n) 的原因
- 常见错误：
  - 合并时忘记处理某个子数组已遍历完的情况
  - 临时数组的大小和下标计算错误
- 反馈重点：merge 操作中两个指针的移动逻辑
