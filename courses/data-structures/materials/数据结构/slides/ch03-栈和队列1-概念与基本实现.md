## 第 30 页

栈顶指针：top 用于指示当前栈顶元素的位置。它是一个整数，通常表示数组 data 的索引。
栈空判断：当 top 的值为 -1 时，表示栈为空。这是判断栈是否为空的依据。
栈满判断：当 top 的值等于 MAXSIZE - 1 时，表示栈已满（因为数组的索引是从 0 到 MAXSIZE - 1），无法继续入栈。
操作指引：
入栈（Push）：新元素被放入 data[top + 1]，然后 top 自增（top++）。
出栈（Pop）：返回 data[top]，然后 top 自减（top--）。
获取栈顶元素（Peek）：直接返回 data[top]，不修改 top。

## 第 43 页

1. 指针 vs. 下标
第一种方式（SElemType *base, SElemType *top）：
base 和 top 直接是 指针，指向动态数组的某个位置。
计算栈的长度：top - base（指针运算，更高效）。
访问栈顶元素：*(top - 1)（直接解引用，无需计算偏移量）。
适合动态分配内存的栈（如 malloc 分配的数组）。

第二种方式（unsigned int base, unsigned int top）：
base 和 top 是 数组下标，需要额外存储 SElemType *s 来指向数组。
计算栈的长度：top - base（仍然是整数运算，但不如指针直观）。
访问栈顶元素：s[top - 1]（需要额外解引用 s）。
适用于固定大小的栈（如静态数组 SElemType s[MAXSIZE]）。

---

## 第 44 页

2. 动态内存管理的便利性
指针方式（第一种）：
base 直接指向动态分配的数组首地址，top 直接移动指针即可。
扩容时只需 realloc(base, new_size)，无需修改 base 和 top 的类型。
下标方式（第二种）：
需要额外维护 SElemType *s，扩容时不仅要调整 s，还要保证 base 和 top 仍然是有效的下标。
3. 代码可读性和操作一致性
指针方式：
入栈：*top++ = e;（直接操作指针，简洁高效）。
出栈：e = *(--top);（直接操作指针，无需计算偏移量）。
下标方式：
入栈：s[top++] = e;（需要额外解引用 s）。
出栈：e = s[--top];（同样需要解引用 s）。
如果栈是 固定大小的静态数组（如 SElemType data[MAXSIZE]），可以用前面讲到的 int top 表示栈顶下标

## 第 106 页

回答：
1、为什么顺序存储的队列要用循环方式来合理利用存储空间，而栈则无需考虑用循环方式合理利用存储空间这个问题？
队列：先进先出（FIFO），入队和出队在两端，操作时需要队头、队尾两个指针来维护队列；会存在假溢出的情况。

栈：后进先出（LIFO），入栈、出栈均在栈顶，操作时仅需一个栈顶指针，空间利用率高，不会出现空间碎片
