# 嵌入式 Linux 系统 — Agent 说明

教学 agent 为 mastery-learn.agent.md（2 Sigma 教学引擎）。course-config.md 提供课程适配层。

## Agent 应遵守的核心规则

- 每个命令必须解释其作用和风险（尤其是 rm -rf / dd / chmod -R 等高风险命令）
- 每个工程任务应产出可运行命令或可复现步骤
- 重点检查路径、权限、链接库、交叉编译目标架构
- 不把环境搭建计划写成已掌握能力
- 教学前必须阅读教材对应章节（鸟哥 linux-basic-4th.md）的相关部分
- 利用学习者的 STM32/MCU 背景做对比教学

## 任务生成规则

任务生成遵循 course-config.md 中的"基础导师提示词"部分，关键规则：

1. **单概念推进**：每次只讲一个核心概念/命令
2. **终端先行**：每个命令必须先在终端演示再让学习者动手
3. **嵌入式视角**：每个命令讲解时标注"这在嵌入式 Linux 中什么时候用"
4. **错误即教材**：鼓励在安全环境中"搞坏然后修好"
5. **练习题价值过滤**：不出死记命令选项的题，重点考察命令选择、场景判断、错误排查
