# 自动化机器学习（AutoML）

- 自动化机器学习（AutoML）自动化机器学习工作流中的任务，如特征工程（Feature Engineering）、算法选择和超参数调优（Hyperparameter Tuning），使模型构建更快、更容易。
- 虽然手动训练涉及编写代码并迭代调整，但 AutoML 减少了重复性工作和对专业技能的需求。
- AutoML 使用户能够专注于核心机器学习问题和数据，而不是陷入模型开发周期中的手动任务。
- 本模块探讨使用 AutoML 的优势和局限性、常见模式以及如何将其应用于机器学习项目，要求具备基本机器学习概念的先验知识。

如果你正在开始一个新的机器学习（ML）项目，你可能想知道手动训练是否是构建机器学习模型的唯一选择。使用手动训练时，你使用 ML 框架编写代码来创建模型。在此过程中，你选择要探索的算法，并迭代调整超参数以找到合适的模型。

当然，模型训练并不是你唯一需要考虑的事情。在实践中，从原型到生产构建机器学习模型涉及重复性任务和专业技能。一个简单的探索性 ML 工作流大致如下：

**重复性任务** - ML 工作流可能涉及重复性工作和实验。例如，在模型开发过程中，你通常需要探索不同的算法和超参数组合，以确定最合适的模型。使用手动训练时，你编写专门的代码来训练模型，然后调整代码以使用不同的 ML 算法和超参数运行实验，找到最佳模型。对于小型或探索性项目，这种手动过程可能不是问题，但对于较大的项目，这些重复性任务可能非常耗时。

**专业技能** - 手动开发 ML 模型涉及专业技能。在实践中，并非每个计划开发机器学习模型的团队都具备这些技能。如果团队没有专门的数据科学家，手动完成这项工作甚至可能不可行。

幸运的是，模型开发中的某些步骤可以自动化，以减轻重复性工作和对专业技能的需求。自动化这些任务就是本模块关于自动化机器学习（AutoML）的主题。

## 什么是 AutoML？

[**AutoML**](</machine-learning/glossary#automl>)是自动化机器学习工作流中某些任务的过程。你可以将 AutoML 视为一组工具和技术，使构建机器学习模型更快、更易于更广泛的用户群体使用。虽然自动化可以在整个 ML 工作流中提供帮助，但通常与 AutoML 相关的任务是图 1 中所示的模型开发周期中包含的任务。这些重复性任务包括：

- **数据工程**
  - 特征工程。
  - 特征选择。
- **训练**
  - 确定合适的 ML 算法。
  - 选择最佳超参数。
- **分析**
  - 基于测试和验证数据集评估训练过程中生成的指标。

借助 AutoML，你可以专注于 ML 问题和数据，而不是特征选择、调整超参数和选择正确的算法。
