## 线性分类

在上一节中，我们介绍了图像分类问题，即将单一标签分配给来自固定类别集合的图像。我们还描述了 k 近邻（kNN）分类器，它通过将图像与训练集中的（已标注）图像进行比较来标记图像。正如我们所看到的，kNN 有一些缺点：

  * 分类器必须「记住」所有训练数据并存储以供将来与测试数据比较。这在空间上是低效的，因为数据集很容易达到千兆字节的规模。
  * 对测试图像分类代价高昂，因为它需要与所有训练图像进行比较。

**概述**。我们现在将开发一种更强大的图像分类方法，最终将自然扩展到全神经网络和卷积神经网络。该方法有两个主要组件：一个将原始数据映射到类别得分的**得分函数**，以及一个量化预测得分与真实标签之间一致性的**损失函数**。然后我们将此构建为一个优化问题，在其中最小化损失函数关于得分函数参数的值。

### 从图像到标签得分的参数化映射

该方法的第一个组件是定义将图像像素值映射到每个类别置信度得分的得分函数。我们将用具体例子展开。如前所述，假设有一个训练数据集 \\( x_i \in R^D \\)，每个关联一个标签 \\( y_i \\)。这里 \\( i = 1 \dots N \\)，\\( y_i \in { 1 \dots K } \\)。也就是说，我们有 **N** 个样本（每个维度 **D**）和 **K** 个不同类别。在 CIFAR-10 中，我们有 **N** = 50,000 张图像的训练集，每张 **D** = 32 x 32 x 3 = 3072 像素，**K** = 10，因为有 10 个不同类别。现在我们定义得分函数 \\(f: R^D \mapsto R^K\\)，将原始图像像素映射到类别得分。

**线性分类器。** 在本模块中，我们将从可以说最简单的函数开始——线性映射：

\\[f(x_i, W, b) = W x_i + b\\]

在上述方程中，我们假设图像 \\(x_i\\) 的所有像素已展平为形状 [D x 1] 的列向量。矩阵 **W**（大小 [K x D]）和向量 **b**（大小 [K x 1]）是函数的**参数**。在 CIFAR-10 中，\\(x_i\\) 是 [3072 x 1] 列向量，**W** 是 [10 x 3072]，**b** 是 [10 x 1]，3072 个数字输入（原始像素值），10 个数字输出（类别得分）。**W** 中的参数常称为**权重**（weights），**b** 称为**偏置向量**（bias vector），因为它影响输出得分但不与实际数据 \\(x_i\\) 交互。不过，你经常会听到人们互换使用「权重」和「参数」两个术语。

有几点需要注意：

  * 首先，单次矩阵乘法 \\(W x_i\\) 实际上并行评估了 10 个独立的分类器（每个类一个），每个分类器是 **W** 的一行。
  * 注意我们将输入数据 \\( (x_i, y_i) \\) 视为给定且固定的，但我们对参数 **W,b** 的设置有控制权。我们的目标是以计算得分与整个训练集的真实标签匹配的方式设置这些参数。直觉上，我们希望正确类别的得分高于不正确类别的得分。
  * 这种方法的一个优点是训练数据用于学习参数 **W,b**，但学习完成后我们可以丢弃整个训练集，只保留学习到的参数。因为新测试图像可以简单地通过函数前向传播，根据计算出的得分分类。
  * 最后，注意对测试图像分类只需一次矩阵乘法和加法，这比将测试图像与所有训练图像比较要快得多。

> 预告：卷积神经网络将完全如上所示将图像像素映射到得分，但映射 f 将更复杂并包含更多参数。

### 线性分类器的解释

注意线性分类器计算类别的得分为其所有像素值在所有 3 个颜色通道上的加权和。根据我们设置的权重值，函数有能力「喜欢」或「不喜欢」图像中某些位置的某些颜色（取决于每个权重的符号）。例如，你可以想象如果图像两侧有很多蓝色（可能对应水面），则「船」类别可能更有可能。你可能期望「船」分类器在其蓝色通道权重上有大量正值（蓝色的存在增加船的得分），在红色/绿色通道上有负值（红/绿的存在降低船的得分）。

![](/assets/imagemap.jpg)

图像到类别得分的映射示例。为了可视化，我们假设图像只有 4 个像素（4 个灰度像素，本例不考虑颜色通道以简洁），有 3 个类别（红色（猫）、绿色（狗）、蓝色（船）类）。我们将图像像素展平为一列，执行矩阵乘法得到每个类别的得分。注意这组特定的权重 W 完全不好：权重给猫图像的猫得分非常低。这组权重似乎确信它看到的是一只狗。

**图像作为高维点的类比。** 由于图像被展平为高维列向量，我们可以将每张图像解释为空间中的一个点（例如 CIFAR-10 中的每张图像是 3072 维空间中的一个点）。类似地，整个数据集是一组（标记的）点。

由于我们将每个类别的得分定义为所有图像像素的加权和，每个类别得分是此空间上的线性函数。我们无法可视化 3072 维空间，但如果想象将所有维度压缩到只有两个维度，那么我们可以尝试可视化分类器的行为：

![](/assets/pixelspace.jpeg)

图像空间的卡通表示，其中每张图像是一个点，可视化了三个分类器。以汽车分类器（红色）为例，红线显示空间中所有得分为零的点。红箭头显示增加方向，所以红线右侧的所有点有正（线性增加）得分，左侧所有点有负（线性减少）得分。

如上所述，\\(W\\) 的每一行是一个类别的分类器。这些数字的几何解释是，当我们改变 \\(W\\) 的一行时，像素空间中对应的线将向不同方向旋转。偏置 \\(b\\) 则允许我们的分类器平移这些线。特别要注意，如果没有偏置项，将 \\( x_i = 0 \\) 代入总是给出零分，无论权重如何，所以所有线都被迫通过原点。

**线性分类器作为模板匹配的解释。** 权重 \\(W\\) 的另一种解释是，\\(W\\) 的每一行对应一个类别的「模板」（有时也称为「原型」）。每个类别对图像的得分然后通过逐个将每个模板与图像使用「内积」（或「点积」）比较来获得，找到「最匹配」的那个。用这种术语，线性分类器在做模板匹配，其中模板是学习得到的。另一种理解方式是我们仍在有效地做最近邻，但不是使用数千张训练图像，而是每个类别只使用一张图像（虽然我们会学习它，而且它不一定是训练集中的图像），并且我们使用（负的）内积作为距离，而非 L1 或 L2 距离。

![](/assets/templates.jpg)

跳过一点：CIFAR-10 学习结束时的示例学习到的权重。注意，例如船模板包含许多蓝色像素（如预期）。因此，当使用内积与海洋上的船只图像匹配时，此模板将给出高分。

此外，注意马模板似乎包含一匹双头马，这是因为数据集中同时有朝左和朝右的马。线性分类器将数据中马的这两种模式「合并」为一个模板。类似地，汽车分类器似乎将多种模式合并为一个模板，必须从所有方向和所有颜色识别汽车。特别是，这个模板最终是红色的，这暗示 CIFAR-10 数据集中红色汽车比其他任何颜色都多。线性分类器太弱，无法正确处理不同颜色的汽车，但正如我们稍后将看到的，神经网络将允许我们执行此任务。展望一下，神经网络将能够在隐藏层中开发中间神经元来检测特定类型的汽车（例如朝左的绿色汽车、朝前的蓝色汽车等），下一层的神经元可以通过各个汽车检测器的加权和将这些组合成更准确的汽车得分。

**偏置技巧。** 在继续之前，我们想提到一个常见的简化技巧，将两个参数 \\(W,b\\) 表示为一个。回顾我们定义的得分函数：

\\[f(x_i, W, b) = W x_i + b\\]

在后续学习中，分别跟踪两组参数（偏置 \\(b\\) 和权重 \\(W\\)）有些繁琐。一个常用技巧是将两组参数合并到一个矩阵中，通过在向量 \\(x_i\\) 中额外增加一个维度，该维度始终持有常数 \\(1\\)——一个默认的「偏置维度」。有了额外维度，新的得分函数将简化为单次矩阵乘法：

\\[f(x_i, W) = W x_i\\]

在 CIFAR-10 例子中，\\(x_i\\) 现在是 [3073 x 1] 而不是 [3072 x 1]——（额外维度持有常数 1），\\(W\\) 现在是 [10 x 3073] 而不是 [10 x 3072]。\\(W\\) 现在的额外列对应偏置 \\(b\\)。

![](/assets/wb.jpeg)

偏置技巧图示。执行矩阵乘法然后加偏置向量（左）等价于给所有输入向量添加值为 1 的偏置维度并将权重矩阵扩展一列——偏置列（右）。因此，如果我们通过在所有向量末尾追加 1 来预处理数据，我们只需要学习一个权重矩阵，而不是分别的权重和偏置矩阵。

**图像数据预处理。** 快速提醒，在上述例子中我们使用了原始像素值（范围 [0...255]）。在机器学习中，对输入特征进行归一化是非常常见的做法（在图像的情况下，每个像素被视为一个特征）。特别是，通过从每个特征中减去均值来**对数据进行中心化**非常重要。在图像的情况下，这对应于计算训练图像的「均值图像」并从每张图像中减去它，得到像素范围约 [-127 ... 127] 的图像。进一步常见的预处理是将每个输入特征缩放到 [-1, 1] 范围。其中，零均值中心化可以说更重要，但我们要等到理解梯度下降的动态后才能解释原因。

### 损失函数

在上一节中，我们定义了一个从像素值到类别得分的函数，由一组权重 \\(W\\) 参数化。此外，我们看到我们对数据 \\( (x_i,y_i) \\) 没有控制权（它是固定给定的），但我们确实有权控制这些权重，并且我们希望设置它们使得预测的类别得分与训练数据中的真实标签一致。

例如，回到猫的示例图像及其对类别「猫」、「狗」、「船」的得分，我们看到那组特定的权重完全不好：我们输入了描绘猫的像素，但猫得分非常低（-96.8），而其他类别（狗得分 437.9，船得分 61.95）则很高。我们将用一个**损失函数**（有时也称为**代价函数**或**目标函数**）来衡量我们对此类结果的不满。直观上，如果我们对训练数据的分类效果差，损失就高；效果好，损失就低。

#### 多类别支持向量机损失

定义损失函数细节有几种方式。第一个例子我们将开发一种常用的损失——**多类别支持向量机**（SVM）损失。SVM 损失的设置使得 SVM「希望」每个图像的正确类别的得分比不正确类别高出某个固定间隔 \\(\Delta\\)。注意，有时将损失函数拟人化是有帮助的：SVM「希望」某种结果，因为该结果会产生更低的损失（这是好事）。

让我们更精确一些。回顾对于第 i 个样本，我们给定图像 \\( x_i \\) 的像素和标签 \\( y_i \\)，指定正确类别的索引。得分函数接收像素并计算类别得分向量 \\( f(x_i, W) \\)，我们简写为 \\(s\\)。例如，第 j 个类别的得分是第 j 个元素：\\( s_j = f(x_i, W)_j \\)。第 i 个样本的多类别 SVM 损失形式化如下：

\\[L_i = \sum_{j\neq y_i} \max(0, s_j - s_{y_i} + \Delta)\\]

**示例。** 让我们用例子来理解它的工作方式。假设我们有三个类别，得分 \\( s = [13, -7, 11]\\)，第一个类别是真实类别（即 \\(y_i = 0\\)）。同时假设 \\(\Delta\\)（一个超参数）为 10。上面的表达式对所有不正确类别（\\(j \neq y_i\\)）求和，所以我们得到两项：

\\[L_i = \max(0, -7 - 13 + 10) + \max(0, 11 - 13 + 10)\\]

你可以看到第一项给零，因为 [-7 - 13 + 10] 给出负数，然后被 \\(max(0,-)\\) 函数阈值化为零。这对没有损失，因为正确类别得分（13）比不正确类别得分（-7）至少高出间隔 10。实际上差距是 20，远大于 10，但 SVM 只关心差距至少为 10；超过间隔的任何额外差异都被 max 操作钳制为零。第二项计算 [11 - 13 + 10] 得到 8。也就是说，即使正确类别得分高于不正确类别（13 > 11），但没有高出所需的间隔 10。差距只有 2，因此损失为 8（即差距需要提高多少才能满足间隔）。总结来说，SVM 损失函数希望正确类别 \\(y_i\\) 的得分比不正确类别得分至少高出 \\(\Delta\\)。如果不是这样，我们将累积损失。

注意在此特定模块中，我们使用线性得分函数（ \\( f(x_i; W) = W x_i \\) ），因此我们也可以用以下等价形式重写损失函数：

\\[L_i = \sum_{j\neq y_i} \max(0, w_j^T x_i - w_{y_i}^T x_i + \Delta)\\]

其中 \\(w_j\\) 是 \\(W\\) 的第 j 行变形为列向量。但一旦我们开始考虑更复杂形式的得分函数 \\(f\\)，情况就不一定如此了。

我们在此节结束前要提到的最后一个术语是，在零处截断的 \\(max(0,-)\\) 函数常被称为**合页损失**（hinge loss）。你有时会听到人们使用平方合页损失 SVM（或 L2-SVM），其形式为 \\(max(0,-)^2\\)，对违反间隔的惩罚更强烈（二次而非线性）。非平方版本更标准，但在某些数据集上平方合页损失可能效果更好。这可以在交叉验证中确定。

> 损失函数量化了我们对训练集预测的不满

![](/assets/margin.jpg)

多类别支持向量机「希望」正确类别的得分至少高出所有其他得分一个间隔 delta。如果任何类别的得分在红色区域内（或更高），则会累积损失。否则损失为零。我们的目标是找到权重，同时对训练数据中的所有样本满足此约束，并给出尽可能低的总损失。

**正则化。** 我们上面提出的损失函数有一个缺陷。假设我们有一个数据集和一组参数 **W**，能正确分类每个样本（即所有得分都满足间隔，\\(L_i = 0\\) 对所有 i）。问题是这组 **W** 不一定是唯一的：可能存在许多相似的 **W** 都能正确分类样本。一个简单的方法是，如果某些参数 **W** 能正确分类所有样本（每个样本损失为零），那么这些参数的任何倍数 \\( \lambda W \\)（其中 \\( \lambda > 1 \\)）也会给出零损失，因为此变换均匀拉伸所有得分大小，从而也拉伸了它们的绝对差异。

换句话说，我们希望编码对某组权重 **W** 的偏好以消除这种模糊性。我们可以通过在损失函数中添加**正则化惩罚** \\(R(W)\\) 来实现。最常见的正则化惩罚是平方 **L2** 范数，通过在所有参数上逐元素二次惩罚来阻止大权重：

\\[R(W) = \sum_k\sum_l W_{k,l}^2\\]

在上面的表达式中，我们对 \\(W\\) 的所有平方元素求和。注意正则化函数不是数据的函数，它仅基于权重。包含正则化惩罚构成了完整的多类别支持向量机损失，由两部分组成：**数据损失**（所有样本的平均损失 \\(L_i\\)）和**正则化损失**。完整的多类别 SVM 损失变为：

\\[L = \underbrace{ \frac{1}{N} \sum_i L_i }_\text{数据损失} + \underbrace{ \lambda R(W) }_\text{正则化损失} \\\\\\\\\\]

完整展开形式：

\\[L = \frac{1}{N} \sum_i \sum_{j\neq y_i} \left[ \max(0, f(x_i; W)_j - f(x_i; W)_{y_i} + \Delta) \right] + \lambda \sum_k\sum_l W_{k,l}^2\\]

其中 \\(N\\) 是训练样本数。如你所见，我们将正则化惩罚附加到损失目标上，由超参数 \\(\lambda\\) 加权。设置此超参数没有简单方法，通常通过交叉验证确定。

除了我们上面提供的动机外，包含正则化惩罚还有许多理想属性，我们将在后面的章节中回到其中一些。例如，包含 L2 惩罚导致 SVM 中吸引人的**最大间隔**属性（如果你感兴趣，请参阅 [CS229](http://cs229.stanford.edu/notes/cs229-notes3.pdf) 讲义获取完整细节）。

最有吸引力的属性是惩罚大权重倾向于改善泛化，因为它意味着没有输入维度可以独自对得分产生非常大的影响。例如，假设我们有输入向量 \\(x = [1,1,1,1] \\) 和两个权重向量 \\(w_1 = [1,0,0,0]\\)、\\(w_2 = [0.25,0.25,0.25,0.25] \\)。那么 \\(w_1^Tx = w_2^Tx = 1\\)，所以两个权重向量产生相同的点积，但 \\(w_1\\) 的 L2 惩罚是 1.0，而 \\(w_2\\) 的 L2 惩罚只有 0.25。因此，根据 L2 惩罚，权重向量 \\(w_2\\) 会更受青睐，因为它实现了更低的正则化损失。直观上，这是因为 \\(w_2\\) 中的权重更小、更分散。由于 L2 惩罚偏好更小、更分散的权重向量，最终分类器被鼓励少量地考虑所有输入维度，而不是非常强烈地考虑少数输入维度。正如我们将在课程后面看到的，这可以改善分类器在测试图像上的泛化性能，减少「过拟合」。

注意偏置没有同样的效果，因为与权重不同，它们不控制输入维度的影响强度。因此，通常只正则化权重 \\(W\\) 而不正则化偏置 \\(b\\)。但在实践中，这往往影响微乎其微。最后，注意由于正则化惩罚，我们永远不能在所有样本上达到正好 0.0 的损失，因为这只有在 \\(W = 0\\) 的病态情况下才可能。

**代码。** 以下是损失函数（不含正则化）的 Python 实现，包含未向量化和半向量化形式：


    def L_i(x, y, W):
      """
      未向量化版本。计算单个样本 (x,y) 的多类别 svm 损失
      - x 是表示图像的列向量（例如 CIFAR-10 中为 3073 x 1）
        第 3073 个位置附加了偏置维度（即偏置技巧）
      - y 是正确类别索引的整数（例如 CIFAR-10 中在 0 到 9 之间）
      - W 是权重矩阵（例如 CIFAR-10 中为 10 x 3073）
      """
      delta = 1.0 # 关于 delta 的说明见本节后面
      scores = W.dot(x) # scores 变为大小 10 x 1，每个类别的得分
      correct_class_score = scores[y]
      D = W.shape[0] # 类别数，例如 10
      loss_i = 0.0
      for j in range(D): # 遍历所有错误类别
        if j == y:
          # 跳过真实类别，只遍历不正确类别
          continue
        # 累积第 i 个样本的损失
        loss_i += max(0, scores[j] - correct_class_score + delta)

### 实际考量

**设置 Delta。** 注意我们略过了超参数 \\(\Delta\\) 及其设置。它应该设为什么值，我们需要交叉验证它吗？事实证明这个超参数可以安全地设为 \\(\Delta = 1.0\\)。超参数 \\(\Delta\\) 和 \\(\lambda\\) 看起来像是两个不同的超参数，但实际上它们都控制着同一个权衡：目标中数据损失和正则化损失之间的权衡。理解这一点的关键是权重 \\(W\\) 的大小直接影响得分（从而也影响它们的差异）：当缩小 \\(W\\) 中的所有值时，得分差异会变小；当放大权重时，得分差异会变大。因此，得分之间间隔的确切值（例如 \\(\Delta = 1\\) 或 \\(\Delta = 100\\)）在某种意义上是无意义的，因为权重可以任意缩小或拉伸差异。因此，唯一真正的权衡是我们允许权重增长多大（通过正则化强度 \\(\lambda\\)）。

**与二元支持向量机的关系。** 你可能带着二元支持向量机的经验来上这门课，其中第 i 个样本的损失可以写为：

\\[L_i = C \max(0, 1 - y_i w^Tx_i) + R(W)\\]

其中 \\(C\\) 是超参数，\\(y_i \in \\{ -1,1 \\} \\)。你可以说服自己，我们在本节提出的公式包含二元 SVM 作为只有两个类别时的特例。另外，此公式中的 \\(C\\) 和我们公式中的 \\(\\lambda\\) 控制着相同的权衡，通过倒数关系 \\(C \propto \frac{1}{\\lambda}\\) 相关联。

**附注：原始形式的优化。** 如果你带着 SVM 的先验知识来上课，你可能还听说过核、对偶、SMO 算法等。在本课程中（正如神经网络一般的情况），我们始终以无约束原始形式处理优化目标。许多这些目标在技术上是不可微的（例如 max(x,y) 函数在 x=y 时有「折点」），但在实践中这不是问题，使用次梯度是常见的。

**附注：其他多类别 SVM 公式。** 值得注意的是，本节介绍的多类别 SVM 是制定多类别 SVM 的少数方式之一。另一种常用形式是「一对多」（OVA）SVM，为每个类别对所有其他类别训练独立的二元 SVM。相关但在实践中较少见的是「全对全」（AVA）策略。我们的公式遵循 [Weston and Watkins 1999 (pdf)](https://www.elen.ucl.ac.be/Proceedings/esann/esannpdf/es1999-461.pdf) 版本，比 OVA 更强大（你可以构造多类别数据集，此版本可以实现零数据损失，但 OVA 不能）。你可能看到的最后一个公式是「结构化 SVM」，它最大化正确类别得分与最高得分的错误亚军类别得分之间的间隔。理解这些公式之间的差异超出了本课程范围。

### Softmax 分类器

SVM 是两种常见分类器之一。另一种流行选择是 **Softmax 分类器**，它有不同的损失函数。如果你听说过二元逻辑回归分类器，Softmax 分类器是其向多类别的推广。与 SVM 将输出 \\(f(x_i,W)\\) 视为（未校准且可能难以解释的）类别得分不同，Softmax 分类器给出更直观的输出（归一化的类别概率），也有我们稍后描述的概率解释。在 Softmax 分类器中，函数映射 \\(f(x_i; W) = W x_i\\) 保持不变，但我们现在将这些得分解释为每个类别的未归一化对数概率，并用**交叉熵损失**替换「合页损失」：

\\[L_i = -\log\left(\frac{e^{f_{y_i}}}{ \sum_j e^{f_j} }\right) \hspace{0.5in} \text{或等价地} \hspace{0.5in} L_i = -f_{y_i} + \log\sum_j e^{f_j}\\]

其中我们使用符号 \\(f_j\\) 表示类别得分向量 \\(f\\) 的第 j 个元素。如前所述，数据集的完整损失是所有训练样本上 \\(L_i\\) 的均值加上正则化项 \\(R(W)\\)。函数 \\(f_j(z) = \frac{e^{z_j}}{\sum_k e^{z_k}} \\) 称为 **softmax 函数**：它接收任意实值得分向量（在 \\(z\\) 中），将其压缩为零到一之间的值向量，总和为一。包含 softmax 函数的完整交叉熵损失第一次看到可能很吓人，但相对容易理解。

**信息论视角。**「真实」分布 \\(p\\) 和估计分布 \\(q\\) 之间的「交叉熵」定义为：

\\[H(p,q) = - \sum_x p(x) \log q(x)\\]

Softmax 分类器因此最小化估计类别概率（ \\(q = e^{f_{y_i}} / \sum_j e^{f_j} \\) ）与「真实」分布之间的交叉熵，在此解释中真实分布是所有概率质量都在正确类别上的分布（即 \\(p = [0, \ldots 1, \ldots, 0]\\) 在 \\(y_i\\) 位置包含单个 1）。此外，由于交叉熵可以用熵和 KL 散度表示为 \\(H(p,q) = H(p) + D_{KL}(p||q)\\)，而 delta 函数 \\(p\\) 的熵为零，这等价于最小化两个分布之间的 KL 散度（一种距离度量）。换句话说，交叉熵目标「希望」预测分布将其所有概率质量放在正确答案上。

**概率解释。** 看这个表达式，我们看到

\\[P(y_i \mid x_i; W) = \frac{e^{f_{y_i}}}{\sum_j e^{f_j} }\\]

可以解释为给定图像 \\(x_i\\) 并由 \\(W\\) 参数化时，分配给正确标签 \\(y_i\\) 的（归一化）概率。Softmax 分类器将输出向量 \\(f\\) 中的得分解释为未归一化的对数概率。对这些值取指数因此给出（未归一化的）概率，除法执行归一化使概率总和为一。在概率解释中，我们因此最小化正确类别的负对数似然，可以解释为执行「最大似然估计」（MLE）。这个视角的一个好处是我们还可以将完整损失函数中的正则化项 \\(R(W)\\) 解释为来自权重矩阵 \\(W\\) 上的高斯先验，其中我们不是执行 MLE 而是执行「最大后验」（MAP）估计。我们提到这些解释来帮助你的直觉，但此推导的完整细节超出了本课程范围。

**实际问题：数值稳定性。** 在实践中编写计算 Softmax 函数的代码时，中间项 \\(e^{f_{y_i}}\\) 和 \\(\sum_j e^{f_j}\\) 可能因指数运算而非常大。大数相除可能在数值上不稳定，因此使用归一化技巧很重要。注意如果我们用常数 \\(C\\) 乘以分数的分子和分母并将其推入求和，我们得到以下（数学等价的）表达式：

\\[\frac{e^{f_{y_i}}}{\sum_j e^{f_j}} = \frac{Ce^{f_{y_i}}}{C\sum_j e^{f_j}} = \frac{e^{f_{y_i} + \log C}}{\sum_j e^{f_j + \log C}}\\]

我们可以自由选择 \\(C\\) 的值。这不会改变任何结果，但我们可以使用此值来改善计算的数值稳定性。\\(C\\) 的常见选择是设 \\(\log C = -\max_j f_j \\)。这简单地表示我们应该将向量 \\(f\\) 内部的值平移，使最大值为零。代码中：


    f = np.array([123, 456, 789]) # 3 个类别的示例，每个都有大得分
    p = np.exp(f) / np.sum(np.exp(f)) # 不好：数值问题，可能溢出

**可能令人困惑的命名约定。** 精确地说，「SVM 分类器」使用「合页损失」，有时也称为「最大间隔损失」。「Softmax 分类器」使用「交叉熵损失」。Softmax 分类器的名字来自「softmax 函数」，用于将原始类别得分压缩为归一化的正值，总和为一，以便应用交叉熵损失。特别要注意，技术上谈论「softmax 损失」没有意义，因为 softmax 只是压缩函数，但这是相对常用的简写。

### SVM vs. Softmax

一张图可能有助于澄清 Softmax 和 SVM 分类器之间的区别：

![](/assets/svmvssoftmax.png)

SVM 和 Softmax 分类器对一个数据点的差异示例。两种情况下我们都计算相同的得分向量 **f**（例如通过本节的矩阵乘法）。区别在于对 **f** 中得分的解释：SVM 将这些视为类别得分，其损失函数鼓励正确类别（第 2 类，蓝色）的得分比其他类别得分高出一个间隔。Softmax 分类器则将得分解释为每个类别的（未归一化）对数概率，然后鼓励正确类别的（归一化）对数概率高（等价地其负值低）。此示例的最终损失 SVM 为 1.58，Softmax 为 1.04（注意这是使用自然对数的 1.04，不是以 2 或 10 为底），但这些数字不可比较；它们只在同一分类器和相同数据内计算的损失中有意义。

**Softmax 分类器为每个类别提供「概率」。** 与 SVM 计算未校准且不易解释的所有类别得分不同，Softmax 分类器允许我们计算所有标签的「概率」。例如，给定一张图像，SVM 分类器可能给你类别「猫」、「狗」、「船」的得分 [12.5, 0.6, -23.0]。Softmax 分类器则可以计算三个标签的概率为 [0.9, 0.09, 0.01]，让你解释其对每个类别的置信度。然而，我们将「概率」加引号的原因是这些概率的峰值或分散程度直接取决于正则化强度 \\(\lambda\\)——这是你作为系统输入来控制的。例如，假设某个三个类别的未归一化对数概率为 [1, -2, 0]。softmax 函数将计算：

\\[[1, -2, 0] \rightarrow [e^1, e^{-2}, e^0] = [2.71, 0.14, 1] \rightarrow [0.7, 0.04, 0.26]\\]

其中步骤是取指数并归一化到总和为一。现在，如果正则化强度 \\(\lambda\\) 更高，权重 \\(W\\) 会被更多惩罚，导致更小的权重。例如，假设权重变为原来的一半（[0.5, -1, 0]）。softmax 将计算：

\\[[0.5, -1, 0] \rightarrow [e^{0.5}, e^{-1}, e^0] = [1.65, 0.37, 1] \rightarrow [0.55, 0.12, 0.33]\\]

概率现在更分散了。此外，在权重因非常强的正则化强度 \\(\lambda\\) 而趋向极小值的极限下，输出概率将近乎均匀。因此，Softmax 分类器计算的概率最好被理解为置信度，其中类似于 SVM，得分的顺序是可解释的，但绝对数字（或其差异）在技术上不是。

**在实践中，SVM 和 Softmax 通常是可比的。** SVM 和 Softmax 之间的性能差异通常很小，不同的人对哪个分类器更好有不同的看法。与 Softmax 分类器相比，SVM 是一个更「局部」的目标，这可以被视为缺陷或特性。考虑一个得分为 [10, -2, 3] 的例子，其中第一个类别是正确的。SVM（例如期望间隔 \\(\Delta = 1\\)）会看到正确类别的得分已经比其他类别高出间隔以上，将计算零损失。SVM 不关心各个得分的细节：如果得分是 [10, -100, -100] 或 [10, 9, 9]，SVM 都无所谓，因为间隔 1 已满足，损失为零。然而，这些场景对 Softmax 分类器并不等价，它对得分 [10, 9, 9] 会累积比 [10, -100, -100] 高得多的损失。换句话说，Softmax 分类器永远不会对其产生的得分完全满意：正确类别总可以有更高概率，错误类别总可以有更低概率，损失总会更好。但 SVM 一旦间隔满足就满意了，不会在此约束之外微调确切的得分。这可以直观地理解为一种特性：例如，汽车分类器可能将其大部分「精力」花在区分汽车和卡车这个困难问题上，不应受青蛙样本的影响，它已经给青蛙分配了很低的得分，青蛙可能聚集在数据云的完全不同的侧面。

### 总结

总结：

  * 我们定义了从图像像素到类别得分的**得分函数**（本节中是依赖于权重 **W** 和偏置 **b** 的线性函数）。
  * 与 kNN 分类器不同，这种**参数化方法**的优点是一旦我们学习了参数就可以丢弃训练数据。此外，对新测试图像的预测很快，因为它只需与 **W** 进行一次矩阵乘法，而不需要与每个训练样本进行详尽比较。
  * 我们介绍了**偏置技巧**，允许我们将偏置向量折叠到权重矩阵中，方便只需跟踪一个参数矩阵。
  * 我们定义了一个**损失函数**（我们介绍了两种线性分类器常用损失：**SVM** 和 **Softmax**），衡量给定参数集与训练数据集中真实标签的兼容性。我们还看到损失函数的定义使得在训练数据上做出好的预测等价于拥有小的损失。

我们现在看到了一种将图像数据集基于一组参数映射到类别得分的方法，并看到了两个损失函数示例，我们可以用来衡量预测质量。但我们如何高效地确定给出最佳（最低）损失的参数？这个过程就是「优化」，它是下一节的主题。
