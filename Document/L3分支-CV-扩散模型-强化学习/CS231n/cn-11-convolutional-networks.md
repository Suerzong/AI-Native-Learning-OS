## 卷积神经网络（CNN / ConvNet）

卷积神经网络与前一章的普通神经网络非常相似：由具有可学习权重和偏置的神经元组成。整个网络仍然表达一个单一的可微得分函数：从一端的原始图像像素到另一端的类别得分。

ConvNet 架构显式假设输入是图像，这允许我们将某些属性编码到架构中。

### 架构概述

ConvNet 的层将神经元排列在三个维度上：**宽度、高度、深度**。

![](/assets/nn1/neural_net2.jpeg) ![](/assets/cnn/cnn.jpeg)

左图：常规 3 层神经网络。右图：ConvNet 将神经元排列在三个维度中。

> ConvNet 由层组成。每层有一个简单 API：将输入 3D 体变换为输出 3D 体，使用某个可能有参数的可微函数。

### 用于构建 ConvNet 的层

我们使用三种主要类型的层构建 ConvNet 架构：**卷积层**、**池化层**和**全连接层**。

#### 卷积层

卷积层是卷积网络的核心构建模块。

**概述。** 卷积层的参数由一组可学习的滤波器组成。每个滤波器在空间上很小（沿宽度和高度），但延伸到输入体的整个深度。在前向传播中，我们在输入体的宽度和高度上滑动（更准确地说，卷积）每个滤波器，计算滤波器条目和输入之间的点积。这产生二维激活图。

**局部连接性。** 每个神经元只连接到输入体的局部区域。此连接的空间范围称为神经元的**感受野**（等价于滤波器大小）。沿深度轴的连接范围始终等于输入体的深度。

**空间排列。** 三个超参数控制输出体的大小：**深度**、**步幅**（stride）和**零填充**（zero-padding）。

  * **深度：** 输出体的深度是超参数，对应我们想要使用的滤波器数量。
  * **步幅：** 滑动滤波器的步幅。步幅为 1 时每次移动一个像素，步幅为 2 时每次跳过 2 个像素。
  * **零填充：** 在边界周围用零填充输入体。当步幅 \\(S = 1\\) 时，设零填充 \\(P = (F - 1)/2\\) 确保输入和输出空间大小相同。

输出大小公式：\\((W - F + 2P)/S + 1\\)

![](/assets/cnn/stride.jpeg)

**参数共享。** 约束每个深度切片中的神经元使用相同的权重和偏置。这大幅减少了参数数量。例如 55x55x96 的输出体如果不用参数共享需要 1 亿多参数，使用参数共享只需 34,944 个参数。

![](/assets/cnn/weights.jpeg)

**总结：**
  * 接受大小为 \\(W_1 \times H_1 \times D_1\\) 的体
  * 需要四个超参数：滤波器数量 \\(K\\)、空间范围 \\(F\\)、步幅 \\(S\\)、零填充 \\(P\\)
  * 产生大小为 \\(W_2 \times H_2 \times D_2\\) 的体：\\(W_2 = (W_1 - F + 2P)/S + 1\\)，\\(D_2 = K\\)

**1x1 卷积。** 1x1 卷积对输入深度执行 3 维点积。

**扩张卷积。** 在滤波器单元之间引入间隔的卷积。

#### 池化层

周期性地在连续卷积层之间插入池化层。其功能是逐步减小表示的空间大小，减少网络中的参数和计算量。最常见的是 2x2 滤波器、步幅 2 的最大池化，丢弃 75% 的激活。

![](/assets/cnn/pool.jpeg) ![](/assets/cnn/maxpool.jpeg)

  * 接受大小为 \\(W_1 \times H_1 \times D_1\\) 的体
  * 需要两个超参数：空间范围 \\(F\\)、步幅 \\(S\\)
  * 产生大小为 \\(W_2 \times H_2 \times D_2\\) 的体：\\(W_2 = (W_1 - F)/S + 1\\)，\\(D_2 = D_1\\)
  * 不引入参数

#### 归一化层

许多类型的归一化层已被提出，但实践中贡献被证明很小（如果有的话）。

#### 全连接层

神经元与前一层的所有激活完全连接，如常规神经网络中所见。

#### 将全连接层转换为卷积层

全连接层和卷积层之间的唯一区别是卷积层中的神经元只连接到输入的局部区域。但功能形式相同，因此可以相互转换。

全连接到卷积的转换在实践中特别有用：允许我们在更大图像的多个空间位置高效地「滑动」原始 ConvNet。

### ConvNet 架构

#### 层模式

最常见的 ConvNet 架构模式：

`INPUT -> [[CONV -> RELU]*N -> POOL?]*M -> [FC -> RELU]*K -> FC`

  * `INPUT -> FC`：线性分类器
  * `INPUT -> CONV -> RELU -> FC`
  * `INPUT -> [CONV -> RELU -> POOL]*2 -> FC -> RELU -> FC`
  * `INPUT -> [CONV -> RELU -> CONV -> RELU -> POOL]*3 -> [FC -> RELU]*2 -> FC`

**偏好使用小滤波器的 CONV 堆叠，而非一个大感受野的 CONV 层。** 三个堆叠的 3x3 CONV 层（中间有非线性）与单个 7x7 CONV 层有相同的感受野，但参数更少（27C² vs 49C²），表达能力更强。

**实践：使用 ImageNet 上效果最好的架构。** 不要自己设计架构，应该查看 ImageNet 上当前最好的架构，下载预训练模型并在你的数据上微调。

#### 案例研究

  * **LeNet。** Yann LeCun 在 1990 年代开发的第一个成功的卷积网络应用。
  * **AlexNet。** Alex Krizhevsky、Ilya Sutskever 和 Geoff Hinton 开发，2012 年 ImageNet 挑战赛获胜者。
  * **ZF Net。** 2013 年 ILSVRC 获胜者，由 Matthew Zeiler 和 Rob Fergus 开发。
  * **GoogLeNet。** 2014 年 ILSVRC 获胜者，来自 Google 的 Szegedy 等人。主要贡献是 Inception Module，大幅减少了参数数量（4M vs AlexNet 的 60M）。
  * **VGGNet。** 2014 年 ILSVRC 亚军，Karen Simonyan 和 Andrew Zisserman 开发。主要贡献是表明网络深度对良好性能至关重要。最终最佳网络包含 16 个 CONV/FC 层。
  * **ResNet。** Kaiming He 等人开发，2015 年 ILSVRC 获胜者。具有特殊的「跳跃连接」和大量使用批归一化。目前是最先进的卷积神经网络模型。

#### 计算考量

构建 ConvNet 架构时要注意的最大瓶颈是内存瓶颈。三个主要内存来源：
  * 中间体大小（激活及其梯度）
  * 参数大小（参数、梯度、优化缓存）
  * 其他内存（图像数据批次等）
