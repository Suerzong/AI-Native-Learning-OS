## 图像分类

**动机**。本节我们将介绍图像分类问题，即将输入图像分配到一个固定类别集合中的某个标签。这是计算机视觉的核心问题之一，尽管简单，但有着广泛的实际应用。此外，正如我们将在课程后面看到的，许多其他看似不同的计算机视觉任务（如目标检测、分割）都可以归约为图像分类。

**示例**。例如，下图中图像分类模型接收一张图像并为 4 个标签 _{cat, dog, hat, mug}_ 分配概率。如图所示，请记住在计算机看来，图像被表示为一个大的三维数字数组。在此示例中，猫图像宽 248 像素、高 400 像素，有三个颜色通道——红、绿、蓝（简称 RGB）。因此，图像由 248 x 400 x 3 = 297,600 个数字组成。每个数字是从 0（黑色）到 255（白色）的整数。我们的任务是将这近三十万个数字转换为单个标签，例如「猫」。

![](/assets/classify.png)

图像分类的任务是为给定图像预测一个单一标签（或标签上的分布，如图所示以表示我们的置信度）。图像是从 0 到 255 的整数构成的三维数组，尺寸为 宽 x 高 x 3。3 代表三个颜色通道红、绿、蓝。

**挑战**。由于识别视觉概念（如猫）对人类来说相对容易，值得从计算机视觉算法的角度考虑所涉及的挑战。在我们列出（非穷尽的）挑战清单时，请记住图像的原始表示是亮度值的 3D 数组：

  * **视角变化**。同一物体实例相对于相机可以有多种方向。
  * **尺度变化**。视觉类别在大小上经常表现出变化（现实世界中的大小，不仅仅是图像中的范围）。
  * **形变**。许多感兴趣的物体不是刚体，可以发生极端的变形。
  * **遮挡**。感兴趣的物体可能被遮挡。有时只有很小部分（甚至几个像素）可见。
  * **光照条件**。光照对像素级别的影响非常显著。
  * **背景杂乱**。感兴趣的物体可能「融入」环境中，使其难以识别。
  * **类内变化**。感兴趣的类别往往相对宽泛，例如「椅子」。这些物体有许多不同类型，各有不同的外观。

一个好的图像分类模型必须对所有这些变化的交叉积保持不变，同时保留对类间变化的敏感性。

![](/assets/challenges.jpeg)

**数据驱动方法**。我们如何编写一个能将图像分类到不同类别的算法？与编写排序数字列表的算法不同，编写识别图像中猫的算法并不显而易见。因此，我们采取的方法类似于教小孩：我们为计算机提供每个类别的许多示例，然后开发学习算法来观察这些示例并学习每个类别的视觉外观。这种方法被称为「数据驱动方法」，因为它依赖于首先积累一个标记图像的「训练数据集」。以下是此类数据集的示例：

![](/assets/trainset.jpg)

四个视觉类别的训练集示例。实际上，我们可能有数千个类别和每个类别数十万张图像。

**图像分类流程**。我们已经看到，图像分类的任务是接收代表单个图像的像素数组并为其分配标签。我们的完整流程可以形式化如下：

  * **输入：** 我们的输入包含一组 _N_ 张图像，每张标注了 _K_ 个不同类别中的一个。我们称这些数据为「训练集」。
  * **学习：** 我们的任务是使用训练集来学习每个类别的外观。我们将此步骤称为「训练分类器」或「学习模型」。
  * **评估：** 最后，我们要求分类器预测一组它从未见过的新图像的标签来评估分类器的质量。然后，我们将这些图像的真实标签与分类器预测的标签进行比较。直观上，我们希望许多预测与真实答案（称为「ground truth」）匹配。

### 最近邻分类器

作为第一个方法，我们将开发所谓的**最近邻分类器**。这个分类器与卷积神经网络无关，在实践中很少使用，但它让我们了解图像分类问题的基本方法。

**示例图像分类数据集：CIFAR-10**。一个流行的玩具图像分类数据集是 [CIFAR-10 数据集](https://www.cs.toronto.edu/~kriz/cifar.html)。该数据集包含 60,000 张 32x32 像素的小图像，每张标注了 10 个类别中的一个（例如「飞机、汽车、鸟等」）。这 60,000 张图像被划分为 50,000 张训练集和 10,000 张测试集。下图你可以看到从 10 个类别中各随机选取的 10 张示例图像：

![](/assets/nn.jpg)

左图：[CIFAR-10 数据集](https://www.cs.toronto.edu/~kriz/cifar.html)的示例图像。右图：第一列显示了几张测试图像，旁边是训练集中根据像素差异得到的前 10 个最近邻。

假设现在我们得到了 CIFAR-10 的 50,000 张图像的训练集（每个标签 5,000 张），我们希望标记剩余的 10,000 张。最近邻分类器将取出一张测试图像，与每一张训练图像进行比较，并预测最接近的训练图像的标签。上图右展示了这种过程的结果。注意只有约 3/10 的案例中检索到了同类图像，而其他 7 个案例则不是。例如第 8 行中，马头最近的训练图像是一辆红色汽车，可能是因为强烈的黑色背景。结果，这张马的图像会被错误标记为汽车。

你可能注意到我们没有指定比较两幅图像的具体细节——在本例中，两幅图像只是两个 32x32x3 的块。最简单的方法之一是逐像素比较图像并累加所有差异。换句话说，给定两幅图像并将其表示为向量 \\( I_1, I_2 \\)，一个合理的比较选择可能是 **L1 距离**（曼哈顿距离）：

\\[d_1 (I_1, I_2) = \sum_{p} \left| I^p_1 - I^p_2 \right|\\]

其中求和遍历所有像素。以下是该过程的可视化：

![](/assets/nneg.jpeg)

使用像素差异和 L1 距离比较两幅图像的示例（本例中只展示一个颜色通道）。两幅图像逐元素相减，然后将所有差异累加为一个数字。如果两幅图像完全相同，结果为零。如果图像差异很大，结果也会很大。

让我们看看如何用代码实现分类器。首先，将 CIFAR-10 数据加载到内存中，存储为 4 个数组：训练数据/标签和测试数据/标签。在下面的代码中，`Xtr`（大小为 50,000 x 32 x 32 x 3）保存训练集中的所有图像，对应的 1 维数组 `Ytr`（长度为 50,000）保存训练标签（从 0 到 9）：


    Xtr, Ytr, Xte, Yte = load_CIFAR10('data/cifar10/') # 我们提供一个魔法函数
    # 将所有图像展平为一维
    Xtr_rows = Xtr.reshape(Xtr.shape[0], 32 * 32 * 3) # Xtr_rows 变为 50000 x 3072
    Xte_rows = Xte.reshape(Xte.shape[0], 32 * 32 * 3) # Xte_rows 变为 10000 x 3072


现在所有图像都展平为行向量，以下是训练和评估分类器的方法：


    nn = NearestNeighbor() # 创建最近邻分类器类
    nn.train(Xtr_rows, Ytr) # 在训练图像和标签上训练分类器
    Yte_predict = nn.predict(Xte_rows) # 预测测试图像的标签
    # 打印分类准确率，即正确预测的样本比例
    print 'accuracy: %f' % ( np.mean(Yte_predict == Yte) )


注意，作为评估标准，通常使用**准确率**（accuracy），它衡量正确预测的比例。注意我们将要构建的所有分类器都遵循这个通用 API：它们有 `train(X,y)` 函数来学习数据和标签，内部类应构建某种标签模型以及如何从数据预测标签。然后有 `predict(X)` 函数接收新数据并预测标签。当然，我们省略了核心内容——分类器本身。以下是使用 L1 距离的简单最近邻分类器的实现：


    import numpy as np

    class NearestNeighbor(object):
      def __init__(self):
        pass

      def train(self, X, y):
        """ X 是 N x D 的矩阵，每行一个样本。Y 是大小为 N 的 1 维数组 """
        # 最近邻分类器只是记住所有训练数据
        self.Xtr = X
        self.ytr = y

      def predict(self, X):
        """ X 是 N x D 的矩阵，每行是我们要预测标签的样本 """
        num_test = X.shape[0]
        # 确保输出类型与输入类型匹配
        Ypred = np.zeros(num_test, dtype = self.ytr.dtype)

        # 遍历所有测试行
        for i in range(num_test):
          # 找到第 i 个测试图像最近的训练图像
          # 使用 L1 距离（绝对值差之和）
          distances = np.sum(np.abs(self.Xtr - X[i,:]), axis = 1)
          min_index = np.argmin(distances) # 获取最小距离的索引
          Ypred[i] = self.ytr[min_index] # 预测最近样本的标签

        return Ypred


如果你运行此代码，你会发现该分类器在 CIFAR-10 上仅达到 **38.6%** 的准确率。这比随机猜测（10 个类别，10% 准确率）要好，但远不及人类表现（[估计约 94%](https://karpathy.github.io/2011/04/27/manually-classifying-cifar10/)），也不及当前最好的卷积神经网络（约 95%，与人类准确率相当，参见 [Kaggle CIFAR-10 竞赛排行榜](https://www.kaggle.com/c/cifar-10/leaderboard)）。

**距离的选择**。计算向量间距离还有许多其他方式。另一个常见选择是使用 **L2 距离**（欧几里得距离），其几何解释是计算两个向量之间的欧几里得距离：

\\[d_2 (I_1, I_2) = \sqrt{\sum_{p} \left( I^p_1 - I^p_2 \right)^2}\\]

换句话说，我们照旧计算像素级差异，但这次将它们全部平方、相加，最后取平方根。在 numpy 中，只需替换一行代码：


    distances = np.sqrt(np.sum(np.square(self.Xtr - X[i,:]), axis = 1))


注意上面包含了 `np.sqrt` 调用，但在实际的最近邻应用中可以省略平方根操作，因为平方根是「单调函数」——它缩放距离的绝对大小但保留顺序，所以有无平方根的最近邻是相同的。如果在 CIFAR-10 上使用此距离运行最近邻分类器，你将获得 **35.4%** 的准确率（略低于 L1 距离的结果）。

**L1 vs. L2**。考虑两种度量之间的差异是有意义的。特别是，L2 距离在面对两向量之间的差异时比 L1 距离更加「不宽容」。也就是说，L2 距离倾向于多个中等差异而非一个大差异。L1 和 L2 距离（或等价地，一对图像差异的 L1/L2 范数）是 [p-范数](https://planetmath.org/vectorpnorm)的最常见特例。

### k 近邻分类器

你可能注意到，仅使用最近图像的标签进行预测是很奇怪的。事实上，使用 **k 近邻分类器**几乎总能做得更好。思路很简单：不是找到训练集中单个最接近的图像，而是找到前 **k** 个最接近的图像，让它们对测试图像的标签进行投票。特别地，当 _k = 1_ 时，我们恢复了最近邻分类器。直观上，较高的 **k** 值具有平滑效果，使分类器对异常值更鲁棒：

![](/assets/knn.jpeg)

最近邻和 5 近邻分类器差异的示例，使用 2 维点和 3 个类别（红、蓝、绿）。彩色区域显示了使用 L2 距离的分类器产生的**决策边界**。白色区域显示分类不明确的点（即至少两个类别的投票并列）。注意在 NN 分类器的情况下，异常数据点（例如蓝色点云中的绿色点）会创建小的可能错误预测岛屿，而 5-NN 分类器平滑了这些不规则性，可能在测试数据上产生更好的**泛化**。另请注意，5-NN 图中的灰色区域是由最近邻之间投票并列引起的（例如 2 个邻居是红色，接下来 2 个是蓝色，最后 1 个是绿色）。

在实践中，你几乎总是想用 k 近邻。但应该用什么 _k_ 值呢？接下来我们讨论这个问题。

### 用于超参数调优的验证集

k 近邻分类器需要设置 _k_ 值。但哪个值效果最好？此外，我们看到还有许多不同的距离函数可以选择：L1 范数、L2 范数，还有许多我们甚至没有考虑过的选择（例如点积）。这些选择被称为**超参数**，它们在许多从数据中学习的机器学习算法设计中经常出现。通常并不明显应该选择什么值/设置。

你可能想建议我们应该尝试许多不同的值，看看哪个效果最好。这是个好主意，也确实是我们要做的，但这必须非常小心地进行。特别是，**我们不能使用测试集来调整超参数**。在设计机器学习算法时，你应该将测试集视为非常宝贵的资源，理想情况下直到最后才碰一次。否则，你可能会将超参数调整得在测试集上表现良好，但如果部署模型，性能可能会显著下降。在实践中，我们会说你**过拟合**了测试集。另一种理解方式是，如果在测试集上调优超参数，你实际上是将测试集作为训练集使用，因此在测试集上取得的性能相对于实际部署时可能观察到的性能过于乐观。但如果只在最后使用一次测试集，它仍然是衡量分类器**泛化**能力的良好代理（我们将在课程后面看到更多关于泛化的讨论）。

> 只在最后对测试集评估一次。

幸运的是，有一种正确的超参数调优方法，它完全不接触测试集。思路是将训练集分为两部分：一个稍小的训练集和一个**验证集**。以 CIFAR-10 为例，我们可以用 49,000 张训练图像进行训练，留下 1,000 张用于验证。这个验证集本质上是用作调整超参数的模拟测试集。

以下是在 CIFAR-10 中的具体做法：


    # 假设我们有 Xtr_rows, Ytr, Xte_rows, Yte 如前所述
    # 回顾 Xtr_rows 是 50,000 x 3072 的矩阵
    Xval_rows = Xtr_rows[:1000, :] # 取前 1000 个用于验证
    Yval = Ytr[:1000]
    Xtr_rows = Xtr_rows[1000:, :] # 保留后 49,000 个用于训练
    Ytr = Ytr[1000:]

    # 找到在验证集上效果最好的超参数
    validation_accuracies = []
    for k in [1, 3, 5, 10, 20, 50, 100]:

      # 使用特定的 k 值在验证数据上评估
      nn = NearestNeighbor()
      nn.train(Xtr_rows, Ytr)
      # 假设一个修改版的 NearestNeighbor 类可以接受 k 作为输入
      Yval_predict = nn.predict(Xval_rows, k = k)
      acc = np.mean(Yval_predict == Yval)
      print 'accuracy: %f' % (acc,)

      # 记录在验证集上有效的参数
      validation_accuracies.append((k, acc))


在此过程结束时，我们可以绘制一个图表，显示哪些 _k_ 值效果最好。然后我们将使用这个值在实际测试集上评估一次。

> 将训练集分为训练集和验证集。使用验证集调整所有超参数。最后在测试集上运行一次并报告性能。

**交叉验证**。在训练数据（因此验证数据）量较小的情况下，人们有时使用一种更复杂的超参数调优技术，称为**交叉验证**。在前面的例子中，不是随意选择前 1000 个数据点作为验证集，而是可以通过在不同的验证集上迭代并平均性能来获得更好、更少噪声的 _k_ 值估计。例如，在 5 折交叉验证中，我们将训练数据分成 5 等份，用其中 4 份训练，1 份验证。然后我们迭代使用不同的折作为验证折，评估性能，最后平均不同折的性能。

![](/assets/cvplot.png)

参数 **k** 的 5 折交叉验证示例。对于每个 **k** 值，我们在 4 个折上训练，在第 5 个折上评估。因此，对于每个 **k** 我们得到 5 个验证折上的准确率（y 轴是准确率，每个结果是一个点）。趋势线是每个 **k** 结果的平均值，误差条表示标准差。注意在此特定情况下，交叉验证表明 **k** ≈ 7 在此数据集上效果最好（对应图中的峰值）。如果我们使用超过 5 折，我们可能会看到更平滑（即噪声更少）的曲线。

**在实践中**。人们倾向于避免交叉验证，而使用单一验证分割，因为交叉验证的计算成本很高。常见的分割是将 50%-90% 的训练数据用于训练，其余用于验证。但这取决于多种因素：例如如果超参数数量很多，你可能倾向于使用更大的验证分割。如果验证集中的样本数很少（可能只有几百个），使用交叉验证更安全。实践中常见的折数是 3 折、5 折或 10 折交叉验证。

![](/assets/crossval.jpeg)

常见的数据分割。给定训练集和测试集。训练集被分成若干折（本例为 5 折）。折 1-4 成为训练集。一个折（例如黄色的折 5）被标记为验证折，用于调整超参数。交叉验证更进一步，迭代选择哪个折作为验证折，从 1 到 5 分别进行。这被称为 5 折交叉验证。最后，一旦模型训练完成且所有最佳超参数确定，模型在测试数据（红色）上评估一次。

**最近邻分类器的优缺点**

值得考虑最近邻分类器的一些优点和缺点。显然，一个优点是它非常简单易实现和理解。此外，分类器训练不需要时间，因为只需要存储（并可能索引）训练数据。但我们在测试时付出了计算代价，因为分类一个测试样本需要与每一个训练样本进行比较。这与实际需求相反，因为我们通常更关心测试时的效率而非训练时的效率。实际上，我们稍后在本课程中开发的深度神经网络将这种权衡推向了另一个极端：训练非常昂贵，但一旦训练完成，分类新的测试样本就非常便宜。这种操作模式在实践中更受欢迎。

顺便提一下，最近邻分类器的计算复杂度是一个活跃的研究领域，存在多种**近似最近邻**（ANN）算法和库可以加速数据集中的最近邻查找（例如 [FLANN](https://github.com/mariusmuja/flann)）。这些算法允许在最近邻检索的正确性和空间/时间复杂度之间进行权衡，通常依赖于预处理/索引阶段，包括构建 kdtree 或运行 k-means 算法。

最近邻分类器在某些场景下可能是不错的选择（特别是当数据是低维时），但在实际图像分类中很少适用。一个问题是图像是高维对象（即通常包含许多像素），高维空间上的距离可能非常违反直觉。下图说明了我们上面开发的基于像素的 L2 相似性与感知相似性非常不同：

![](/assets/samenorm.png)

高维数据（尤其是图像）上的像素距离可能非常不直观。原始图像（左）和旁边的三幅图像，基于 L2 像素距离它们与原始图像距离相等。显然，像素级距离与感知或语义相似性完全不符。

以下是另一个可视化，以说服你使用像素差异比较图像是不够的。我们可以使用一种称为 [t-SNE](https://lvdmaaten.github.io/tsne/) 的可视化技术将 CIFAR-10 图像嵌入二维空间，使得它们的（局部）两两距离得到最好保留。在这个可视化中，彼此靠近的图像根据我们上面开发的 L2 像素距离被认为是相近的：

![](/assets/pixels_embed_cifar10.jpg)

使用 t-SNE 将 CIFAR-10 图像嵌入二维空间。在此图像中彼此靠近的图像基于 L2 像素距离被认为是相近的。注意背景的强烈影响而非语义类别差异。点击[这里](/assets/pixels_embed_cifar10_big.jpg)查看更大版本。

特别要注意的是，彼此靠近的图像更多是图像的一般颜色分布或背景类型的函数，而不是它们的语义身份。例如，一只狗可能离一只青蛙很近，因为两者恰好都在白色背景上。理想情况下，我们希望所有 10 个类别的图像形成各自的簇，使得同一类别的图像彼此靠近，而不受无关特征和变化（如背景）的影响。然而，要获得这一特性，我们必须超越原始像素。

### 总结

总结：

  * 我们介绍了**图像分类**问题，其中我们被给定一组标注了单一类别的图像，然后要求在一组新的测试图像上预测这些类别并衡量预测准确率。
  * 我们介绍了一个简单分类器——**最近邻分类器**。我们看到有多个超参数（如 k 值或用于比较样本的距离类型）与该分类器相关联，且没有明显的方法选择它们。
  * 我们了解到设置这些超参数的正确方法是将训练数据分成两部分：训练集和模拟测试集（称为**验证集**）。我们尝试不同的超参数值，保留在验证集上表现最好的值。
  * 如果担心训练数据不足，我们讨论了一种称为**交叉验证**的程序，可以帮助减少估计最佳超参数时的噪声。
  * 找到最佳超参数后，我们在实际测试集上进行一次**评估**。
  * 我们看到最近邻在 CIFAR-10 上可以获得约 40% 的准确率。它实现简单，但需要存储整个训练集且评估测试图像代价高昂。
  * 最后，我们看到在原始像素值上使用 L1 或 L2 距离是不够的，因为这些距离与图像的背景和颜色分布的相关性远强于与语义内容的相关性。

在接下来的课程中，我们将着手解决这些挑战，最终到达能给出 90% 准确率的解决方案，让我们在学习完成后完全丢弃训练集，并能在不到一毫秒内评估测试图像。

### 总结：在实践中应用 kNN

如果你希望在实践中应用 kNN（希望不是用于图像，或者仅作为基线），请按以下步骤进行：

  1. 预处理数据：将数据中的特征（如图像中的每个像素）归一化为零均值和单位方差。我们将在后面的章节中更详细地介绍这一点，在此略过因为图像中的像素通常是同质的，不需要数据归一化。
  2. 如果数据维度非常高，考虑使用降维技术如 PCA（[wiki 参考](https://en.wikipedia.org/wiki/Principal_component_analysis)、[CS229 参考](http://cs229.stanford.edu/notes/cs229-notes10.pdf)）、NCA（[wiki 参考](https://en.wikipedia.org/wiki/Neighbourhood_components_analysis)、[博客参考](https://kevinzakka.github.io/2020/02/10/nca/)）或甚至[随机投影](https://scikit-learn.org/stable/modules/random_projection.html)。
  3. 将训练数据随机分为训练/验证集。一般规则，70-90% 的数据通常用于训练。这取决于超参数的数量及其预期影响。如果超参数很多，应偏向使用更大的验证集。如果担心验证数据规模，最好将训练数据分成折并执行交叉验证。如果计算预算允许，交叉验证总是更安全的（折越多越好，但也越昂贵）。
  4. 在验证数据上（如果做交叉验证则对所有折）训练和评估 kNN 分类器，尝试多种 **k** 值（越多越好）和不同的距离类型（L1 和 L2 是好的候选）。
  5. 如果 kNN 分类器运行时间太长，考虑使用近似最近邻库（如 [FLANN](https://github.com/mariusmuja/flann)）来加速检索（牺牲一些准确率）。
  6. 记下给出最佳结果的超参数。关于是否应该使用完整的训练集配合最佳超参数存在疑问，因为如果将验证数据纳入训练集，最优超参数可能会改变（因为数据量会更大）。实践中更干净的做法是在最终分类器中不使用验证数据，认为它已「消耗」在超参数估计上。在测试集上评估最佳模型。报告测试集准确率并声明该结果为 kNN 分类器在你的数据上的性能。
