# 欢迎使用 fastai

fastai 借助现代最佳实践，简化了快速、准确的神经网络训练过程

[](https://github.com/fastai/fastai/actions/workflows/main.yml) [](https://pypi.org/project/fastai/#description) [](https://anaconda.org/fastai/fastai)

## 安装

你可以通过 [Google Colab](https://colab.research.google.com/) 无需任何安装即可使用 fastai。事实上，本文档的每个页面也可以作为交互式笔记本使用——点击任何页面顶部的"在 Colab 中打开"即可打开（请务必将 Colab 运行时更改为"GPU"以使其运行更快！）更多信息请参阅 fast.ai 文档中关于 [使用 Colab](https://course19.fast.ai/start_colab.html) 的说明。

你可以使用以下命令在自己的机器上安装 fastai：`pip install fastai`。

为确保你的机器上有最新版本的 PyTorch，建议先[安装](https://pytorch.org/get-started/locally/) PyTorch。

如果你计划自行开发 fastai，或想使用最新版本，可以使用可编辑安装（如果这样做，还应同时使用 [fastcore](https://github.com/fastai/fastcore) 的可编辑安装）。首先安装 PyTorch，然后：

```bash
git clone https://github.com/fastai/fastai
pip install -e "fastai[dev]"
```

## 学习 fastai

开始使用 fastai（和深度学习）的最好方式是阅读[书籍](https://www.amazon.com/Deep-Learning-Coders-fastai-PyTorch/dp/1492045527)，并完成[免费课程](https://course.fast.ai)。

要了解 fastai 能做什么，请查看[快速入门](https://docs.fast.ai/quick_start.html)，它展示了如何用大约 5 行代码构建图像分类器、图像分割模型、文本情感模型、推荐系统和表格模型。对于每个应用，代码都非常相似。

阅读[教程](https://docs.fast.ai/tutorial.html)以学习如何在自己的数据集上训练自己的模型。使用导航侧栏浏览 fastai 文档。每个类、函数和方法都有文档记录。

要了解该库的设计和动机，请阅读[同行评审论文](https://www.mdpi.com/2078-2489/11/2/108/htm)。

## 关于 fastai

fastai 是一个深度学习库，为从业者提供高级组件，可以快速轻松地在标准深度学习领域提供最先进的结果；同时为研究人员提供低级组件，可以混合搭配以构建新方法。它的目标是在易用性、灵活性或性能方面不做重大妥协地同时实现这两件事。这得益于精心设计的分层架构，它通过解耦的抽象来表达许多深度学习和数据处理技术的共同底层模式。这些抽象可以利用底层 Python 语言的动态性和 PyTorch 库的灵活性来简洁清晰地表达。fastai 包括：

  * 一种新的 Python 类型分派系统，以及张量的语义类型层次结构
  * 一个 GPU 优化的计算机视觉库，可以用纯 Python 扩展
  * 一种优化器，将现代优化器的通用功能重构为两个基本部分，使得优化算法可以用 4-5 行代码实现
  * 一种新颖的双向回调系统，可以在训练过程中随时访问数据、模型或优化器的任何部分并进行修改
  * 一种新的数据块 API
  * 以及更多...

fastai 围绕两个主要设计目标组织：易于上手且能快速产出成果，同时也能深度定制和配置。它建立在低级 API 的层次结构之上，这些 API 提供了可组合的构建模块。这样，想要重写部分高级 API 或添加特定行为以满足需求的用户不必学习如何使用最低级别的 API。

## 从其他库迁移

从纯 PyTorch、Ignite 或任何其他基于 PyTorch 的库迁移非常容易，甚至可以将 fastai 与其他库结合使用。通常，你将能够使用所有现有的数据处理代码，但可以减少训练所需的代码量，并更容易利用现代最佳实践。以下是一些流行库的迁移指南，帮助你顺利过渡：

  * [纯 PyTorch](https://docs.fast.ai/examples/migrating_pytorch.html)
  * [Ignite](https://docs.fast.ai/examples/migrating_ignite.html)
  * [Lightning](https://docs.fast.ai/examples/migrating_lightning.html)
  * [Catalyst](https://docs.fast.ai/examples/migrating_catalyst.html)

## Windows 支持

由于 Jupyter 和 Windows 上的 Python 多进程问题，`DataLoader` 的 `num_workers` 会自动重置为 0，以避免 Jupyter 卡死。这使得在 Windows 上的 Jupyter 中执行计算机视觉等任务比在 Linux 上慢很多倍。如果你从脚本使用 fastai，则不存在此限制。

参阅[此示例](https://github.com/fastai/fastai/blob/master/nbs/examples/dataloader_spawn.py)以在 Windows 上充分利用 fastai API。

我们建议改用 Windows Subsystem for Linux（WSL）——如果这样做，你可以使用常规的 Linux 安装方法，不会遇到 `num_workers` 相关的问题。

## 测试

要并行运行测试，请执行：

```bash
nbdev_test
```

要使所有测试通过，你需要安装 settings.ini 中 dev_requirements 指定的依赖项：

```bash
pip install -e .[dev]
```

测试使用 `nbdev` 编写，例如参阅 `test_eq` 的文档。

## 贡献

克隆此仓库后，请确保在终端中运行了 `nbdev_install_hooks`。这会安装 Jupyter 和 git 钩子，以自动清理、信任和修复笔记本中的合并冲突。

在仓库中进行更改后，你应该运行 `nbdev_prepare` 并进行额外的必要更改，以通过所有测试。
