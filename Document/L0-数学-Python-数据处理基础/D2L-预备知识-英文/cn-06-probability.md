# 2.6. 概率与统计（Probability and Statistics）

不管怎样，机器学习都是关于不确定性的。在监督学习中，我们想要根据已知信息（_特征_）预测未知的东西（_目标_）。根据我们的目标，我们可能试图预测目标最可能的值。或者我们可能预测与目标期望距离最小的值。有时我们不仅希望预测一个特定的值，还希望 _量化我们的不确定性_。例如，给定描述患者的某些特征，我们可能想知道他们在未来一年内心脏病发作的 _可能性有多大_。在无监督学习中，我们经常关注不确定性。要确定一组测量值是否异常，了解在感兴趣的群体中观察到这些值的可能性有多大是有帮助的。此外，在强化学习中，我们希望开发能在各种环境中智能行动的智能体。这需要推理环境可能如何变化，以及对每个可用动作可能遇到的奖励。

_概率_（Probability）是关注不确定性下推理的数学领域。给定某个过程的概率模型，我们可以推理各种事件的可能性。使用概率来描述可重复事件（如抛硬币）的频率是相当无争议的。事实上，_频率主义_（frequentist）学者坚持一种只适用于此类可重复事件的概率解释。相比之下，_贝叶斯_（Bayesian）学者更广泛地使用概率语言来形式化不确定性下的推理。贝叶斯概率有两个独特的特点：(i) 为不可重复事件分配信念度，例如大坝崩溃的 _概率_ 是多少？；(ii) 主观性。虽然贝叶斯概率为人们应该如何根据新证据更新信念提供了明确的规则，但它允许不同的个体从不同的 _先验_（prior）信念开始。_统计学_（Statistics）帮助我们反向推理，从收集和组织数据开始，回溯到我们可以对生成数据的过程得出什么推论。每当我们分析数据集，寻找我们希望可能表征更广泛群体的模式时，我们都在运用统计思维。许多课程、专业、论文、职业、院系、公司和机构都致力于概率和统计的研究。虽然本节只是触及表面，但我们将为你提供开始构建模型所需的基础知识。

## 2.6.1. 一个简单的例子：抛硬币

想象我们计划抛一枚硬币，并想量化看到正面（相对于反面）的可能性有多大。如果硬币是 _公平的_，那么两种结果（正面和反面）的可能性相同。此外，如果我们计划抛硬币 \\(n\\) 次，那么我们 _期望_ 看到的正面比例应该与反面的 _期望_ 比例完全一致。一种直观的方式来理解这一点是对称性：对于每一个有 \\(n_\textrm{h}\\) 次正面和 \\(n_\textrm{t} = (n - n_\textrm{h})\\) 次反面的可能结果，都有一个同样可能的结果，其中有 \\(n_\textrm{t}\\) 次正面和 \\(n_\textrm{h}\\) 次反面。请注意，只有在我们期望平均有 \\(1/2\\) 的抛掷出现正面和 \\(1/2\\) 出现反面时这才可能。当然，如果你用 \\(n=1000000\\) 次抛掷进行多次这个实验，你可能永远不会看到一个 \\(n_\textrm{h} = n_\textrm{t}\\) 精确相等的试验。

从形式上讲，量 \\(1/2\\) 称为 _概率_（probability），它在这里捕获了任何给定抛掷出现正面的确定程度。概率将介于 \\(0\\) 和 \\(1\\) 之间的分数分配给感兴趣的事件（event）。这里感兴趣的事件是 \\(\textrm{heads}\\)（正面），我们用 \\(P(\textrm{heads})\\) 表示相应的概率。概率为 \\(1\\) 表示绝对确定（想象一枚两面都是正面的魔术硬币），概率为 \\(0\\) 表示不可能（例如两面都是反面）。频率 \\(n_\textrm{h}/n\\) 和 \\(n_\textrm{t}/n\\) 不是概率，而是 _统计量_（statistics）。概率是 _理论_ 量，是数据生成过程的基础。这里，概率 \\(1/2\\) 是硬币本身的属性。相比之下，统计量是 _经验_ 量，是作为观测数据的函数计算的。我们对概率和统计量的兴趣密不可分。我们经常设计称为 _估计量_（estimators）的特殊统计量，给定数据集，产生模型参数（如概率）的 _估计值_（estimates）。此外，当这些估计量满足一个称为 _一致性_（consistency）的良好性质时，我们的估计值将收敛到相应的概率。反过来，这些推断出的概率告诉我们来自同一群体的未来可能遇到的数据的可能统计特性。

假设我们偶然发现了一枚我们不知道真实 \\(P(\textrm{heads})\\) 的硬币。要用统计方法研究这个量，我们需要：(i) 收集一些数据；(ii) 设计一个估计量。这里的数据获取很简单；我们可以多次抛硬币并记录所有结果。从形式上讲，从某个底层随机过程中抽取实现称为 _采样_（sampling）。正如你可能猜到的，一个自然的估计量是观测到的 _正面_ 次数与总抛掷次数的比率。

现在，假设硬币实际上是公平的，即 \\(P(\textrm{heads}) = 0.5\\)。为了模拟公平硬币的抛掷，我们可以调用任何随机数生成器。有一些简单的方法可以抽取概率为 \\(0.5\\) 的事件的样本。例如，Python 的 `random.random` 生成区间 \\([0,1]\\) 内的数字，其中位于任何子区间 \\([a, b] \subset [0,1]\\) 的概率等于 \\(b-a\\)。因此，我们可以测试返回的浮点数是否大于 `0.5` 来以 `0.5` 的概率分别得到 `0` 和 `1`：

    num_tosses = 100
    heads = sum([random.random() > 0.5 for _ in range(num_tosses)])
    tails = num_tosses - heads
    print("heads, tails: ", [heads, tails])

    heads, tails:  [44, 56]

更一般地说，我们可以通过调用多项式函数来模拟从具有有限数量可能结果的变量（如抛硬币或掷骰子）中多次抽取，将第一个参数设置为抽取次数，第二个参数设置为与每个可能结果相关联的概率列表。为了模拟十次公平硬币抛掷，我们分配概率向量 `[0.5, 0.5]`，将索引 0 解释为正面，索引 1 解释为反面。该函数返回一个长度等于可能结果数量（此处为 2）的向量，其中第一个分量告诉我们正面出现的次数，第二个分量告诉我们反面出现的次数。

    fair_probs = torch.tensor([0.5, 0.5])
    Multinomial(100, fair_probs).sample()

    tensor([50., 50.])

    fair_probs = [0.5, 0.5]
    multinomial(100, fair_probs)

    array([46, 54], dtype=int64)

    fair_probs = [0.5, 0.5]
    np.random.multinomial(100, fair_probs)

    array([44, 56])

    fair_probs = tf.ones(2) / 2
    tfd.Multinomial(100, fair_probs).sample()

    <tf.Tensor: shape=(2,), dtype=float32, numpy=array([49., 51.], dtype=float32)>

每次运行这个采样过程时，你都会收到一个可能与之前结果不同的新随机值。除以抛掷次数得到我们数据中每个结果的 _频率_（frequency）。请注意，这些频率与它们旨在估计的概率一样，总和为 \\(1\\)。

    Multinomial(100, fair_probs).sample() / 100

    tensor([0.4800, 0.5200])

    multinomial(100, fair_probs) / 100

    array([0.53, 0.47])

    np.random.multinomial(100, fair_probs) / 100

    array([0.5, 0.5])

    tfd.Multinomial(100, fair_probs).sample() / 100

    <tf.Tensor: shape=(2,), dtype=float32, numpy=array([0.59, 0.41], dtype=float32)>

在这里，即使我们模拟的硬币是公平的（我们自己设置了概率 `[0.5, 0.5]`），正面和反面的计数可能不完全相同。这是因为我们只抽取了相对少量的样本。如果我们没有自己实施模拟，只看到了结果，我们如何知道硬币是稍微不公平的，还是偏离 \\(1/2\\) 只是小样本量的假象？让我们看看当我们模拟 10,000 次抛掷时会发生什么。

    counts = Multinomial(10000, fair_probs).sample()
    counts / 10000

    tensor([0.4966, 0.5034])

    counts = multinomial(10000, fair_probs).astype(np.float32)
    counts / 10000

    array([0.4952, 0.5048])

    counts = np.random.multinomial(10000, fair_probs).astype(np.float32)
    counts / 10000

    array([0.5007, 0.4993], dtype=float32)

    counts = tfd.Multinomial(10000, fair_probs).sample()
    counts / 10000

    <tf.Tensor: shape=(2,), dtype=float32, numpy=array([0.5019, 0.4981], dtype=float32)>

一般来说，对于重复事件（如抛硬币）的平均值，随着重复次数的增加，我们的估计保证会收敛到真实的底层概率。这种现象的数学表述称为 _大数定律_（law of large numbers），而 _中心极限定理_（central limit theorem）告诉我们，在许多情况下，随着样本量 \\(n\\) 的增长，这些误差应以 \\((1/\sqrt{n})\\) 的速率下降。让我们通过研究当抛掷次数从 1 增长到 10,000 时我们的估计如何演变来获得更多直觉。

    counts = Multinomial(1, fair_probs).sample((10000,))
    cum_counts = counts.cumsum(dim=0)
    estimates = cum_counts / cum_counts.sum(dim=1, keepdims=True)
    estimates = estimates.numpy()

    d2l.set_figsize((4.5, 3.5))
    d2l.plt.plot(estimates[:, 0], label=("P(coin=heads)"))
    d2l.plt.plot(estimates[:, 1], label=("P(coin=tails)"))
    d2l.plt.axhline(y=0.5, color='black', linestyle='dashed')
    d2l.plt.gca().set_xlabel('Samples')
    d2l.plt.gca().set_ylabel('Estimated probability')
    d2l.plt.legend();

每条实线对应于硬币的两个值之一，并给出了在每组实验后硬币出现该值的估计概率。黑色虚线给出了真实的底层概率。随着我们通过进行更多实验获得更多数据，这些曲线收敛到真实概率。你可能已经开始看到统计学家关注的一些更高级问题的轮廓：这种收敛发生得有多快？如果我们已经测试了同一工厂生产的许多硬币，我们如何纳入这些信息？

## 2.6.2. 更正式的处理

我们已经取得了很大进展：提出概率模型、生成合成数据、运行统计估计量、凭经验评估收敛性以及报告误差指标（检查偏差）。然而，要走得更远，我们需要更加精确。

在处理随机性时，我们用 \\(\mathcal{S}\\) 表示可能结果的集合，称为 _样本空间_（sample space）或 _结果空间_（outcome space）。这里，每个元素是一个不同的可能 _结果_（outcome）。在抛一枚硬币的情况下，\\(\mathcal{S} = \\{\textrm{heads}, \textrm{tails}\\}\\)。对于一个骰子，\\(\mathcal{S} = \\{1, 2, 3, 4, 5, 6\\}\\)。当抛两枚硬币时，可能的结果是 \\(\\{(\textrm{heads}, \textrm{heads}), (\textrm{heads}, \textrm{tails}), (\textrm{tails}, \textrm{heads}), (\textrm{tails}, \textrm{tails})\\}\\)。_事件_（events）是样本空间的子集。例如，事件"第一次抛硬币出现正面"对应于集合 \\(\\{(\textrm{heads}, \textrm{heads}), (\textrm{heads}, \textrm{tails})\\}\\)。每当随机实验的结果 \\(z\\) 满足 \\(z \in \mathcal{A}\\) 时，事件 \\(\mathcal{A}\\) 就发生了。对于掷一次骰子，我们可以定义事件"看到 \\(5\\)"（\\(\mathcal{A} = \\{5\\}\\)）和"看到奇数"（\\(\mathcal{B} = \\{1, 3, 5\\}\\)）。在这种情况下，如果骰子出现 \\(5\\)，我们会说 \\(\mathcal{A}\\) 和 \\(\mathcal{B}\\) 都发生了。另一方面，如果 \\(z = 3\\)，则 \\(\mathcal{A}\\) 没有发生，但 \\(\mathcal{B}\\) 发生了。

_概率_（probability）函数将事件映射到实值 \\({P: \mathcal{A} \subseteq \mathcal{S} \rightarrow [0,1]}\\)。给定样本空间 \\(\mathcal{S}\\) 中事件 \\(\mathcal{A}\\) 的概率，记为 \\(P(\mathcal{A})\\)，具有以下性质：

  * 任何事件 \\(\mathcal{A}\\) 的概率是非负实数，即 \\(P(\mathcal{A}) \geq 0\\)；

  * 整个样本空间的概率为 \\(1\\)，即 \\(P(\mathcal{S}) = 1\\)；

  * 对于任意可数的 _互斥_（mutually exclusive）事件序列 \\(\mathcal{A}_1, \mathcal{A}_2, \ldots\\)（即对所有 \\(i \neq j\\) 有 \\(\mathcal{A}_i \cap \mathcal{A}_j = \emptyset\\)），其中任何一个发生的概率等于它们各自概率的和，即 \\(P(\bigcup_{i=1}^{\infty} \mathcal{A}_i) = \sum_{i=1}^{\infty} P(\mathcal{A}_i)\\)。

这些概率论公理由 Kolmogorov ([1933](../chapter_references/zreferences.html#id151 "Kolmogorov, A. \(1933\). Sulla determinazione empirica di una legge di distribuzione. Inst. Ital. Attuari, Giorn., 4, 83–91.")) 提出，可以迅速推导出许多重要的结果。例如，任何事件 \\(\mathcal{A}\\) _或_ 其补集 \\(\mathcal{A}'\\) 发生的概率为 1（因为 \\(\mathcal{A} \cup \mathcal{A}' = \mathcal{S}\\)）。我们还可以证明 \\(P(\emptyset) = 0\\)，因为 \\(1 = P(\mathcal{S} \cup \mathcal{S}') = P(\mathcal{S} \cup \emptyset) = P(\mathcal{S}) + P(\emptyset) = 1 + P(\emptyset)\\)。因此，任何事件 \\(\mathcal{A}\\) _和_ 其补集 \\(\mathcal{A}'\\) 同时发生的概率为 \\(P(\mathcal{A} \cap \mathcal{A}') = 0\\)。非正式地说，这告诉我们不可能事件发生的概率为零。

## 2.6.3. 随机变量（Random Variables）

当我们谈到掷骰子出现奇数或第一次抛硬币出现正面这样的事件时，我们是在引用 _随机变量_（random variable）的概念。从形式上讲，随机变量是从底层样本空间到一组（可能很多）值的映射。你可能想知道随机变量与样本空间有什么不同，因为两者都是结果的集合。重要的是，随机变量可以比原始样本空间粗糙得多。即使底层样本空间是无限的（例如 0 和 1 之间的线段上的点），我们也可以定义一个二元随机变量，如"大于 0.5"。此外，多个随机变量可以共享相同的底层样本空间。例如，"我家的报警器是否响了"和"我家是否被盗"都是共享底层样本空间的二元随机变量。因此，知道一个随机变量取的值可以告诉我们另一个随机变量可能取的值。如果知道报警器响了，我们可能会怀疑房子很可能被盗了。

随机变量取的每个值对应于底层样本空间的一个子集。因此，随机变量 \\(X\\) 取值 \\(v\\) 的情况，记为 \\(X=v\\)，是一个 _事件_，而 \\(P(X=v)\\) 表示其概率。有时这种记法可能很笨拙，当上下文清楚时我们可以滥用记法。例如，我们可以使用 \\(P(X)\\) 来广泛指代 \\(X\\) 的 _分布_（distribution），即告诉我们 \\(X\\) 取任何给定值的概率的函数。其他时候我们写像 \\(P(X,Y) = P(X) P(Y)\\) 这样的表达式，作为对随机变量 \\(X\\) 和 \\(Y\\) 可以取的所有值都成立的简写，即对所有 \\(i,j\\) 有 \\(P(X=i \textrm{ and } Y=j) = P(X=i)P(Y=j)\\)。还有些时候，当随机变量从上下文中清楚时，我们通过写 \\(P(v)\\) 来滥用记法。由于概率论中的事件是样本空间的一组结果，我们可以指定随机变量取值的范围。例如，\\(P(1 \leq X \leq 3)\\) 表示事件 \\(\\{1 \leq X \leq 3\\}\\) 的概率。

请注意，_离散_（discrete）随机变量（如抛硬币或掷骰子）与 _连续_（continuous）随机变量（如从群体中随机抽样的人的体重和身高）之间有一个微妙的区别。在这种情况下，我们很少真正关心某人的精确身高。此外，如果我们进行足够精确的测量，我们会发现地球上没有两个人的身高完全相同。事实上，测量足够精确的话，你醒来时和睡觉时的身高都不会相同。询问某人身高恰好为 1.801392782910287192 米的概率几乎没有意义。相反，我们通常更关心能否说出某人的身高是否落在某个给定区间内，例如在 1.79 米和 1.81 米之间。在这些情况下，我们使用概率 _密度_（densities）。恰好 1.80 米的身高没有概率，但有非零密度。要计算分配给某个区间的概率，我们必须对该区间上的密度取 _积分_（integral）。

## 2.6.4. 多个随机变量

你可能已经注意到，如果不涉及多个随机变量之间的交互，我们甚至无法完成上一节（回忆 \\(P(X,Y) = P(X) P(Y)\\)）。机器学习的大部分内容都关注这样的关系。在这里，样本空间将是感兴趣的群体，例如与企业交易的客户、互联网上的照片或生物学家已知的蛋白质。每个随机变量将代表一个不同属性的（未知）值。每当我们从群体中抽样一个个体时，我们就观察到每个随机变量的一个实现。由于随机变量取的值对应于可能重叠、部分重叠或完全不相交的样本空间子集，知道一个随机变量取的值可能导致我们更新关于另一个随机变量哪些值可能的信念。如果一个患者走进医院，我们观察到他们呼吸困难并丧失了嗅觉，那么我们相信他们患 COVID-19 的可能性比他们没有呼吸困难且嗅觉完全正常时更大。

当处理多个随机变量时，我们可以构造对应于变量可以联合取的每个值组合的事件。将概率分配给这些组合（例如 \\(A=a\\) 和 \\(B=b\\)）的概率函数称为 _联合概率_（joint probability）函数，它简单地返回分配给样本空间相应子集交集的概率。随机变量 \\(A\\) 和 \\(B\\) 分别取值 \\(a\\) 和 \\(b\\) 的事件的 _联合概率_ 记为 \\(P(A = a, B = b)\\)，其中逗号表示"且"。请注意，对于任何值 \\(a\\) 和 \\(b\\)，有

(2.6.1)\\[P(A=a, B=b) \leq P(A=a) \textrm{ 且 } P(A=a, B=b) \leq P(B = b),\\]

因为要使 \\(A=a\\) 和 \\(B=b\\) 同时发生，\\(A=a\\) 必须发生 _且_ \\(B=b\\) 也必须发生。有趣的是，联合概率告诉了我们在概率意义上关于这些随机变量的所有信息，并可用于推导许多其他有用的量，包括恢复各个分布 \\(P(A)\\) 和 \\(P(B)\\)。为了恢复 \\(P(A=a)\\)，我们只需对随机变量 \\(B\\) 可以取的所有值 \\(v\\) 求和 \\(P(A=a, B=v)\\)：\\(P(A=a) = \sum_v P(A=a, B=v)\\)。

比率 \\(\frac{P(A=a, B=b)}{P(A=a)} \leq 1\\) 被证明极其重要。它被称为 _条件概率_（conditional probability），用"\\(\mid\\)"符号表示：

(2.6.2)\\[P(B=b \mid A=a) = P(A=a,B=b)/P(A=a).\\]

它告诉我们，在以 \\(A=a\\) 发生为条件后，与事件 \\(B=b\\) 相关的新概率。我们可以将这个条件概率视为仅关注与 \\(A=a\\) 相关的样本空间子集，然后重新归一化使所有概率之和为 1。条件概率实际上就是普通的概率，因此只要我们将所有项都以同一事件为条件，即关注相同的样本空间，就遵守所有公理。例如，对于不相交的事件 \\(\mathcal{B}\\) 和 \\(\mathcal{B}'\\)，我们有 \\(P(\mathcal{B} \cup \mathcal{B}' \mid A = a) = P(\mathcal{B} \mid A = a) + P(\mathcal{B}' \mid A = a)\\)。

使用条件概率的定义，我们可以推导出著名的结果，称为 _贝叶斯定理_（Bayes' theorem）。根据构造，我们有 \\(P(A, B) = P(B\mid A) P(A)\\) 和 \\(P(A, B) = P(A\mid B) P(B)\\)。将两个方程组合得到 \\(P(B\mid A) P(A) = P(A\mid B) P(B)\\)，因此

(2.6.3)\\[P(A \mid B) = \frac{P(B\mid A) P(A)}{P(B)}.\\]

这个简单的方程有深远的影响，因为它允许我们反转条件的顺序。如果我们知道如何估计 \\(P(B\mid A)\\)、\\(P(A)\\) 和 \\(P(B)\\)，那么我们就可以估计 \\(P(A\mid B)\\)。我们经常发现直接估计一个项比另一个更容易，贝叶斯定理可以在这里派上用场。例如，如果我们知道某种疾病的症状流行率，以及该疾病和症状的总体流行率，我们就可以根据症状确定某人患该病的可能性。在某些情况下，我们可能无法直接获得 \\(P(B)\\)，例如症状的流行率。在这种情况下，贝叶斯定理的简化版本就派上用场了：

(2.6.4)\\[P(A \mid B) \propto P(B \mid A) P(A).\\]

因为我们知道 \\(P(A \mid B)\\) 必须归一化为 \\(1\\)，即 \\(\sum_a P(A=a \mid B) = 1\\)，我们可以用它来计算

(2.6.5)\\[P(A \mid B) = \frac{P(B \mid A) P(A)}{\sum_a P(B \mid A=a) P(A = a)}.\\]

在贝叶斯统计中，我们认为观察者拥有对可用假设的某种（主观的）先验信念，编码在 _先验_（prior）\\(P(H)\\) 中，以及一个 _似然函数_（likelihood function），它说明了在假设类中收集的证据的任何值的可能性有多大 \\(P(E \mid H)\\)。贝叶斯定理被解释为告诉我们如何根据可用证据 \\(E\\) 更新初始 _先验_ \\(P(H)\\) 以产生 _后验_（posterior）信念 \\(P(H \mid E) = \frac{P(E \mid H) P(H)}{P(E)}\\)。非正式地说，这可以表述为"后验等于先验乘以似然，再除以证据"。现在，由于证据 \\(P(E)\\) 对所有假设都是相同的，我们可以简单地在假设上进行归一化。

请注意，\\(\sum_a P(A=a \mid B) = 1\\) 也允许我们对随机变量进行 _边缘化_（marginalize）。也就是说，我们可以从联合分布（如 \\(P(A, B)\\)）中删除变量。毕竟，我们有

(2.6.6)\\[\sum_a P(B \mid A=a) P(A=a) = \sum_a P(B, A=a) = P(B).\\]

独立性（Independence）是另一个基本重要的概念，它构成了许多统计学重要思想的支柱。简而言之，如果以 \\(A\\) 的值为条件不会导致与 \\(B\\) 相关的概率分布发生任何变化，反之亦然，则两个变量是 _独立的_（independent）。更正式地说，独立性，记为 \\(A \perp B\\)，要求 \\(P(A \mid B) = P(A)\\)，因此 \\(P(A,B) = P(A \mid B) P(B) = P(A) P(B)\\)。独立性通常是一个合理的假设。例如，如果随机变量 \\(A\\) 代表抛一枚公平硬币的结果，随机变量 \\(B\\) 代表抛另一枚硬币的结果，那么知道 \\(A\\) 是否出现正面不应该影响 \\(B\\) 出现正面的概率。

当我们的数据从某个底层分布中连续抽取时保持独立性（使我们能够得出强有力的统计结论），或者当它在我们数据中的各种变量之间保持独立时（使我们能够使用更简单的编码这种独立性结构的模型），独立性特别有用。另一方面，估计随机变量之间的依赖关系通常是学习的真正目的。我们关心根据症状估计疾病的概率，正是因为我们认为疾病和症状 _不是_ 独立的。

请注意，由于条件概率是适当的概率，独立性和依赖性的概念也适用于它们。两个随机变量 \\(A\\) 和 \\(B\\) 在给定第三个变量 \\(C\\) 的情况下 _条件独立_（conditionally independent），当且仅当 \\(P(A, B \mid C) = P(A \mid C)P(B \mid C)\\)。有趣的是，两个变量在一般情况下可以是独立的，但在以第三个变量为条件时变得相关。当两个随机变量 \\(A\\) 和 \\(B\\) 对应于某个第三个变量 \\(C\\) 的原因时，这种情况经常发生。例如，骨折和肺癌在一般人群中可能是独立的，但如果我们以在医院为条件，那么我们可能发现骨折与肺癌呈负相关。这是因为骨折 _解释了_ 某人在医院的原因，从而降低了他们因肺癌住院的概率。

反过来，两个相关的随机变量在以第三个变量为条件时可以变得独立。当两个原本不相关的事件有共同原因时，这种情况经常发生。小学生的鞋码和阅读水平高度相关，但如果我们以年龄为条件，这种相关性就消失了。

## 2.6.5. 一个例子

让我们将我们的技能付诸实践。假设一位医生对患者进行了 HIV 检测。这个检测相当准确，如果患者是健康的但被报告为患病，即健康患者检测为阳性的概率只有 1%。此外，如果患者确实患有 HIV，它永远不会漏检。我们使用 \\(D_1 \in \\{0, 1\\}\\) 表示诊断结果（\\(0\\) 为阴性，\\(1\\) 为阳性），使用 \\(H \in \\{0, 1\\}\\) 表示 HIV 状态。

条件概率 | \\(H=1\\) | \\(H=0\\)
---|---|---
\\(P(D_1 = 1 \mid H)\\) | 1 | 0.01
\\(P(D_1 = 0 \mid H)\\) | 0 | 0.99

请注意，列总和都是 1（但行总和不是），因为它们是条件概率。让我们计算如果检测结果为阳性，患者确实患有 HIV 的概率，即 \\(P(H = 1 \mid D_1 = 1)\\)。直觉上这将取决于这种疾病有多常见，因为它影响了误报的数量。假设人群中这种疾病相当少见，例如 \\(P(H=1) = 0.0015\\)。为了应用贝叶斯定理，我们需要应用边缘化来确定

(2.6.7)\\[\begin{split}\begin{aligned} P(D_1 = 1) =& P(D_1=1, H=0) + P(D_1=1, H=1) \\\ =& P(D_1=1 \mid H=0) P(H=0) + P(D_1=1 \mid H=1) P(H=1) \\\ =& 0.011485. \end{aligned}\end{split}\\]

这导致我们

(2.6.8)\\[P(H = 1 \mid D_1 = 1) = \frac{P(D_1=1 \mid H=1) P(H=1)}{P(D_1=1)} = 0.1306.\\]

换句话说，即使检测相当准确，患者确实患有 HIV 的概率也只有 13.06%。正如我们所看到的，概率可能是反直觉的。患者收到如此可怕的消息后应该怎么做？很可能，患者会要求医生再做一次检测以获得清晰的结果。第二次检测有不同的特征，不如第一次好。

条件概率 | \\(H=1\\) | \\(H=0\\)
---|---|---
\\(P(D_2 = 1 \mid H)\\) | 0.98 | 0.03
\\(P(D_2 = 0 \mid H)\\) | 0.02 | 0.97

不幸的是，第二次检测结果也是阳性。让我们通过假设条件独立来计算调用贝叶斯定理所需的概率：

(2.6.9)\\[\begin{split}\begin{aligned} P(D_1 = 1, D_2 = 1 \mid H = 0) & = P(D_1 = 1 \mid H = 0) P(D_2 = 1 \mid H = 0) =& 0.0003, \\\ P(D_1 = 1, D_2 = 1 \mid H = 1) & = P(D_1 = 1 \mid H = 1) P(D_2 = 1 \mid H = 1) =& 0.98. \end{aligned}\end{split}\\]

现在我们可以应用边缘化来获得两次检测都呈阳性的概率：

(2.6.10)\\[\begin{split}\begin{aligned} &P(D_1 = 1, D_2 = 1)\\\ &= P(D_1 = 1, D_2 = 1, H = 0) + P(D_1 = 1, D_2 = 1, H = 1) \\\ &= P(D_1 = 1, D_2 = 1 \mid H = 0)P(H=0) + P(D_1 = 1, D_2 = 1 \mid H = 1)P(H=1)\\\ &= 0.00176955. \end{aligned}\end{split}\\]

最后，两次检测都呈阳性时患者患有 HIV 的概率为

(2.6.11)\\[P(H = 1 \mid D_1 = 1, D_2 = 1) = \frac{P(D_1 = 1, D_2 = 1 \mid H=1) P(H=1)}{P(D_1 = 1, D_2 = 1)} = 0.8307.\\]

也就是说，第二次检测使我们更有信心并非一切安好。尽管第二次检测的准确性远不如第一次，它仍然显著改善了我们的估计。两次检测条件独立的假设对于我们生成更准确估计的能力至关重要。考虑极端情况，我们运行相同的检测两次。在这种情况下，我们期望两次得到相同的结果，因此再次运行相同的检测不会获得额外的洞察力。敏锐的读者可能已经注意到，诊断的行为就像隐藏在眼前的分类器，随着我们获得更多特征（检测结果），我们判断患者是否健康的能力也在增强。

## 2.6.6. 期望（Expectations）

通常，做出决策不仅需要查看分配给各个事件的概率，还需要将它们组合成有用的聚合来为我们提供指导。例如，当随机变量取连续标量值时，我们通常关心知道 _平均_ 期望值是多少。这个量正式称为 _期望_（expectation）。如果我们正在进行投资，第一个感兴趣的量可能是我们可以期望的回报，对所有可能结果取平均（并以适当概率加权）。例如，假设投资有 50% 的概率完全失败，有 40% 的概率提供 2\\(\times\\) 回报，有 10% 的概率提供 10\\(\times\\) 回报。要计算期望回报，我们对所有回报求和，每个乘以其发生的概率。这产生了期望 \\(0.5 \cdot 0 + 0.4 \cdot 2 + 0.1 \cdot 10 = 1.8\\)。因此期望回报是 1.8\\(\times\\)。

一般来说，随机变量 \\(X\\) 的 _期望_（或平均值）定义为

(2.6.12)\\[E[X] = E_{x \sim P}[x] = \sum_{x} x P(X = x).\\]

同样，对于密度，我们得到 \\(E[X] = \int x \;dp(x)\\)。有时我们对 \\(x\\) 的某个函数的期望值感兴趣。我们可以如下计算这些期望：

(2.6.13)\\[E_{x \sim P}[f(x)] = \sum_x f(x) P(x) \textrm{ 和 } E_{x \sim P}[f(x)] = \int f(x) p(x) \;dx\\]

分别对应于离散概率和密度。回到上面的投资例子，\\(f\\) 可能是与回报相关的 _效用_（utility）（幸福感）。行为经济学家早就注意到，与赚一美元相比，人们对亏钱有更大的负效用。此外，金钱的价值往往是次线性的。拥有 10 万美元与零美元之间的差异可能是支付房租、吃得好、享受优质医疗保健与遭受无家可归之间的区别。另一方面，拥有 20 万与拥有 10 万之间的收益不那么显著。这样的推理推动了"金钱的效用是对数的"这一陈词滥调。

如果完全损失相关的效用是 \\(-1\\)，而回报 \\(1\\)、\\(2\\) 和 \\(10\\) 相关的效用分别是 \\(1\\)、\\(2\\) 和 \\(4\\)，那么投资的期望幸福感是 \\(0.5 \cdot (-1) + 0.4 \cdot 2 + 0.1 \cdot 4 = 0.7\\)（效用期望损失 30%）。如果这确实是你的效用函数，你最好把钱存银行。

对于财务决策，我们可能还想衡量一项投资有多 _风险_（risky）。在这里，我们不仅关心期望值，还关心实际值相对于这个值的 _变化_ 程度。请注意，我们不能简单地取实际值与期望值之差的期望。这是因为差值的期望等于期望的差值，即 \\(E[X - E[X]] = E[X] - E[E[X]] = 0\\)。然而，我们可以查看这个差值的任何非负函数的期望。随机变量的 _方差_（variance）通过查看 _平方_ 差值的期望值来计算：

(2.6.14)\\[\textrm{Var}[X] = E\left[(X - E[X])^2\right] = E[X^2] - E[X]^2.\\]

这里的等式通过展开 \\((X - E[X])^2 = X^2 - 2 X E[X] + E[X]^2\\) 并对每项取期望得到。方差的平方根是另一个有用的量，称为 _标准差_（standard deviation）。虽然它和方差传达相同的信息（可以从一个计算另一个），但标准差有一个很好的性质，即它以与随机变量所代表的原始量相同的单位表示。

最后，随机变量函数的方差类似地定义为

(2.6.15)\\[\textrm{Var}_{x \sim P}[f(x)] = E_{x \sim P}[f^2(x)] - E_{x \sim P}[f(x)]^2.\\]

回到我们的投资例子，我们现在可以计算投资的方差。它由 \\(0.5 \cdot 0 + 0.4 \cdot 2^2 + 0.1 \cdot 10^2 - 1.8^2 = 8.36\\) 给出。从各方面来看，这都是一项高风险投资。请注意，根据数学惯例，均值和方差通常引用为 \\(\mu\\) 和 \\(\sigma^2\\)。这在我们用它来参数化高斯分布时尤其如此。

就像我们为 _标量_ 随机变量引入期望和方差一样，我们也可以为向量值随机变量这样做。期望很容易，因为我们可以逐元素应用它们。例如，\\(\boldsymbol{\mu} \stackrel{\textrm{def}}{=} E_{\mathbf{x} \sim P}[\mathbf{x}]\\) 的坐标为 \\(\mu_i = E_{\mathbf{x} \sim P}[x_i]\\)。_协方差_（covariances）更复杂。我们通过取随机变量与其均值之差的 _外积_（outer product）的期望来定义它们：

(2.6.16)\\[\boldsymbol{\Sigma} \stackrel{\textrm{def}}{=} \textrm{Cov}_{\mathbf{x} \sim P}[\mathbf{x}] = E_{\mathbf{x} \sim P}\left[(\mathbf{x} - \boldsymbol{\mu}) (\mathbf{x} - \boldsymbol{\mu})^\top\right].\\]

这个矩阵 \\(\boldsymbol{\Sigma}\\) 被称为协方差矩阵。观察其效果的一种简单方法是考虑某个与 \\(\mathbf{x}\\) 大小相同的向量 \\(\mathbf{v}\\)。由此可得

(2.6.17)\\[\mathbf{v}^\top \boldsymbol{\Sigma} \mathbf{v} = E_{\mathbf{x} \sim P}\left[\mathbf{v}^\top(\mathbf{x} - \boldsymbol{\mu}) (\mathbf{x} - \boldsymbol{\mu})^\top \mathbf{v}\right] = \textrm{Var}_{x \sim P}[\mathbf{v}^\top \mathbf{x}].\\]

因此，\\(\boldsymbol{\Sigma}\\) 允许我们通过简单的矩阵乘法计算 \\(\mathbf{x}\\) 的任何线性函数的方差。非对角线元素告诉我们坐标之间的相关性：值为 0 表示不相关，较大的正值表示它们正相关性更强。

## 2.6.7. 讨论

在机器学习中，有很多事情是不确定的！我们可能对给定输入的标签值不确定。我们可能对参数的估计值不确定。我们甚至可能对部署时到达的数据是否与训练数据来自同一分布都不确定。

_偶然不确定性_（aleatoric uncertainty）是指问题固有的不确定性，由观测变量未能解释的真实随机性导致。_认知不确定性_（epistemic uncertainty）是指对模型参数的不确定性，是那种我们希望通过收集更多数据来减少的不确定性。我们可能对硬币出现正面的概率有认知不确定性，但即使我们知道了这个概率，我们对任何未来抛掷结果仍有偶然不确定性。无论我们看某人抛公平硬币多久，我们对下一次抛掷出现正面的确信度永远不会超过或低于 50%。这些术语来自机械建模（参见 Der Kiureghian 和 Ditlevsen ([2009](../chapter_references/zreferences.html#id56 "Der Kiureghian, A., & Ditlevsen, O. \(2009\). Aleatory or epistemic? does it matter? Structural Safety, 31\(2\), 105–112.")) 关于[不确定性量化](https://en.wikipedia.org/wiki/Uncertainty_quantification)这方面的综述）。然而，值得注意的是，这些术语构成了对语言的轻微滥用。术语 _认知_（epistemic）指任何与 _知识_ 有关的事物，因此从哲学意义上说，所有不确定性都是认知的。

我们看到从某个未知概率分布中采样数据可以为我们提供用于估计数据生成分布参数的信息。也就是说，这种可能性发生的速率可能相当慢。在我们的抛硬币例子（以及许多其他例子）中，我们所能做的最好就是设计收敛速率为 \\(1/\sqrt{n}\\) 的估计量，其中 \\(n\\) 是样本量（例如抛掷次数）。这意味着从 10 个观测到 1000 个观测（通常是非常可实现的任务），我们看到不确定性减少了十倍，而接下来的 1000 个观测帮助相对较小，仅提供 1.41 倍的减少。这是机器学习的一个持续特征：虽然通常有容易的收益，但需要非常大量的数据，以及通常随之而来的巨大计算量，才能获得进一步的收益。有关大规模语言模型这一事实的经验综述，请参见 Revels _et al._ ([2016](../chapter_references/zreferences.html#id236 "Revels, J., Lubin, M., & Papamarkou, T. \(2016\). Forward-mode automatic differentiation in Julia. ArXiv:1607.07892."))。

我们还磨练了统计建模的语言和工具。在这个过程中，我们学习了条件概率和统计学中最重要的方程之一——贝叶斯定理。它是通过似然项 \\(P(B \mid A)\\)（解决观测 \\(B\\) 与参数选择 \\(A\\) 的匹配程度）和先验概率 \\(P(A)\\)（控制 \\(A\\) 的特定选择最初的可信度）来解耦数据所传达信息的有效工具。特别是，我们看到了如何应用这条规则来根据检测的有效性 _和_ 疾病本身的流行率（即我们的先验）为诊断分配概率。

最后，我们引入了关于特定概率分布效果的第一组非平凡问题，即期望和方差。虽然概率分布不仅仅有线性和二次期望，但这两者已经提供了关于分布可能行为的大量知识。例如，[切比雪夫不等式](https://en.wikipedia.org/wiki/Chebyshev%27s_inequality)表明 \\(P(|X - \mu| \geq k \sigma) \leq 1/k^2\\)，其中 \\(\mu\\) 是期望，\\(\sigma^2\\) 是分布的方差，\\(k > 1\\) 是我们选择的置信参数。它告诉我们，从分布中抽取的值至少有 50% 的概率落在以期望为中心的 \\([-\sqrt{2} \sigma, \sqrt{2} \sigma]\\) 区间内。

## 2.6.8. 练习

1. 举一个例子，其中观察更多数据可以将关于结果的不确定性减少到任意低的水平。

2. 举一个例子，其中观察更多数据只能将不确定性减少到一定程度就不再减少。解释为什么会这样，以及你预期这种情况何时发生。

3. 我们凭经验展示了抛硬币均值的收敛。计算在抽取 \\(n\\) 个样本后我们看到正面的估计概率的方差。

   1. 方差如何随观测数量缩放？

   2. 使用切比雪夫不等式来约束与期望的偏差。

   3. 它与中心极限定理有何关系？

4. 假设我们从零均值和单位方差的概率分布中抽取 \\(m\\) 个样本 \\(x_i\\)。计算平均值 \\(z_m \stackrel{\textrm{def}}{=} m^{-1} \sum_{i=1}^m x_i\\)。我们能否对每个 \\(z_m\\) 独立应用切比雪夫不等式？为什么不能？

5. 给定两个概率为 \\(P(\mathcal{A})\\) 和 \\(P(\mathcal{B})\\) 的事件，计算 \\(P(\mathcal{A} \cup \mathcal{B})\\) 和 \\(P(\mathcal{A} \cap \mathcal{B})\\) 的上限和下限。提示：使用[维恩图](https://en.wikipedia.org/wiki/Venn_diagram)绘制情况。

6. 假设我们有一系列随机变量，比如 \\(A\\)、\\(B\\) 和 \\(C\\)，其中 \\(B\\) 只依赖于 \\(A\\)，\\(C\\) 只依赖于 \\(B\\)，你能简化联合概率 \\(P(A, B, C)\\) 吗？提示：这是一个[马尔可夫链](https://en.wikipedia.org/wiki/Markov_chain)。

7. 在第 2.6.5 节中，假设两次检测的结果不是独立的。特别地，假设每次检测的假阳性率为 10%，假阴性率为 1%。即，假设 \\(P(D =1 \mid H=0) = 0.1\\) 且 \\(P(D = 0 \mid H=1) = 0.01\\)。此外，假设对于 \\(H = 1\\)（感染）检测结果是条件独立的，即 \\(P(D_1, D_2 \mid H=1) = P(D_1 \mid H=1) P(D_2 \mid H=1)\\)，但对于健康患者，结果通过 \\(P(D_1 = D_2 = 1 \mid H=0) = 0.02\\) 耦合。

   1. 根据你目前掌握的信息，计算给定 \\(H=0\\) 时 \\(D_1\\) 和 \\(D_2\\) 的联合概率表。

   2. 推导一次检测返回阳性后患者患病（\\(H=1\\））的概率。你可以假设与之前相同的基线概率 \\(P(H=1) = 0.0015\\)。

   3. 推导两次检测都返回阳性后患者患病（\\(H=1\\））的概率。

8. 假设你是一家投资银行的资产经理，你可以选择股票 \\(s_i\\) 进行投资。你的投资组合需要加起来为 \\(1\\)，每只股票的权重为 \\(\alpha_i\\)。这些股票的平均回报为 \\(\boldsymbol{\mu} = E_{\mathbf{s} \sim P}[\mathbf{s}]\\)，协方差为 \\(\boldsymbol{\Sigma} = \textrm{Cov}_{\mathbf{s} \sim P}[\mathbf{s}]\\)。

   1. 计算给定投资组合 \\(\boldsymbol{\alpha}\\) 的期望回报。

   2. 如果你想最大化投资组合的回报，应该如何选择投资？

   3. 计算投资组合的 _方差_。

   4. 制定一个在将方差约束到上限的同时最大化回报的优化问题。这是获得诺贝尔奖的[马科维茨投资组合](https://en.wikipedia.org/wiki/Markowitz_model)（[Mangram, 2013](../chapter_references/zreferences.html#id184 "Mangram, M. E. \(2013\). A simplified perspective of the Markowitz portfolio theory. Global Journal of Business Research, 7\(1\), 59–70.")）。要解决它，你需要一个二次规划求解器，这远远超出了本书的范围。
