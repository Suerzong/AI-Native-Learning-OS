# 高级概念

这是构建 LLM 应用程序时经常遇到的高级概念的快速指南。

## 大语言模型（LLMs）

LLM 是推动 LlamaIndex 的基本创新。它们是一种人工智能（AI）计算机系统，能够理解、生成和操作自然语言，包括根据其训练数据或在查询时提供给它的数据回答问题。

## 智能体应用（Agentic Applications）

当 LLM 在应用程序中使用时，通常用于做出决策、采取行动和/或与世界交互。这是**智能体应用**的核心定义。

关键特征：
  * **LLM 增强**：LLM 通过工具、记忆和/或动态提示进行增强
  * **提示链**：多个 LLM 调用相互构建
  * **路由**：LLM 用于将应用路由到下一个适当的步骤或状态
  * **并行性**：应用可以并行执行多个步骤或行动
  * **编排**：使用 LLM 的层级结构来编排低级操作
  * **反思**：LLM 用于反思和验证先前步骤的输出

在 LlamaIndex 中，你可以使用 `Workflow` 类来编排步骤序列和 LLM。

## 智能体（Agents）

智能体是"智能体应用"的特定实例。智能体是半自主执行任务的软件，通过将 LLM 与其他工具和记忆结合，在推理循环中编排，决定接下来使用哪个工具。

## 检索增强生成（RAG）

RAG 是使用 LlamaIndex 构建数据支持的 LLM 应用程序的核心技术。它允许 LLM 通过在查询时将数据提供给 LLM 来回答关于你的私有数据的问题，而不是在你的数据上训练 LLM。

RAG 索引你的数据，并选择性地仅发送相关部分以及你的查询。

## 用例

  * **智能体**：由 LLM 驱动的自动化决策者，通过工具与世界交互
  * **工作流**：LlamaIndex 中特定的事件驱动抽象，允许编排步骤序列
  * **结构化数据提取**：使用 Pydantic 从非结构化源中提取结构化数据
  * **查询引擎**：端到端流程，允许你对数据提问
  * **聊天引擎**：端到端的多轮对话流程
