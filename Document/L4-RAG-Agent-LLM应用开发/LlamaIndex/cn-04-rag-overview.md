# RAG 简介

LLM 在大量数据上进行了训练，但它们没有在**你的**数据上训练。检索增强生成（RAG）通过将你的数据添加到 LLM 已经可以访问的数据中来解决这个问题。

在 RAG 中，你的数据被加载并准备好用于查询或"索引"。用户查询作用于索引，将你的数据过滤到最相关的上下文。然后这个上下文和你的查询与提示一起发送给 LLM，LLM 提供响应。

## RAG 中的阶段

RAG 有五个关键阶段：

  * **加载**：将数据从来源（文本文件、PDF、网站、数据库、API 等）获取到工作流中。[LlamaHub](https://llamahub.ai/) 提供数百个连接器。
  * **索引**：创建允许查询数据的数据结构。对于 LLM，这几乎总是意味着创建向量嵌入（vector embeddings）——数据含义的数值表示。
  * **存储**：索引数据后，通常需要存储索引和元数据，以避免重新索引。
  * **查询**：对于任何给定的索引策略，有多种方式可以利用 LLM 和 LlamaIndex 数据结构进行查询，包括子查询、多步骤查询和混合策略。
  * **评估**：检查效果相对于其他策略或当做出更改时的关键步骤。

## RAG 中的重要概念

### 加载阶段

  * **节点和文档**：`Document` 是任何数据源的容器（PDF、API 输出、数据库数据等）。`Node` 是 LlamaIndex 中数据的原子单位，表示源 `Document` 的一个"块"。
  * **连接器**：数据连接器（通常称为 `Reader`）从不同数据源和格式获取数据到 `Document` 和 `Node` 中。

### 索引阶段

  * **索引**：LlamaIndex 帮助你将数据索引为易于检索的结构。这通常涉及生成存储在称为向量存储（vector store）的专用数据库中的向量嵌入。
  * **嵌入**：LLM 生成数据的数值表示称为嵌入。LlamaIndex 将查询转换为嵌入，向量存储找到与查询嵌入数值相似的数据。

### 查询阶段

  * **检索器**：定义在给定查询时如何高效地从索引中检索相关上下文。
  * **路由器**：确定哪个检索器将用于从知识库中检索相关上下文。
  * **节点后处理器**：对检索到的节点应用转换、过滤或重新排序逻辑。
  * **响应合成器**：使用 LLM 根据用户查询和一组检索到的文本块生成响应。
