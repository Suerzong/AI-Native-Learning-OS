# 上下文工程（Context Engineering）

上下文工程是为智能体构建有效上下文的实践，确保模型在每次调用时获得正确的信息以产生高质量的响应。

## 什么是上下文工程？

上下文工程是为 AI 智能体构建有效上下文的过程。它不仅仅是编写提示词，而是关于确保模型在每次调用时都能获得正确的信息。

上下文的来源包括：

  * **系统提示（System Prompt）**——定义智能体的角色和行为
  * **对话历史（Message History）**——之前的交互记录
  * **检索结果（Retrieval Results）**——从外部数据源检索的文档
  * **工具描述（Tool Descriptions）**——可用工具的名称、描述和参数
  * **用户上下文（User Context）**——用户 ID、偏好、权限等
  * **运行时信息（Runtime Info）**——当前时间、环境变量等

## 上下文管理策略

### 消息修剪

当对话历史过长时，删除较旧的消息以适应模型的上下文窗口：

  * 基于 token 数量修剪
  * 保留系统提示和最近的消息
  * 摘要压缩较旧的消息

### 摘要

将较长的对话历史压缩为较短的摘要，保留关键信息：

  * 自动摘要：使用模型自动摘要旧消息
  * 手动摘要：使用中间件自定义摘要逻辑

### 动态上下文

使用中间件根据运行时条件动态构建上下文：

  * 根据用户角色调整系统提示
  * 根据对话阶段加载相关文档
  * 根据权限过滤可用工具

## LangChain 中的上下文工程

LangChain 提供多种工具来管理上下文：

  * **中间件（Middleware）**——在模型调用前后处理上下文
  * **动态提示（Dynamic Prompt）**——根据请求动态生成系统提示
  * **工具过滤（Tool Filtering）**——根据状态、上下文或权限过滤工具
  * **检索集成（Retrieval Integration）**——无缝集成外部数据源
